# tmpl-6: second make-template with the same name fails

Per SRD §10: make-template does NOT overwrite. Callers wanting to
clobber must delete the file and re-run.

## Setup

Per-case DB-reset.

## Steps

1. First `make-template --name tmpl6 --cwd /tmp` → exit 0.
2. Second `make-template --name tmpl6 --cwd /opt` → exit non-zero;
   stderr envelope's err_name is `ErrTemplateExists`.

## Pass criteria

- First call exits 0.
- Second call exits non-zero with `err_name=ErrTemplateExists`.
- Template file content is unchanged after the failed second call.

## Script

```bash
set -euo pipefail
agent-director make-template --name tmpl6 --cwd /tmp >/dev/null
target="$HOME/.agent-director/templates/tmpl6.toml"
before=$(cat "$target")

stderr_file=$(mktemp)
if agent-director make-template --name tmpl6 --cwd /opt 2>"$stderr_file" >/dev/null; then
    echo "FAIL: second make-template exited 0"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrTemplateExists" ]; then
    echo "FAIL: err_name=$err_name; want ErrTemplateExists"; exit 1
fi

after=$(cat "$target")
if [ "$before" != "$after" ]; then
    echo "FAIL: template body was modified by failing call"; exit 1
fi
echo "tmpl-6 ok: overwrite rejected; file untouched"
```
