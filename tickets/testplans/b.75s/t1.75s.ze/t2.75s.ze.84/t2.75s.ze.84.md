# tmpl-3: per-call --cwd overrides template

Per SRD §7.1: scalar per-call non-empty REPLACES the template value.

## Setup

Per-case DB-reset.

## Steps

1. `make-template --name tmpl3 --cwd /tmp --label env=dev`.
2. `spawn --template tmpl3 --cwd /var` → capture id.
3. `get` → cwd is `/var` (per-call wins); label `env=dev` still present
   (template merge).

## Pass criteria

- Row cwd = `/var`.
- Row labels still contain `env=dev`.

## Script

```bash
set -euo pipefail
agent-director make-template --name tmpl3 --cwd /tmp --label env=dev >/dev/null
id=$(agent-director spawn --template tmpl3 --cwd /var | jq -r '.claude_instance_id')
test -n "$id"
row=$(agent-director get --claude-instance-id "$id")
cwd=$(printf '%s' "$row" | jq -r '.cwd')
label=$(printf '%s' "$row" | jq -r '.labels.env // empty')
if [ "$cwd" != "/var" ]; then
    echo "FAIL: cwd=$cwd; want /var (per-call override)"; exit 1
fi
if [ "$label" != "dev" ]; then
    echo "FAIL: labels.env=$label; want dev (template fallback)"; exit 1
fi
echo "tmpl-3 ok: --cwd overrides template; labels still merged"
```
