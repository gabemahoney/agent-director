# release-pipeline-1-dry-run-end-to-end

Pins Epic 7 AC: `skills/release-agent-director/release.sh --dry-run`
from a clean checkout completes successfully with every phase
green. Also pins the phase-prefixed-logging AC by grepping for
`^\[preflight\]`, `^\[notes\]`, …, `^\[report\]` markers in the
output — every phase must contribute at least one line.

## Setup

No DB state. Operates on a per-case writable copy of
`/work/source` so the script's in-place mutations to `package.json`
files have somewhere to land. The script's EXIT trap then rolls
them back; the case asserts `git status --porcelain` is empty after
the run.

## Pass criteria

- Exit code 0.
- Output contains `[report] ==== dry-run OK ====`.
- Output contains every phase prefix: `[preflight]`, `[notes]`,
  `[build]`, `[verify]`, `[tag]`, `[publish]`, `[gh-release]`,
  `[report]`.
- After the run, `git status --porcelain` is empty (dry-run rolled
  back any mutations).
- Final line matches `release-pipeline-1 ok: ...`.

## Script

```bash
set -euo pipefail

# Copy the read-only source bind-mount into a writable workdir.
SRC=/work/source
WORK=$(mktemp -d)
trap 'rm -rf "$WORK"' EXIT
cp -a "$SRC/." "$WORK/"
cd "$WORK"

# Reinitialize git in the writable copy so the script's preflight
# checks (clean tree, branch, tag-not-exists) operate on a known
# baseline. We seed a single commit so `git rev-parse HEAD` works.
rm -rf .git
git init -q .
git config user.email "test@example.com"
git config user.name  "Test"
git checkout -q -b main
git add -A
git commit -q -m "release-pipeline test baseline"
# Seed a previous-release tag so notes-phase has a range to log.
git tag v0.0.1 HEAD

OUT=$(mktemp)
trap 'rm -rf "$WORK" "$OUT"' EXIT

# Exercise the script's dry-run path. AD_RELEASE_SKIP_CABI=1 skips
# the cabi-matrix lookup (which would fail inside the container —
# the writable copy has no green workflow run). --no-build skips
# `make release-binaries` so the case does not need the Go
# toolchain to be wired into the harness image.
if ! VERSION=v0.99.0 AD_RELEASE_SKIP_CABI=1 \
        bash skills/release-agent-director/release.sh \
        --dry-run --branch main --no-build > "$OUT" 2>&1; then
    echo "FAIL: release.sh --dry-run exited non-zero"
    sed 's/^/    /' "$OUT"
    exit 1
fi

# Every phase must contribute at least one line.
for phase in preflight notes build verify tag publish gh-release report; do
    if ! grep -q "^\[${phase}\]" "$OUT"; then
        echo "FAIL: no [${phase}] line in output"
        sed 's/^/    /' "$OUT"
        exit 1
    fi
done

# The final report must declare dry-run OK.
if ! grep -q '^\[report\] ==== dry-run OK ====$' "$OUT"; then
    echo "FAIL: missing '[report] ==== dry-run OK ====' marker"
    sed 's/^/    /' "$OUT"
    exit 1
fi

# Working tree must be clean after dry-run (rollback honored).
DIRTY=$(git -C "$WORK" status --porcelain)
if [[ -n "$DIRTY" ]]; then
    echo "FAIL: working tree dirty after dry-run:"
    printf '%s\n' "$DIRTY" | sed 's/^/    /'
    exit 1
fi

echo "release-pipeline-1 ok: every phase green, working tree clean"
```
