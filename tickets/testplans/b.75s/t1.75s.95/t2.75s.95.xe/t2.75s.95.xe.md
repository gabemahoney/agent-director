# Test case: tampered PRAGMA user_version surfaces ErrSchemaMismatch

Maps to Epic 1 AC #5.

## Setup

HOME is set to a fresh `$tmpdir`. `./bin/agent-director` exists. The
sqlite3 CLI is on PATH inside the container.

## Steps

1. Run `HOME=$tmpdir ./bin/agent-director help` to bootstrap the dir + db.
2. Confirm it succeeded (exit 0) and that `$tmpdir/.agent-director/state.db`
   exists.
3. Use `sqlite3 $tmpdir/.agent-director/state.db 'PRAGMA user_version = 99'`
   to tamper with the schema version.
4. Run `HOME=$tmpdir ./bin/agent-director help` again and capture stdout,
   stderr, exit code.
5. Parse stderr as JSON.

## Pass criteria

- Step 1 exits 0.
- Step 4 exits NON-ZERO.
- Step 4 stdout is empty.
- Step 4 stderr parses as JSON with `err_name == "ErrSchemaMismatch"` and a
  non-empty `err_description`.

## Notes

This protects against a binary running against a DB it doesn't understand.
The error is surfaced unconditionally — even on `help`, which doesn't
otherwise need to touch the DB. That's deliberate: the schema check is the
gate, and it must run on every invocation so an operator never silently
corrupts a future schema with an older binary.
