# subprocess-coverage-4-env-inherit

Per SRD §SR-1.4: the subprocess Client snapshots `process.env` at call
time and passes it through to the child via `Bun.spawn({env: …})`,
overlaying only HOME (when `storePath` follows the
`<home>/.agent-director/<file>` convention) and PATH (when
`tmuxCommand` is set). Consumer-process env vars must therefore reach
the subprocess unchanged.

The contract is independent of any CLI-side semantics of any
particular env var name: SR-1.4 is "what the consumer sets, the
subprocess inherits". The case proves that by exporting two probe
vars (`AGENT_DIRECTOR_STORE_PATH` and a generic `SUB_COV_PROBE`) in
the consumer, then exercising a stub CLI binary that echoes both back
to stdout as JSON fields. The Client returns the envelope to the
driver, which asserts the values round-tripped.

Using a stub binary keeps the test deterministic: a real CLI's
handling of `AGENT_DIRECTOR_STORE_PATH` is a separate concern (and
the CLI does not, today, honor a flag of that name). What this case
pins is solely subprocess env-snapshot inheritance.

## Setup

- `${HOME}=$(mktemp -d)`.
- `bun install file://...tgz` into `${HOME}/proj`.
- Stage a stub CLI binary that echoes `AGENT_DIRECTOR_STORE_PATH`
  and `SUB_COV_PROBE` from its environment back through a `version`
  envelope.

## Steps

1. The driver sets `process.env.AGENT_DIRECTOR_STORE_PATH = "/tmp/override.db"`
   and `process.env.SUB_COV_PROBE = "magic-probe-value"`.
2. The driver calls `client.version({})`.
3. The driver asserts the returned envelope reflects both values.

## Pass criteria

- `result.env_store_path === "/tmp/override.db"`.
- `result.env_probe === "magic-probe-value"`.

## Script

```bash
set -euo pipefail
HOME_DIR="$HOME"
PROJ="$HOME_DIR/proj"
rm -rf "$PROJ"
mkdir -p "$PROJ"
cd "$PROJ"

. /work/source/tickets/testplans/b.eiv/t1.19d.6c/build-tarball.sh

cat > "$PROJ/package.json" <<'EOF'
{
  "name": "subprocess-coverage-ev-consumer",
  "version": "0.0.0",
  "type": "module"
}
EOF

bun install "file://${TARBALL}" >/dev/null

DEST="$PROJ/node_modules/@agent-director/linux-x64"
mkdir -p "$DEST/bin"
cp "$PLATFORM_PKG_DIR/package.json" "$DEST/package.json"

# Stub CLI: echo two env vars back through a valid version envelope.
cat > "$DEST/bin/agent-director" <<'STUB_EOF'
#!/usr/bin/env bash
# JSON-escape any double quotes / backslashes in the env values; the test
# values are plain ASCII so a no-op printf is sufficient.
store_path="${AGENT_DIRECTOR_STORE_PATH:-}"
probe="${SUB_COV_PROBE:-}"
printf '{"version":"0.0.0-stub","commit":"stub","env_store_path":"%s","env_probe":"%s"}\n' \
    "$store_path" "$probe"
STUB_EOF
chmod +x "$DEST/bin/agent-director"

cp /work/source/tickets/testplans/b.eiv/t1.eiv.cv/t2.eiv.cv.ev/driver.mjs "$PROJ/driver.mjs"

out=$(bun run "$PROJ/driver.mjs" 2>&1)
echo "$out"
grep -q "OK env-inherit" <<<"$out" || { echo "FAIL: env-inherit assertion missing"; exit 1; }
echo "subprocess-coverage-4-env-inherit ok"
```
