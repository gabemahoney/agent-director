# Epic 7 — `epic-07-templates` testplan

t1 collector for Epic 7 (`t1.8op.9q` — Spawn templates: make-template +
--template). Each t2 is a plain-English case the Docker harness
executes. The Epic 7 Progression Contract requires every t2 to pass
under `make test-docker EPIC=epic-07-templates`.

## Sourced from

- Epic ticket: `t1.8op.9q` (Acceptance Criteria #1–#4).
- SRD §7.1 (per-call/template merge), §10 (template storage + safety),
  §13.1 (template errors).

## Cases

| case | what it pins |
|---|---|
| `tmpl-1-make-readable-toml` | `make-template --name dev --cwd /tmp --label project=foo --allow 'Bash(npm test)'` → file exists at `~/.claude-director/templates/dev.toml`; contents readable; round-trips through `claude-director help` (template-aware code paths still parse). |
| `tmpl-2-spawn-defaults-applied` | After `make-template dev`, `spawn --template dev` produces a row whose cwd matches the template; `get` confirms. |
| `tmpl-3-scalar-override` | `spawn --template dev --cwd /var` produces a row whose cwd is `/var` (per-call override); other fields still from template. |
| `tmpl-4-label-merge` | Template has `project=foo`; `spawn --template dev --label other=bar` produces a row whose labels JSON carries BOTH keys. |
| `tmpl-5-allow-concat` | Template has `permissions.allow=[A]`; `spawn --template dev --allow B` synthesizes a `--settings` with BOTH entries (verified by reading the tmux session env to find the synth'd settings, or by inspecting the row's claude_args containing the settings JSON). |
| `tmpl-6-second-make-fails` | Second `make-template --name dev ...` → ErrTemplateExists on stderr. |
| `tmpl-7-unsafe-name-rejected` | `make-template --name ../escape ...` → ErrTemplateNameUnsafe. |
| `tmpl-8-malformed-template` | Hand-edit a template file with an unknown top-level key; `spawn --template` → ErrTemplateMalformed. |

## Driver mode

All cases are written for `DRIVER_MODE=shell` — the orchestrator runs
the harness with `make test-docker EPIC=epic-07-templates`. Each t2
body has a single fenced ` ```bash ` block the driver executes after
the per-case DB-reset fixture.

The DB-reset fixture deliberately leaves
`~/.claude-director/templates/` alone (it only removes `state.db`),
so a case can write a template and use it in the same case. To avoid
cross-case bleed each case uses unique template names (e.g.
`tmpl1`, `tmpl2`, …); a stray template from one case is invisible to
the next because the cases reference names by exact match.

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-07-templates`
  yourself, inspect the JSON stream case-by-case.
- Cases 2 / 3 / 4 / 5 use the real `spawn` pipeline. tmux launches a
  `claude` process inside the session but no conversation is driven;
  the row inspection covers the merge behavior.
- Case 5's `--settings` proof reads the row's `claude_args` column —
  the synthesized `--settings <json>` argv pair is the second + third
  element. Decoding the JSON exposes the merged permissions arrays.

## tmux session sweep

Cases that exercise real `spawn` (2 / 3 / 4 / 5) leave sessions under
the `cd-*` prefix. The DB-reset fixture's `cd-*` sweep cleans them.
