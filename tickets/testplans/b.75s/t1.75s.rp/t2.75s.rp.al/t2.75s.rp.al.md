# release-pipeline-5-asset-list-shape

Pins Epic 7 AC: the gh-release phase attaches exactly **4 CLI
binaries + 3 cabi shared libs + 1 canonical header = 8 assets**
on every release. Verified by parsing the dry-run gh-release
"would attach" log lines.

## Setup

Writable copy of `/work/source`, dry-run mode, AD_RELEASE_SKIP_CABI
to avoid the build phase, --no-build for the same reason. The
asset-list emission happens regardless of whether the artifacts
actually exist on disk: `release_assets()` enumerates the canonical
asset paths and ghrelease_phase's dry-run branch logs each one.

## Pass criteria

- Exit code 0 (dry-run succeeds).
- The output contains exactly four CLI-binary asset names:
  `agent-director-linux-amd64`, `agent-director-linux-arm64`,
  `agent-director-darwin-amd64`, `agent-director-darwin-arm64`.
- Exactly three cabi shared-library asset names:
  `libagent_director.so` (linux-amd64),
  `libagent_director.dylib` × 2 (darwin-amd64, darwin-arm64).
- Exactly one canonical header asset:
  `dist/cabi/include/libagent_director.h`.
- The `[gh-release] (dry-run) would attach 8 assets:` count line
  reports 8.
- Final line matches `release-pipeline-5 ok: ...`.

## Script

```bash
set -euo pipefail

SRC=/work/source
WORK=$(mktemp -d)
OUT=$(mktemp)
trap 'rm -rf "$WORK" "$OUT"' EXIT
cp -a "$SRC/." "$WORK/"
cd "$WORK"
rm -rf .git
git init -q .
git config user.email "test@example.com"
git config user.name  "Test"
git checkout -q -b main
git add -A
git commit -q -m "release-pipeline test baseline"
git tag v0.0.1 HEAD

if ! VERSION=v0.99.0 AD_RELEASE_SKIP_CABI=1 \
        bash skills/release-agent-director/release.sh \
        --dry-run --branch main --no-build > "$OUT" 2>&1; then
    echo "FAIL: release.sh --dry-run exited non-zero"
    sed 's/^/    /' "$OUT"
    exit 1
fi

# The dry-run gh-release line summarising the asset count.
if ! grep -qE '^\[gh-release\] \(dry-run\) would attach 8 assets:' "$OUT"; then
    echo "FAIL: expected '[gh-release] (dry-run) would attach 8 assets:' line"
    grep '^\[gh-release\]' "$OUT" | sed 's/^/    /'
    exit 1
fi

# Required CLI binaries (exactly four).
for cli in agent-director-linux-amd64 agent-director-linux-arm64 \
           agent-director-darwin-amd64 agent-director-darwin-arm64; do
    if ! grep -qE "^\[gh-release\] +${cli} " "$OUT"; then
        echo "FAIL: missing CLI binary $cli in gh-release asset list"
        grep '^\[gh-release\]' "$OUT" | sed 's/^/    /'
        exit 1
    fi
done

# Cabi shared libraries: one .so on linux-amd64, two .dylib on darwin-*.
SO_COUNT=$(grep -cE '^\[gh-release\] +libagent_director\.so ' "$OUT" || true)
DYLIB_COUNT=$(grep -cE '^\[gh-release\] +libagent_director\.dylib ' "$OUT" || true)
if [[ "$SO_COUNT" -ne 1 ]]; then
    echo "FAIL: expected exactly 1 libagent_director.so line, got $SO_COUNT"
    exit 1
fi
if [[ "$DYLIB_COUNT" -ne 2 ]]; then
    echo "FAIL: expected exactly 2 libagent_director.dylib lines, got $DYLIB_COUNT"
    exit 1
fi

# Canonical header path.
if ! grep -qE '^\[gh-release\] +libagent_director\.h \(dist/cabi/include/libagent_director\.h\)' "$OUT"; then
    echo "FAIL: missing canonical header dist/cabi/include/libagent_director.h"
    grep '^\[gh-release\]' "$OUT" | sed 's/^/    /'
    exit 1
fi

echo "release-pipeline-5 ok: 4 CLI + 3 cabi libs + 1 header attached"
```
