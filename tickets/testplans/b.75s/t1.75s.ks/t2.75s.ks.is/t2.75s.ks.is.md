# kill-2: kill on an ended row is a no-op success

Per SRD §5 + Epic 5 task ticket: terminal states (ended/missing) are
idempotent no-ops. The verb returns 0 without invoking tmux.

## Setup

Per-case DB-reset. No tmux session — the row's state is `ended`, so
nothing to kill.

## Steps

1. Seed `spawns` row at state `ended`, session_name `cd-k2`.
2. `claude-director kill --claude-instance-id id-k2`.

## Pass criteria

- kill exits 0.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-k2', 'ended', '/tmp', 'cd-k2', 'off');"
claude-director kill --claude-instance-id id-k2 >/dev/null
echo "kill-2 ok: ended row → kill → exit 0 (no-op)"
```
