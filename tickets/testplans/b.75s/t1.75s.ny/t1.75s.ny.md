# template-overwrite testplan

t1 collector for the v0.4.3 `MakeTemplate --overwrite` flag (work bee
`b.q96`, Epic 2 `t1.q96.zs`). Each t2 is a plain-English case the
Docker harness executes under `make test-docker EPIC=template-overwrite`
with `DRIVER_MODE=shell` against a clean linux/x86_64 container.

## Sourced from

- Work bee: `b.q96` (MakeTemplate overwrite flag plan).
- Work Epic: `t1.q96.zs` (cross-channel parity + coverage + docs).
- PRD acceptance criteria 1, 2, 3 + edge case (concurrent writers).

## Cases

| case | what it pins | maps to |
|---|---|---|
| `overwrite-1-replaces-existing` | second `make-template --name foo --overwrite` replaces the on-disk body of an existing template | PRD AC 1 |
| `overwrite-2-no-overwrite-still-errors` | second `make-template --name foo` (no flag) exits non-zero with `ErrTemplateExists`; on-disk content unchanged | PRD AC 2 |
| `overwrite-3-creates-when-absent` | `make-template --name fresh --overwrite` against empty templates dir succeeds and creates the file | PRD AC 3 |
| `overwrite-4-concurrent-atomicity` | two background `make-template --name racer --overwrite` writers race; on-disk body matches one of the two inputs (no zero-byte, no half-written body); 3 iterations | PRD edge case |
