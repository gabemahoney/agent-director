# mcp-7: unknown JSON-RPC method returns -32601

A request whose method isn't one the server implements must return
the JSON-RPC standard "method not found" error code.

## Setup

Per-case DB-reset.

## Steps

1. Pipe initialize + a bogus method into the server.
2. Assert the second response has error.code = -32601.

## Pass criteria

- error.code = -32601.

## Script

```bash
set -euo pipefail
out=$(printf '%s
%s
' \
    '{"jsonrpc":"2.0","id":1,"method":"initialize"}' \
    '{"jsonrpc":"2.0","id":2,"method":"totally/made_up"}' \
    | claude-director serve --stdio 2>/dev/null)
code=$(printf '%s' "$out" | sed -n '2p' | jq -r '.error.code')
if [ "$code" != "-32601" ]; then
    echo "FAIL: code=$code; want -32601 (method not found)"; exit 1
fi
echo "mcp-7 ok: unknown method → -32601"
```
