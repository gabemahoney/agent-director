# release-2: magic bytes match the expected per-OS format

Per SRD §16.1: linux/* must be ELF (magic 7f 45 4c 46); darwin/*
must be Mach-O 64 LE (magic cf fa ed fe). Reading the magic bytes
directly via `od` avoids depending on `file(1)`.

## Setup

Per-case DB-reset.

## Steps

1. Build via copy-source + make release-binaries.
2. For each artifact, read the first 4 bytes via od and check
   against the expected magic.

## Pass criteria

- linux-{amd64,arm64} have ELF magic.
- darwin-arm64 has Mach-O 64 LE magic.
- darwin-amd64 was dropped from v1 on 2026-05-24 and is not built.

## Script

```bash
set -euo pipefail
work=$(mktemp -d)
cp -a /work/source/* "$work/"
cd "$work"
make release-binaries >/dev/null 2>&1

check_magic() {
    local out="$1" want="$2"
    local got=$(od -A n -t x1 -N 4 "$out" | tr -d ' ')
    if [ "$got" != "$want" ]; then
        echo "FAIL: $out magic=$got; want $want"; exit 1
    fi
}

check_magic dist/agent-director-linux-amd64  7f454c46
check_magic dist/agent-director-linux-arm64  7f454c46
check_magic dist/agent-director-darwin-arm64 cffaedfe

# darwin/amd64 was dropped from v1 (2026-05-24); the Makefile no longer
# emits it. Treat its presence as a regression.
if [ -e dist/agent-director-darwin-amd64 ]; then
    echo "FAIL: dist/agent-director-darwin-amd64 should not exist (dropped from v1 2026-05-24)"
    exit 1
fi
echo "release-2 ok: ELF / Mach-O 64 LE magic bytes match"
```
