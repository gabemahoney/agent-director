# mcp-3: tools/list includes key verbs with underscore naming

MCP tool names use underscores; manifest verb names use hyphens. This
case pins the mapping for the verbs that have multi-word names:
`send_keys`, `read_pane`, `find_missing`, `make_template`. Plus a
sanity check that common single-word verbs (`spawn`, `decide`) appear
without modification.

## Setup

Per-case DB-reset.

## Steps

1. Pipe initialize + tools/list.
2. Extract the tool names array.
3. Assert each expected name is present.

## Pass criteria

- All of `spawn`, `status`, `send_keys`, `read_pane`, `decide`,
  `find_missing`, `make_template`, `kill`, `pause`, `resume`,
  `list`, `expire`, `delete`, `get`, `help`, `version` appear.
- Neither `hook` nor `serve` appears.

## Script

```bash
set -euo pipefail
out=$(printf '%s
%s
' \
    '{"jsonrpc":"2.0","id":1,"method":"initialize"}' \
    '{"jsonrpc":"2.0","id":2,"method":"tools/list"}' \
    | agent-director serve --stdio 2>/dev/null)
names=$(printf '%s' "$out" | sed -n '2p' | jq -r '.result.tools[].name' | sort)

want="decide
delete
expire
find_missing
get
help
kill
list
make_template
pause
read_pane
resume
send_keys
spawn
status
version"

if [ "$names" != "$want" ]; then
    echo "FAIL: names differ"
    diff <(printf '%s' "$names") <(printf '%s' "$want") || true
    exit 1
fi
echo "mcp-3 ok: tools/list contains the expected 16 underscore-named tools"
```
