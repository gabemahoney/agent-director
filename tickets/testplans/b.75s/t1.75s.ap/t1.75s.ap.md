## Driver mode

All cases use `DRIVER_MODE=shell`. Each t2 body has a single fenced `bash`
block executed by the driver after the per-case DB-reset fixture.

Cases 1, 2, 3, 5 build / inspect the source tree. They copy `/work/source`
to a writable temp directory under `/tmp/` and operate from there;
`/work/source` itself is never written. Case 4 only reads
`/work/source/pkg/api/errnames/catalog.json` — no copy required.

## Notes for the orchestrator

- Audit: `make test-docker EPIC=pkg-api-facade`; inspect the JSON stream
  case-by-case.
- All five cases need `go` in PATH (already present in `test/Dockerfile`).
  Cases 4 and 5 also need `jq` (already present).
- None of these cases use `agent-director spawn` or tmux directly. The
  db-reset fixture's `cd-*` session sweep is a no-op for all five cases.
- Case 1 deliberately scopes the test run to `./pkg/api/...` rather than
  `./...` — the full suite hits tmux / sqlite hot paths that need their own
  fixture setup. The pkg/api package tests cover the refactored surface
  (manifest, errnames, methods, public_surface, examples) and are the
  strongest signal a post-refactor regression would trip. `go build ./...`
  catches whole-tree compile breaks; together they pin the "build is green
  after the refactor" AC.

## tmux session sweep

No `cd-*` sessions are created. The db-reset fixture's session sweep is a
no-op for these cases.
