# pkg-api-1-cli-still-green

Pins E1 AC: the post-refactor build is green. The Epic refactored
`internal/api` into the exported `pkg/api` package and threaded every CLI
+ MCP verb through `pkg/api.Client`; that refactor is regressed by any
whole-tree compile break or any failure of the pkg/api unit tests.

This case runs the two strongest signals:

1. `go build ./...` — every package in the module compiles. Catches a
   bad import path or a broken signature anywhere in the tree.
2. `go test -count=1 ./pkg/api/...` — the refactored surface's unit
   tests (manifest, errnames, public_surface, examples, methods, params)
   pass. These cover the AC bodies of E1 directly.

We deliberately do NOT run the full `go test ./...` here — the cross-
package suite exercises tmux + sqlite hot paths that have their own
fixture setup and would balloon this case's wall-clock without adding
signal beyond what cases 2/3/5 already pin via static checks.

## Setup

No DB state needed. `go test` writes to its module cache and may emit
test-binary artifacts, so the script copies `/work/source` to a writable
temp dir under `/tmp/` before invoking `go build` / `go test`.

## Steps

1. Assert `go` is in PATH; fail with a clear message if not.
2. Create a writable workspace under `/tmp/` and `cp -a /work/source/.` into it.
3. `cd <workspace> && go build ./...`; assert exit 0.
4. `cd <workspace> && go test -count=1 ./pkg/api/...`; assert exit 0.

## Pass criteria

- `go build ./...` exits 0.
- `go test -count=1 ./pkg/api/...` exits 0.
- Script exits 0; final line matches `pkg-api-1-cli-still-green ok: ...`.

## Script

```bash
set -euo pipefail

if ! command -v go >/dev/null 2>&1; then
    echo "FAIL: go not found in PATH"
    exit 1
fi

WORK=$(mktemp -d /tmp/pkgapi1_XXXXXX)
cp -a /work/source/. "$WORK/"

cd "$WORK"

BUILD_OUT=$(mktemp /tmp/pkgapi1_build.XXXXXX)
BUILD_EXIT=0
go build ./... > "$BUILD_OUT" 2>&1 || BUILD_EXIT=$?
if [[ "$BUILD_EXIT" -ne 0 ]]; then
    echo "FAIL: go build ./... exited with code $BUILD_EXIT"
    tail -200 "$BUILD_OUT" >&2
    exit 1
fi

TEST_OUT=$(mktemp /tmp/pkgapi1_test.XXXXXX)
TEST_EXIT=0
go test -count=1 ./pkg/api/... > "$TEST_OUT" 2>&1 || TEST_EXIT=$?
if [[ "$TEST_EXIT" -ne 0 ]]; then
    echo "FAIL: go test -count=1 ./pkg/api/... exited with code $TEST_EXIT"
    tail -200 "$TEST_OUT" >&2
    exit 1
fi

# Sanity: ensure at least one PASS line appeared so '-count=1 ./pkg/api/...'
# matching nothing doesn't silently pass.
PASS_COUNT=$(grep -cE '^ok\s+github\.com/gabemahoney/agent-director/pkg/api' "$TEST_OUT" || true)
if [[ "$PASS_COUNT" -lt 1 ]]; then
    echo "FAIL: no 'ok' lines for pkg/api in go test output — did the package selector match anything?"
    tail -50 "$TEST_OUT" >&2
    exit 1
fi

echo "pkg-api-1-cli-still-green ok: go build ./... clean, $PASS_COUNT pkg/api package(s) tested green"
```
