# ci-matrix-2-three-legs-with-expected-runs-on

Pins Epic 6 AC: the workflow declares exactly three matrix legs
`linux-amd64`, `darwin-amd64`, and `darwin-arm64`, whose `runs-on`
values are respectively `ubuntu-22.04`, `macos-13`, and the full
self-hosted label tuple `[self-hosted, macOS, ARM64, darwin-arm64]`.
Bare `self-hosted` is intentionally forbidden — that would let the
workflow route to a different self-hosted runner the operator may
add later.

## Setup

No DB state. The check is a pure inspection of the committed file at
`/work/source/.github/workflows/cabi-matrix.yml`.

## Steps

1. Assert the workflow file exists.
2. Assert each of the three `name:` declarations is present.
3. Pair each `name:` with its immediately-following `runs-on:` (within
   the next 5 lines, to absorb intervening comments) and verify the
   declared runner image / label tuple.
4. Spot-check that the `darwin-arm64` runs-on declares all four
   labels in the conjunctive tuple.
5. Emit pass marker.

## Pass criteria

- `name: linux-amd64` is paired with `runs-on: ubuntu-22.04`.
- `name: darwin-amd64` is paired with `runs-on: macos-13`.
- `name: darwin-arm64` is paired with a `runs-on:` line referencing
  all four of `self-hosted`, `macOS`, `ARM64`, and `darwin-arm64`.
- Final line matches `ci-matrix-2 ok: ...`.

## Script

```bash
set -euo pipefail

YML="/work/source/.github/workflows/cabi-matrix.yml"
if [[ ! -f "$YML" ]]; then
    echo "FAIL: $YML not found"
    exit 1
fi

# Each leg has a `- name: <leg>` line followed by a `runs-on:` line
# within 5 lines. We capture the runs-on line that pairs with each
# name and assert its contents.

pair_runs_on() {
    local name="$1"
    grep -A 5 -F "name: ${name}" "$YML" | grep -E '^[[:space:]]*runs-on:' | head -n 1
}

linux_runs=$(pair_runs_on linux-amd64)
[[ "$linux_runs" == *"ubuntu-22.04"* ]] \
    || { echo "FAIL: linux-amd64 runs-on missing ubuntu-22.04: ${linux_runs:-<empty>}"; exit 1; }

dx64_runs=$(pair_runs_on darwin-amd64)
[[ "$dx64_runs" == *"macos-13"* ]] \
    || { echo "FAIL: darwin-amd64 runs-on missing macos-13: ${dx64_runs:-<empty>}"; exit 1; }

darm_runs=$(pair_runs_on darwin-arm64)
for label in "self-hosted" "macOS" "ARM64" "darwin-arm64"; do
    if [[ "$darm_runs" != *"$label"* ]]; then
        echo "FAIL: darwin-arm64 runs-on missing required label '$label': ${darm_runs:-<empty>}"
        exit 1
    fi
done

# Belt-and-suspenders: a bare `runs-on: self-hosted` on the
# darwin-arm64 leg would slip past the per-label loop above (every
# substring matches the unquoted form). Verify the runs-on is a
# bracketed list, not a bare scalar.
if [[ "$darm_runs" != *"["* || "$darm_runs" != *"]"* ]]; then
    echo "FAIL: darwin-arm64 runs-on not declared as a label-tuple list: ${darm_runs:-<empty>}"
    exit 1
fi

echo "ci-matrix-2 ok: three legs with expected runs-on values"
```
