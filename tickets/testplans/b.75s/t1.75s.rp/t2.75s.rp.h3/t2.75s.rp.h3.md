# release-pipeline-2-h3-halt-in-live-mode

Pins Epic 7 AC: live (`--release`) run halts cleanly when any of the four
published `package.json` files re-acquires the `@CHANGEME-H3/...` placeholder,
before any `npm publish` runs, with a clear error message naming H3 and listing
every offending `package.json` file.

H3 was resolved on 2026-05-24, so the committed sources no longer carry the
placeholder. This case **synthetically re-introduces** the placeholder into a
copy of the worktree (the real source is never mutated) and asserts the
publish-phase tripwire still fires. The tripwire is exactly the regex
`^@?(CHANGEME-H3|TBD)/` in `skills/release-agent-director/release.sh`.

## Setup

Same writable copy of `/work/source` as case 1. The script mutates the copy's
four `package.json` files to put `@CHANGEME-H3/...` back in the `name` field,
then runs release.sh in live mode. Tests halt path so no `npm publish`
actually runs (the script exits before that step).

## Pass criteria

- Exit code 6 (publish-phase failure).
- Output contains `[publish] H3 unresolved: claim npm name first`.
- Output names all four placeholder package.jsons.
- Output does NOT contain `+ @CHANGEME-H3/agent-director` (the
  marker `npm publish` would emit if it had actually run).
- Final line matches `release-pipeline-2 ok: ...`.

## Script

```bash
set -euo pipefail

SRC=/work/source
WORK=$(mktemp -d)
ORIGIN=$(mktemp -d)
OUT=$(mktemp)
trap 'rm -rf "$WORK" "$ORIGIN" "$OUT"' EXIT
cp -a "$SRC/." "$WORK/"
cd "$WORK"

# Re-introduce the H3 placeholder synthetically into the copy. The real
# worktree under /work/source is read-only; only this temp copy is mutated.
# We rewrite the `name` field in each of the four package.jsons to match
# the placeholder pattern the publish-phase tripwire detects. Each gets a
# distinct suffix so the halt output enumerates four files.
sed -i.bak -E 's/("name":[[:space:]]*)"agent-director"/\1"@CHANGEME-H3\/agent-director"/' \
    pkg/ts-bun-client/package.json
sed -i.bak -E 's/("name":[[:space:]]*)"@agent-director\/linux-x64"/\1"@CHANGEME-H3\/agent-director-linux-x64"/' \
    pkg/ts-bun-client/platforms/linux-x64/package.json
sed -i.bak -E 's/("name":[[:space:]]*)"@agent-director\/darwin-x64"/\1"@CHANGEME-H3\/agent-director-darwin-x64"/' \
    pkg/ts-bun-client/platforms/darwin-x64/package.json
sed -i.bak -E 's/("name":[[:space:]]*)"@agent-director\/darwin-arm64"/\1"@CHANGEME-H3\/agent-director-darwin-arm64"/' \
    pkg/ts-bun-client/platforms/darwin-arm64/package.json
rm -f pkg/ts-bun-client/package.json.bak \
      pkg/ts-bun-client/platforms/linux-x64/package.json.bak \
      pkg/ts-bun-client/platforms/darwin-x64/package.json.bak \
      pkg/ts-bun-client/platforms/darwin-arm64/package.json.bak

# Sanity: all four package.jsons must now carry the sentinel; if any sed
# failed silently the case below would mis-report a regression.
for f in \
    pkg/ts-bun-client/package.json \
    pkg/ts-bun-client/platforms/linux-x64/package.json \
    pkg/ts-bun-client/platforms/darwin-x64/package.json \
    pkg/ts-bun-client/platforms/darwin-arm64/package.json; do
    if ! grep -q '"name":[[:space:]]*"@\?CHANGEME-H3/' "$f"; then
        echo "FAIL: sed-injected placeholder did not stick in $f"
        sed -n '1,8p' "$f" >&2
        exit 1
    fi
done

rm -rf .git
git init -q .
git config user.email "test@example.com"
git config user.name  "Test"
git checkout -q -b main
git add -A
git commit -q -m "release-pipeline test baseline"
git tag v0.0.1 HEAD

# Live mode needs a real `origin` to push the tag to: stand up a
# local bare repo and wire it in. (The tag phase runs BEFORE the
# publish phase; without a working origin the script would fail
# at exit 7 in tag-phase, never reaching the H3 publish halt
# we're trying to pin.)
git init -q --bare "$ORIGIN"
git remote add origin "$ORIGIN"

# Live mode requires NPM_TOKEN; supply a dummy so we exercise the
# H3 halt rather than the missing-token preflight halt.
set +e
NPM_TOKEN=dummy VERSION=v0.99.0 AD_RELEASE_SKIP_CABI=1 \
    bash skills/release-agent-director/release.sh \
    --release --branch main --no-build > "$OUT" 2>&1
RC=$?
set -e

if [[ "$RC" -ne 6 ]]; then
    echo "FAIL: expected exit code 6 (publish-phase H3 halt), got $RC"
    sed 's/^/    /' "$OUT"
    exit 1
fi

if ! grep -q '\[publish\] H3 unresolved: claim npm name first' "$OUT"; then
    echo "FAIL: missing '[publish] H3 unresolved' message"
    sed 's/^/    /' "$OUT"
    exit 1
fi

# All four package.jsons must be named in the error output.
for f in \
    pkg/ts-bun-client/package.json \
    pkg/ts-bun-client/platforms/linux-x64/package.json \
    pkg/ts-bun-client/platforms/darwin-x64/package.json \
    pkg/ts-bun-client/platforms/darwin-arm64/package.json; do
    if ! grep -qF "$f" "$OUT"; then
        echo "FAIL: H3 halt did not name $f"
        sed 's/^/    /' "$OUT"
        exit 1
    fi
done

# Belt-and-suspenders: no actual `npm publish` line.
if grep -q '^\+ @CHANGEME-H3/agent-director' "$OUT"; then
    echo "FAIL: npm publish ran despite H3 placeholder"
    sed 's/^/    /' "$OUT"
    exit 1
fi

echo "release-pipeline-2 ok: live run halted on H3 before any npm publish"
```
