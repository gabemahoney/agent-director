# Envelope-diff (`envelope-diff`) testplan

t1 collector for the envelope-diff harness (`test/envelope-diff/` — Epic 3 of
`b.qe2`). Each t2 is a plain-shell case the Docker harness executes. Runs under
`make test-docker EPIC=envelope-diff`; the driver resolves the slug by
case-insensitive substring match against the t1 frontmatter title (see
`test/driver/run-testplan.sh`), so the canonical substring `envelope-diff` must
remain in the title verbatim.

## Sourced from

- Source Epic: `t1.qe2.pc` (envelope-diff regression harness acceptance criteria).
- Testplan-adding Task: `t2.qe2.pc.by`.

## Tests included

| t2 | what it pins |
|---|---|
| `env-diff-1-all-callable-verbs-match` | `go test ./test/envelope-diff/...` against a clean baseline exits 0 — every callable verb's CLI and Client envelopes are structurally identical after normalization and nondeterministic-field suppression. |
| `env-diff-2-injected-drift-fails` | A `sed`-injected stray field in a CLI verb's marshal path causes `TestEnvelopeDiff_Success` to exit non-zero and report the differing field, proving the harness catches envelope drift. |
| `env-diff-3-manifest-coverage-gate` | Deleting a verb key from `nondeterministic.json` causes `make nondet-coverage` to exit non-zero with stderr naming the missing verb, proving the coverage gate fires on under-specified manifests. |
| `env-diff-4-doc-error-prefix-parity` | `go test -run TestEnvelopeDiff_Error ./test/envelope-diff/` passes against a clean baseline — every documented error case produces matching `err_name` and prefix-compatible `err_description` on both CLI and Client paths. |

## Driver mode

All cases use `DRIVER_MODE=shell`. Each t2 body carries a single fenced `bash`
block executed by the driver after the per-case DB-reset fixture.

Cases 1, 2, and 4 build the CLI binary (`go build ./cmd/agent-director`) as a
setup step inside the script. Cases 2 and 3 mutate source files and must
restore them unconditionally via `trap` to keep the working tree clean.

## Notes for the orchestrator

- Audit: `make test-docker EPIC=envelope-diff`; inspect the JSON stream case-by-case.
- The Dockerfile already includes `gcc`, `binutils`, `jq`, and `sqlite3` (added in Epic 2 T8).
- None of these cases require tmux running. The db-reset fixture's session sweep is a no-op for all four cases.
- Case 2 uses `sed -i` to inject a stray JSON field; the trap ensures the file is restored even on failure.
- Case 3 mutates `test/envelope-diff/nondeterministic.json` in-place via `jq` and restores via backup.
