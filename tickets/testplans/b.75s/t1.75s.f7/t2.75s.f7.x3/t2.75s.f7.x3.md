# install-4: missing tmux on PATH aborts cleanly

Per SRD §16.2 step 1: pre-flight rejects missing dependencies.

## Setup

Per-case DB-reset. PATH stripped of tmux for the install.sh
invocation only (the rest of the harness still has tmux).

## Steps

1. Invoke install.sh with a PATH that doesn't include tmux.
2. Capture exit code + stderr.
3. Verify exit code is 2 (pre-flight failure).
4. Verify stderr names "tmux" as the missing tool.

## Pass criteria

- Exit code is 2.
- Stderr contains "tmux".

## Script

```bash
set -euo pipefail
TESTHOME=$(mktemp -d)
NARROW_PATH="/usr/bin:/bin"  # claude is in /usr/local/bin; tmux is in /usr/bin/tmux normally,
                              # but the harness's tmux is at /usr/bin/tmux (debian default).
# We need tmux MISSING. Easiest: PATH that lacks /usr/bin entirely won't have most tools either.
# Instead, build a sparse PATH that has bash + jq + the claude binary's dir, but NO tmux.

# Make a temp bin dir with only the tools install needs (minus tmux).
TBIN=$(mktemp -d)
for tool in bash jq cat sed grep date stat install ln mv rm mkdir chmod printf ls dirname readlink basename; do
    if real=$(command -v "$tool"); then
        ln -s "$real" "$TBIN/$tool"
    fi
done
# Include claude but explicitly NOT tmux.
ln -s "$(command -v claude)" "$TBIN/claude"

stderr_file=$(mktemp)
set +e
HOME="$TESTHOME" \
    PATH="$TBIN" \
    /opt/skills/install-agent-director/install.sh \
        --binary /usr/local/bin/agent-director --no-symlink \
        >/dev/null 2>"$stderr_file"
ec=$?
set -e

if [ "$ec" != "2" ]; then
    echo "FAIL: exit=$ec; want 2 (pre-flight failure)"; cat "$stderr_file"; exit 1
fi
if ! grep -q 'tmux' "$stderr_file"; then
    echo "FAIL: stderr did not name tmux as missing"; cat "$stderr_file"; exit 1
fi
echo "install-4 ok: missing tmux → exit 2; stderr names tmux"
```
