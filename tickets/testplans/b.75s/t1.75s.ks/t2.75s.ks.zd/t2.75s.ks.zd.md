# pause-3: pause on waiting → row flips to ended → verb returns 0

Per SRD §9 + Epic 5 AC #1: pause on a `waiting` Spawn sends `/exit`,
polls the row's state, and returns 0 when the state becomes `ended`.

In production the SessionEnd hook flips the row to `ended` after
Claude actually exits. In the harness we simulate that hook by
writing the new state directly via `sqlite3` from a background
process that runs ~500ms after `pause` begins.

## Setup

Per-case DB-reset. A `cat` pane under `cd-p3` (so the `/exit` send-keys
calls have somewhere to land).

## Steps

1. Start `tmux new-session -d -s cd-p3 -- cat`.
2. Seed `spawns` row at state `waiting`, session_name `cd-p3`.
3. Schedule a background flip: after 500ms, write
   `state='ended'` for `id-p3`.
4. `agent-director pause --claude-instance-id id-p3` (default
   30s timeout; the flip lands well within it).

## Pass criteria

- pause exits 0.
- Final row state is `ended`.

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-p3 -- cat
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-p3', 'waiting', '/tmp', 'cd-p3', 'off');"
# Background flip: simulate SessionEnd hook arriving 500ms later.
( sleep 0.5; sqlite3 "$HOME/.agent-director/state.db" \
    "UPDATE spawns SET state='ended' WHERE claude_instance_id='id-p3';" ) &
flipper=$!
agent-director pause --claude-instance-id id-p3 >/dev/null
wait "$flipper" || true
final=$(sqlite3 "$HOME/.agent-director/state.db" \
    "SELECT state FROM spawns WHERE claude_instance_id='id-p3';")
if [ "$final" != "ended" ]; then
    echo "FAIL: final state=$final; want ended"; exit 1
fi
echo "pause-3 ok: waiting → /exit → flipped → ended"
```
