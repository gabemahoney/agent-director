# install-5: whitespace in install path is rejected

Per SRD §4.3: tmux's direct-argv invocation does not tolerate spaces
in the binary path; the install script rejects up front rather than
failing mysteriously later.

## Setup

Per-case DB-reset. Construct a $HOME with a space in its path.

## Steps

1. Make a temp dir whose name contains a space.
2. Invoke install.sh with that HOME.
3. Verify exit code is 2 and stderr mentions "whitespace".

## Pass criteria

- Exit code is 2.
- Stderr contains "whitespace".

## Script

```bash
set -euo pipefail
TESTHOME=$(mktemp -d "/tmp/install 5 XXXXXX")  # space in the path

stderr_file=$(mktemp)
set +e
HOME="$TESTHOME" /opt/skills/install-agent-director/install.sh \
    --binary /usr/local/bin/agent-director --no-symlink \
    >/dev/null 2>"$stderr_file"
ec=$?
set -e

if [ "$ec" != "2" ]; then
    echo "FAIL: exit=$ec; want 2 (whitespace rejection)"; cat "$stderr_file"; exit 1
fi
if ! grep -qi 'whitespace' "$stderr_file"; then
    echo "FAIL: stderr did not mention whitespace"; cat "$stderr_file"; exit 1
fi
rm -rf "$TESTHOME"
echo "install-5 ok: whitespace path → exit 2; clear error"
```
