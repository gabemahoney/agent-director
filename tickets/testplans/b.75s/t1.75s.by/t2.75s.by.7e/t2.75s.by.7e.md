# Test case: harness self-check (no spawn required)

Maps to Epic 2 AC #1 (degraded form): driver loads testplan, can see its t2
list, and reports pass without spawning anything.

## Setup

The harness image is built and the container is running with
`tickets/testplans` mounted read-only at `/work/tickets/testplans`. The
DB-reset fixture has already run; `~/.claude-director/state.db` is fresh.

## Steps

1. Confirm `claude --version` reports `2.1.120` (pinned).
2. Confirm the driver artifacts exist at `/opt/driver/run-testplan.sh`,
   `/opt/driver/prompt.md`, and `/opt/driver/db-reset.sh`, all readable.
3. Confirm `claude-director help` exits 0 and emits a JSON object with a
   non-empty `verbs` array.
4. Confirm the harness-smoke t1 collector is visible: find a `t1.*.md` file
   under `/work/tickets/testplans/` whose frontmatter title contains
   `harness-smoke`, and confirm its `children:` list has at least 3 entries.

## Pass criteria

All four steps observable; exit 0.

```bash
set -euo pipefail
claude --version | grep -F '2.1.120' >/dev/null
test -r /opt/driver/run-testplan.sh
test -r /opt/driver/prompt.md
test -r /opt/driver/db-reset.sh
claude-director help | jq -e '.verbs | length > 0' >/dev/null
t1=$(grep -lE '^title:.*harness-smoke' /work/tickets/testplans/*/*/t1.*.md | head -n 1)
test -n "$t1"
children=$(awk '/^---$/{fm=!fm;next}fm&&/^children:/{ic=1;next}fm&&ic&&/^[a-zA-Z_]/{ic=0}fm&&ic&&/^- /{c++}END{print c+0}' "$t1")
test "$children" -ge 3
echo "smoke-1 ok: harness-smoke t1 has ${children} cases"
```
