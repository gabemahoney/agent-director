# ts-bun-4-unsupported-platform-init-error

Pins Epic 5 AC: stubbing the platform tuple to an unsupported value causes
`resolveCliPath()` to throw `ErrUnsupportedPlatform` whose message contains
the tuple string `"linux-mips"`. The error must be catchable before any
subprocess spawn occurs — the guard is pure in-process TypeScript.

`platformResolve.ts::resolveCliPath()` accepts `ResolveCliPathOpts` for DI,
so no monkey-patching of `process.platform` is needed.

## Setup

The script copies `/work/source` to a writable temp dir, runs
`bun install --no-progress`, writes a tiny inline TypeScript probe to the
`pkg/ts-bun-client/` directory, and runs it with `bun run`. The probe
imports `resolveCliPath` and `ErrUnsupportedPlatform` from the local
source. No CLI binary is needed — the error is thrown before any
subprocess spawn.

## Steps

1. Assert `bun` is in PATH; fail clearly if not.
2. Create workspace; `cp -a /work/source/.`.
3. `cd pkg/ts-bun-client && bun install --no-progress`.
4. Write inline TS probe (heredoc) to `pkg/ts-bun-client/tsbun4_probe.ts`:
   - Import `resolveCliPath` from `./src/internal/platformResolve.js`.
   - Import `ErrUnsupportedPlatform` from `./src/errors.js`.
   - Call `resolveCliPath({ platform: "linux", arch: "mips" })`.
   - Assert thrown error `instanceof ErrUnsupportedPlatform`.
   - Assert `e.message` includes `"linux-mips"`.
   - Exit 0 on pass, 1 on any failure.
5. `bun run tsbun4_probe.ts`; assert exit 0.

## Pass criteria

- Inline probe exits 0.
- `ErrUnsupportedPlatform` is thrown (not any other error).
- Error message contains `"linux-mips"` (the platform-arch tuple).
- Script exits 0; final line contains `ts-bun-4-unsupported-platform-init-error ok:`.

## Script

```bash
set -euo pipefail

if ! command -v bun >/dev/null 2>&1; then
    echo "FAIL: bun not found in PATH — check test/Dockerfile BUN_VERSION ARG"
    exit 1
fi

export HOME=/tmp/bun-home-tsbun4
mkdir -p "$HOME"

WORK=$(mktemp -d /tmp/tsbun4_XXXXXX)
cp -a /work/source/. "$WORK/"

cd "$WORK/pkg/ts-bun-client"
bun install --no-progress 2>/dev/null

# Write inline TS probe relative to pkg/ts-bun-client so imports resolve correctly
cat > tsbun4_probe.ts << 'PROBE_EOF'
import { resolveCliPath } from "./src/internal/platformResolve.js";
import { ErrUnsupportedPlatform } from "./src/errors.js";

let caught = false;
try {
    resolveCliPath({ platform: "linux", arch: "mips" });
    console.error("FAIL: expected ErrUnsupportedPlatform to be thrown, but no error was raised");
    process.exit(1);
} catch (e) {
    if (!(e instanceof ErrUnsupportedPlatform)) {
        const name = (e instanceof Error) ? e.constructor.name : typeof e;
        console.error(`FAIL: expected ErrUnsupportedPlatform, got ${name}: ${e}`);
        process.exit(1);
    }
    if (!e.message.includes("linux-mips")) {
        console.error(`FAIL: ErrUnsupportedPlatform message does not include "linux-mips": ${e.message}`);
        process.exit(1);
    }
    caught = true;
}

if (!caught) {
    console.error("FAIL: no error was caught");
    process.exit(1);
}

console.log("ts-bun-4-unsupported-platform-init-error ok: ErrUnsupportedPlatform thrown with linux-mips in message");
PROBE_EOF

OUT=$(mktemp /tmp/tsbun4_out.XXXXXX)
EXIT=0
bun run tsbun4_probe.ts > "$OUT" 2>&1 || EXIT=$?

# Clean up probe file
rm -f tsbun4_probe.ts

if [[ "$EXIT" -ne 0 ]]; then
    echo "FAIL: bun run tsbun4_probe.ts exited with code $EXIT"
    cat "$OUT" >&2
    exit 1
fi

tail -1 "$OUT"
```
