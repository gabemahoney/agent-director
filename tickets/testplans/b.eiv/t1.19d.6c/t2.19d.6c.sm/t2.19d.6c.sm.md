# subprocess-client-1-smoke

Per SRD §SR-10.3: every callable verb must succeed through the
installed-tarball Client, returning a typed result envelope; at least
one error path per verb must surface the correct `Err*` subclass via
`instanceof`. Mirrors the in-repo `subprocess-smoke.test.ts` but runs
against an installed-package layout inside the container.

## Setup

- `${HOME}=$(mktemp -d)`.
- `bun install file:///work/source/dist/agent-director-*.tgz` into
  `${HOME}/proj`.
- The bundled CLI binary is resolved through
  `${HOME}/proj/node_modules/@agent-director/linux-x64/bin/agent-director`.
- `fake-tmux` is on `${PATH}` (the harness image stages it at a known
  path).

## Steps

1. Write a tiny driver script `${HOME}/proj/driver.mjs` that imports
   `agent-director` and exercises every verb (happy path + one error
   path each) exactly as `pkg/ts-bun-client/test/subprocess-smoke.test.ts`
   does.
2. Run `bun run driver.mjs`. Capture stdout/exit code.
3. The script asserts each verb's typed result; throws on any failure.

## Pass criteria

- Driver exits 0.
- Every verb's happy-path case prints `OK <verb>`.
- Every non-allow-listed verb's error-path case prints
  `OK <verb>: <ErrClassName>`.
- No verb is missing from the output (count check on stdout lines).

## Script

```bash
set -euo pipefail
HOME_DIR="$HOME"
PROJ="$HOME_DIR/proj"
rm -rf "$PROJ"
mkdir -p "$PROJ"
cd "$PROJ"

# Build (and cache) the agent-director tarball from /work/source. The shared
# helper stages the package into /tmp so the prepack hook can write dist/.
. /work/source/tickets/testplans/b.eiv/t1.19d.6c/build-tarball.sh

# Minimal consumer package.json. Avoid `bun init -y` so we don't inherit any
# default key (notably "name") that could collide with the agent-director
# dependency entry that `bun install` adds next.
cat > "$PROJ/package.json" <<'EOF'
{
  "name": "subprocess-smoke-consumer",
  "version": "0.0.0",
  "type": "module"
}
EOF

bun install "file://${TARBALL}" >/dev/null
stage_platform_pkg "$PROJ"

cp /work/source/tickets/testplans/b.eiv/t1.19d.6c/t2.19d.6c.sm/driver.mjs "$PROJ/driver.mjs"

# Driver writes one "OK <verb>" line per scenario it passes.
out=$(bun run "$PROJ/driver.mjs" 2>&1)
echo "$out"

# Sanity: every kebab-case verb must appear at least once in stdout.
for v in spawn status get send-keys read-pane kill decide resume find-missing expire delete make-template list pause version; do
    if ! grep -q "OK ${v}" <<< "$out"; then
        echo "FAIL: verb '${v}' missing from driver output"; exit 1
    fi
done
echo "subprocess-client-1-smoke ok"
```

## Driver

The harness mounts `/work/source/tickets/testplans/b.eiv/t1.19d.6c/t2.19d.6c.sm/driver.mjs` from the repo (sibling of this file). The driver
is a small ES-module shim around the installed `agent-director` package
that mirrors the assertions in
`pkg/ts-bun-client/test/subprocess-smoke.test.ts`.
