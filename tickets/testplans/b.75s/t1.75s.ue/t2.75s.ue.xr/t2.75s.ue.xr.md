# resume-4: resume on row with empty claude_session_id returns ErrNoSessionId

A Spawn killed before its first SessionStart hook fired has no
rotated claude_session_id. Without that id there's no JSONL to
point `claude --resume` at.

## Setup

Per-case DB-reset. Seed an `ended` row directly via sqlite3 with
`claude_session_id` left NULL.

## Steps

1. INSERT a row with state=ended, claude_session_id=NULL.
2. `resume` → non-zero; `err_name=ErrNoSessionId`.

## Pass criteria

- resume exits non-zero with the typed error.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode, ended_at)
   VALUES ('id-no-sess', 'ended', '/tmp', 'cd-no-sess', 'off', CURRENT_TIMESTAMP);"

stderr_file=$(mktemp)
if claude-director resume --claude-instance-id id-no-sess 2>"$stderr_file" >/dev/null; then
    echo "FAIL: resume with empty session id exited 0"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrNoSessionId" ]; then
    echo "FAIL: err_name=$err_name; want ErrNoSessionId"; exit 1
fi
echo "resume-4 ok: empty session id → ErrNoSessionId"
```
