# Epic 5 — `epic-05-stop` testplan

t1 collector for Epic 5 (`t1.8op.5f` — Stop live Spawns: kill + pause).
Each t2 is a plain-English case the Docker harness executes. The
Epic 5 Progression Contract requires every t2 to pass under
`make test-docker EPIC=epic-05-stop` per the orchestrator's first-hand
audit.

## Sourced from

- Epic ticket: `t1.8op.5f` (Acceptance Criteria #1–#3).
- SRD §5 (kill semantics, find-missing relationship), §9 (pause
  one-shot, state dispatch, timeout), §12 (verb shapes), §13.1 (error
  catalogue).

## Cases

| case | what it pins |
|---|---|
| `kill-1-live-row` | seed `waiting` row + real tmux session → `kill` → tmux session gone; verb exits 0. |
| `kill-2-ended-noop` | seed `ended` row → `kill` → verb exits 0 with no tmux call (row never had a live session to kill). |
| `kill-3-missing-id` | call `kill --claude-instance-id absent` → ErrSpawnNotFound on stderr; non-zero exit. |
| `kill-4-idempotent` | seed `waiting` + real tmux session → first `kill` removes session; second `kill` against the same id still returns 0 (tmux call inside the verb is now a swallowed no-op). |
| `pause-1-ended-noop` | seed `ended` row → `pause` → verb exits 0; no `/exit` sent. |
| `pause-2-working-rejected` | seed `working` row → `pause` → ErrSpawnNotPausable on stderr; no tmux call. |
| `pause-3-waiting-transitions-to-ended` | seed `waiting` row + cat-pane (a stand-in for a live Claude); pause runs in the background; before the deadline a separate process flips the DB row to `ended` (simulating the SessionEnd hook); pause returns 0. |
| `pause-4-timeout` | seed `waiting` row + cat-pane; `pause.timeout_seconds=2`; row stays `waiting` (no SessionEnd injection) → ErrPauseTimeout on stderr; non-zero exit; elapsed ≥ 2s. |

## Driver mode

All cases are written for `DRIVER_MODE=shell` — the orchestrator runs
the harness with `make test-docker EPIC=epic-05-stop`. Each t2 body
has a single fenced ` ```bash ` block the driver executes after the
per-case DB-reset fixture.

The kill cases that touch a real tmux session create a `cat` pane
under the harness's reserved `cd-*` prefix; the DB-reset fixture
cleans these between cases.

The pause cases exercise the `/exit` → poll-for-ended path against a
synthesized row rather than a real Claude. `pause-3` injects the
post-`/exit` transition by writing directly to the SQLite row, which
mirrors what the real SessionEnd hook would do once Claude actually
exits. `pause-4` proves the deadline math: with
`pause.timeout_seconds=2` configured via a per-test
`~/.agent-director/config.toml`, the verb waits ~2s and returns
`ErrPauseTimeout`.

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-05-stop`
  yourself, inspect the JSON stream case-by-case, confirm each `t2`
  emitted `{"status":"pass"}`.
- `pause-4` uses a 2s timeout deliberately (not the 30s default) so
  the harness runs in well under a minute total.
- All cases avoid `tmux kill-server` — the operator denies that tool.
  Per-case cleanup is via `tmux kill-session -t cd-<slug>` only.

## tmux session sweep

Cases that exercise real tmux leave sessions under the `cd-*` prefix.
The DB-reset fixture kills `cd-*` sessions between cases.
