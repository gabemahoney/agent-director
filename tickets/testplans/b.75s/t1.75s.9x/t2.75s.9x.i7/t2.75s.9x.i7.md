# interact-3: --no-enter leaves text uncommitted

Maps to Epic 4's `press_enter=false` path (CLI flag `--no-enter`).
When the caller passes `--no-enter`, send-keys must emit exactly one
tmux send-keys call (the text) and zero Enter calls.

## Setup

Per-case DB-reset. A `cat` pane.

## Steps

1. Start `tmux new-session -d -s cd-i3 -- cat`.
2. Seed a `spawns` row at state `waiting`.
3. `claude-director send-keys --claude-instance-id id-i3 --text "draft" --no-enter`.
4. Confirm `read-pane` does NOT contain `draft` echoed back (cat only
   echoes lines on LF / CR; absent submission, the bytes are buffered
   inside cat and not yet visible in the pane).
5. Then send `--text "" ` (empty) WITHOUT `--no-enter` → cat now echoes.

## Pass criteria

- Step 3 exit 0.
- After step 3 the pane does NOT yet contain `draft` as an echoed line.
- After step 5, the pane echoes `draft` (the Enter submitted the
  buffered text).

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-i3 -- cat
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i3', 'waiting', '/tmp', 'cd-i3', 'off');"
claude-director send-keys --claude-instance-id id-i3 --text 'draft' --no-enter >/dev/null
sleep 0.2
pane_before="$(claude-director read-pane --claude-instance-id id-i3 | jq -r '.pane')"
# cat has not yet echoed `draft` because there has been no Enter.
if echo "$pane_before" | grep -qE '^draft$'; then
    echo "FAIL: pane shows 'draft' line before Enter was sent"; exit 1
fi
# Now submit (empty text + default press_enter=true).
claude-director send-keys --claude-instance-id id-i3 --text '' >/dev/null
sleep 0.2
pane_after="$(claude-director read-pane --claude-instance-id id-i3 | jq -r '.pane')"
echo "$pane_after" | grep -qE '^draft$'
echo "interact-3 ok: --no-enter buffered; subsequent Enter committed"
```
