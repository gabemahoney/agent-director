# go-client-1-external-consumer-build

Pins Epic 4 AC: an external Go consumer can compile against `pkg/api`
without referencing any `internal/*` package itself. The
`tools/consumer-dryrun/` module is the canonical external-consumer probe
— it has its own `go.mod` with a `replace` directive pointing at the
working tree and exercises every public `Client` method plus the
standalone functions. Successful compilation IS the test; we additionally
assert the consumer's own import graph contains zero `internal/*`
references.

## Setup

No DB state needed. The consumer module lives at
`tools/consumer-dryrun/` in the source tree and uses `replace
github.com/gabemahoney/agent-director => ../../` to bind against the
working copy. Because `/work/source` is read-only, the script copies the
repository to a writable temp directory and runs `go build` from
`tools/consumer-dryrun/` there.

## Steps

1. Assert `go` is available in PATH; fail with a clear message if not.
2. Create a writable workspace under `/tmp/` and `cp -a /work/source/.` into it.
3. `cd <workspace>/tools/consumer-dryrun && go build ./...`; assert exit 0.
4. Run `go list -deps -f '{{.ImportPath}}' ./...` in the consumer module
   and confirm every import path is either the consumer module itself,
   `github.com/gabemahoney/agent-director/pkg/api`, or a third-party / stdlib
   path. Assert no path under
   `github.com/gabemahoney/agent-director/internal/` appears — the
   consumer's own imports must stay out of `internal/*`.
5. Confirm the built binary exists at `<workspace>/tools/consumer-dryrun/consumer-dryrun`
   (or whatever Go's default name resolves to from the module path).

## Pass criteria

- `go build ./...` in `tools/consumer-dryrun/` exits 0.
- `go list -deps` reports zero imports from
  `github.com/gabemahoney/agent-director/internal/` in the consumer
  module's own direct imports.
- Script exits 0; final line matches `go-client-1-external-consumer-build ok: ...`.

## Script

```bash
set -euo pipefail

if ! command -v go >/dev/null 2>&1; then
    echo "FAIL: go not found in PATH"
    exit 1
fi

WORK=$(mktemp -d /tmp/goclient1_XXXXXX)
cp -a /work/source/. "$WORK/"

CONSUMER="$WORK/tools/consumer-dryrun"
if [[ ! -d "$CONSUMER" ]]; then
    echo "FAIL: tools/consumer-dryrun/ missing from source tree"
    exit 1
fi

cd "$CONSUMER"

# Compilation is the primary signal — replace directive in go.mod points
# at ../../ so this builds against the copied working tree.
if ! go build ./... 2> /tmp/goclient1_build.err; then
    echo "FAIL: go build ./... failed inside tools/consumer-dryrun/"
    cat /tmp/goclient1_build.err >&2
    exit 1
fi

# Inspect the consumer's own import graph. The consumer must NOT reference
# any internal/* package directly — Go's visibility rules would prevent
# this anyway for an external module, but we assert it explicitly so a
# regression (e.g. someone making consumer-dryrun a same-module test) is
# caught loudly.
IMPORTS=$(go list -f '{{range .Imports}}{{.}}{{"\n"}}{{end}}' ./...)
if printf '%s\n' "$IMPORTS" | grep -E '^github\.com/gabemahoney/agent-director/internal/' >/dev/null; then
    echo "FAIL: consumer-dryrun imports an internal/ package directly:"
    printf '%s\n' "$IMPORTS" | grep -E '^github\.com/gabemahoney/agent-director/internal/' >&2
    exit 1
fi

# Sanity: confirm the consumer actually pulls in pkg/api (otherwise the
# above check is trivially satisfied — e.g. by an empty main.go).
if ! printf '%s\n' "$IMPORTS" | grep -qx 'github.com/gabemahoney/agent-director/pkg/api'; then
    echo "FAIL: consumer-dryrun does not import github.com/gabemahoney/agent-director/pkg/api"
    echo "imports were:"
    printf '%s\n' "$IMPORTS" >&2
    exit 1
fi

echo "go-client-1-external-consumer-build ok: consumer-dryrun builds against pkg/api with zero internal/* direct imports"
```
