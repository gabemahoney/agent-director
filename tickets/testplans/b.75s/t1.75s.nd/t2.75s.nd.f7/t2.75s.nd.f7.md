# mcp-4: tools/call status routes through to internal/api.Status

Seed a row via sqlite3; tools/call name=status → result.content[0].text
is a JSON-encoded StatusResult with the right state.

## Setup

Per-case DB-reset.

## Steps

1. Seed a row in state `waiting`.
2. Pipe initialize + tools/call status into the server.
3. Parse the second response; extract content[0].text and JSON-parse
   it; assert state="waiting".

## Pass criteria

- tools/call returns a non-error response.
- `content[0].type = "text"`.
- The text payload is the JSON `{"state":"waiting"}` (the verb's
  StatusResult shape).

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-mcp4', 'waiting', '/tmp', 'cd-mcp4', 'off');"

out=$(printf '%s
%s
' \
    '{"jsonrpc":"2.0","id":1,"method":"initialize"}' \
    '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"status","arguments":{"claude_instance_id":"id-mcp4"}}}' \
    | agent-director serve --stdio 2>/dev/null)

second=$(printf '%s' "$out" | sed -n '2p')
content_type=$(printf '%s' "$second" | jq -r '.result.content[0].type')
text_payload=$(printf '%s' "$second" | jq -r '.result.content[0].text')
state=$(printf '%s' "$text_payload" | jq -r '.state')

if [ "$content_type" != "text" ] || [ "$state" != "waiting" ]; then
    echo "FAIL: content_type=$content_type state=$state (expected text/waiting)"
    printf '%s' "$second"; exit 1
fi
echo "mcp-4 ok: tools/call status routes through to api.Status"
```
