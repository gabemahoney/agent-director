# TS/Bun client library (`ts-bun-lib`) testplan

t1 collector for the TypeScript/Bun client library (`pkg/ts-bun-client` — Epic 5
of `b.qe2`). Each t2 is a plain-shell case the Docker harness executes. Runs under
`make test-docker EPIC=ts-bun-lib`; the driver resolves the slug by
case-insensitive substring match against the t1 frontmatter title (see
`test/driver/run-testplan.sh`), so the canonical substring `ts-bun-lib` must
remain in the title verbatim.

## Sourced from

- Source Epic: `t1.qe2.ew` (TS/Bun client library, all verbs + publish pipeline).
- Testplan-adding Task: `t2.qe2.ew.ot`.

## Driver mode

All cases use `DRIVER_MODE=shell`. Each t2 body has a single fenced `bash`
block executed by the driver after the per-case DB-reset fixture.

All cases copy `/work/source` to a writable temp directory under `/tmp/`
and operate from there. `/work/source` itself is never written.

## Notes for the orchestrator

- Audit: `make test-docker EPIC=ts-bun-lib`; inspect the JSON stream case-by-case.
- The Dockerfile must have Bun installed (added to `test/Dockerfile` as part of
  this testplan — `ARG BUN_VERSION=1.3.13`, installed to `/opt/bun`, symlinked
  to `/usr/local/bin/bun`).
- None of these cases use `agent-director spawn` or the harness `agent-director`
  binary at `/usr/local/bin/agent-director`. The DB-reset fixture's cd-session
  sweep is a no-op for all five cases.

## tmux session sweep

No `cd-*` sessions are created. The db-reset fixture's session sweep is a
no-op for these cases.
