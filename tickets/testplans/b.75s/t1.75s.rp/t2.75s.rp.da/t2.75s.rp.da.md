# release-pipeline-3-darwin-arm64-offline-halt

Pins Epic 7 AC: the release halts cleanly with a specific
`darwin/arm64 leg unavailable; bring self-hosted runner online
before retrying` message when the self-hosted runner is offline.

The case exercises the `AD_RELEASE_SIMULATE_DARWIN_ARM64_OFFLINE=1`
escape hatch so the test does not need a real cabi-matrix run.
The other leg is also stubbed (zero-`gh` environment) so the
test cannot mask a hidden dependency on a real workflow.

## Setup

Writable copy of `/work/source`. The case stubs `gh` on PATH with
a script that returns a known-good single-match
`databaseId`/`success` response so `collect_cabi_artifacts` enters
the per-platform download loop where the offline-simulation flag
triggers the halt.

## Pass criteria

- Exit code 3 (build-phase failure).
- Output contains `[build] darwin/arm64 leg unavailable; bring
  self-hosted runner online before retrying`.
- Output contains `[build] halting before tag/publish/gh-release`.
- Output does NOT contain any `[tag]` or `[publish]` line (the
  halt was before those phases).
- Final line matches `release-pipeline-3 ok: ...`.

## Script

```bash
set -euo pipefail

SRC=/work/source
WORK=$(mktemp -d)
OUT=$(mktemp)
STUB_DIR=$(mktemp -d)
trap 'rm -rf "$WORK" "$OUT" "$STUB_DIR"' EXIT

cp -a "$SRC/." "$WORK/"
cd "$WORK"
rm -rf .git
git init -q .
git config user.email "test@example.com"
git config user.name  "Test"
git checkout -q -b main
git add -A
git commit -q -m "release-pipeline test baseline"
git tag v0.0.1 HEAD

# Stub `gh` so `collect_cabi_artifacts` finds a single green run.
# The download stub provides only the linux-amd64 artifact;
# darwin-arm64's slot is taken by the
# AD_RELEASE_SIMULATE_DARWIN_ARM64_OFFLINE=1 short-circuit before
# gh is consulted. (darwin/amd64 was dropped from v1 2026-05-24.)
cat > "$STUB_DIR/gh" <<'GHEOF'
#!/usr/bin/env bash
case "$1" in
    run)
        case "$2" in
            list)
                printf '%s' '[{"databaseId":42,"headSha":"abc","conclusion":"success","status":"completed"}]'
                exit 0
                ;;
            download)
                while [[ $# -gt 0 ]]; do
                    case "$1" in
                        --name) art="$2"; shift 2 ;;
                        --dir)  dir="$2"; shift 2 ;;
                        *) shift ;;
                    esac
                done
                mkdir -p "$dir"
                case "$art" in
                    pkg-cabi-linux-amd64)  touch "$dir/libagent_director.so"  "$dir/libagent_director.h" ;;
                esac
                exit 0
                ;;
        esac
        ;;
    repo) echo '{"nameWithOwner":"gabemahoney/agent-director"}'; exit 0 ;;
esac
GHEOF
chmod +x "$STUB_DIR/gh"

set +e
PATH="$STUB_DIR:$PATH" \
    AD_RELEASE_SIMULATE_DARWIN_ARM64_OFFLINE=1 \
    VERSION=v0.99.0 \
    bash skills/release-agent-director/release.sh \
    --dry-run --branch main --no-build > "$OUT" 2>&1
RC=$?
set -e

if [[ "$RC" -ne 3 ]]; then
    echo "FAIL: expected exit code 3 (build-phase halt), got $RC"
    sed 's/^/    /' "$OUT"
    exit 1
fi

if ! grep -q '\[build\] darwin/arm64 leg unavailable; bring self-hosted runner online before retrying' "$OUT"; then
    echo "FAIL: missing darwin/arm64 self-hosted-runner message"
    sed 's/^/    /' "$OUT"
    exit 1
fi

if ! grep -q '\[build\] halting before tag/publish/gh-release' "$OUT"; then
    echo "FAIL: missing 'halting before tag/publish/gh-release' marker"
    sed 's/^/    /' "$OUT"
    exit 1
fi

if grep -qE '^\[(tag|publish|gh-release)\]' "$OUT"; then
    echo "FAIL: phases past build ran despite darwin/arm64 offline"
    sed 's/^/    /' "$OUT"
    exit 1
fi

echo "release-pipeline-3 ok: build halted on darwin/arm64 offline"
```
