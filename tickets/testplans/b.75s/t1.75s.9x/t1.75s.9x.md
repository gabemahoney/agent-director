# Epic 4 — `epic-04-interact` testplan

t1 collector for Epic 4 (`t1.8op.gm` — External interaction: send-keys
+ read-pane). Each t2 is a plain-English case the Docker harness
executes. The Epic 4 Progression Contract requires every t2 to pass
under `make test-docker EPIC=epic-04-interact` per the orchestrator's
first-hand audit.

## Sourced from

- Epic ticket: `t1.8op.gm` (Acceptance Criteria #1–#3).
- SRD §4.3 (send-keys multiline + `\r` strip), §12 (verb shapes).
- Reference notes: `send-keys-research.md`, `pane-output-research.md`.

## Driver mode

All cases are written for `DRIVER_MODE=shell` — the orchestrator runs
the harness with `make test-docker EPIC=epic-04-interact`. Each t2 body
has a single fenced ` ```bash ` block the driver executes after the
per-case DB-reset fixture.

The harness uses the real `agent-director` binary (built by
`test-image`) plus the real `tmux` inside the Docker image. Sessions
are tagged `cd-*` so the per-case DB-reset's session sweep cleans them
between cases.

Cases that exercise CR-strip / multiline argv inject text via the CLI
`--text` flag with literal `$'...'` (ANSI-C) quoting; bash passes the
resulting bytes through to agent-director, which strips CR and
preserves LF before invoking tmux send-keys. The test then captures
the real tmux pane via `read-pane` (or via fake-tmux argv inspection
in the cases that prove the argv composition rather than the rendered
result).

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-04-interact`
  yourself, inspect the JSON stream case-by-case, confirm each `t2`
  emitted `{"status":"pass"}`.
- The state-guard cases (`interact-7`, `interact-8`) synthesize the
  precondition via direct `sqlite3` row inserts after `db-reset` runs.
  No `claude` invocation is required for any case here.
- The `ErrSendKeysWhileRelayed` stub guard is active in Epic 4 even
  though the full relay path lands in Epic 10. The case here proves
  the precondition surface is correct from day one.

## tmux session sweep

Cases that exercise real tmux leave sessions under the `cd-*` prefix.
The DB-reset fixture kills `cd-*` sessions between cases.
