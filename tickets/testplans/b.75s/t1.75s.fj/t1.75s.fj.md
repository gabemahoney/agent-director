# Epic 6 — `epic-06-discover` testplan

t1 collector for Epic 6 (`t1.8op.ok` — Discover Spawns: labels,
parent_id, list). Each t2 is a plain-English case the Docker harness
executes. The Epic 6 Progression Contract requires every t2 to pass
under `make test-docker EPIC=epic-06-discover` per the orchestrator's
first-hand audit.

## Sourced from

- Epic ticket: `t1.8op.ok` (Acceptance Criteria #1–#3).
- SRD §4.2 (labels JSON column, linear-scan filter), §7.5 (parent_id
  env-read), §11 (DB canonical for labels), §12 (list verb shape).

## Cases

| case | what it pins |
|---|---|
| `discover-1-no-filters` | seed 3 rows → `list` with no filters returns all 3. |
| `discover-2-state-filter` | seed mixed-state rows → `list --state waiting,working` returns the 2 in either state, not the ended one. |
| `discover-3-single-label` | seed 3 rows with different `project` labels → `list --label project=foo` returns only the matches. |
| `discover-4-multi-label-and` | seed 2 rows; only one has both `project=foo` AND `env=dev` → `list --label project=foo --label env=dev` returns that one row only. |
| `discover-5-parent-auto-detect` | real `spawn` parent A; `AGENT_DIRECTOR_INSTANCE_ID=<A> spawn` child B → `get` B shows `parent_id=<A>`; `list --parent <A>` returns only B. |
| `discover-6-cwd-filter` | seed rows with distinct cwds → `list --cwd /tmp` returns only matching. |
| `discover-7-limit-cap` | seed 3 rows → `list --limit 1` returns exactly 1 row. |
| `discover-8-label-key-normalization` | real `spawn --label my-key=val`; `tmux show-environment -t <sess>` carries `AGENT_DIRECTOR_LABEL_MY_KEY=val`; the row's labels JSON keeps the original `my-key`. |

## Driver mode

All cases are written for `DRIVER_MODE=shell` — the orchestrator runs
the harness with `make test-docker EPIC=epic-06-discover`. Each t2
body has a single fenced ` ```bash ` block the driver executes after
the per-case DB-reset fixture.

Pure list-filter cases (1, 2, 3, 4, 6, 7) seed rows directly via
`sqlite3` rather than going through the full `spawn` pipeline. This
keeps the harness API-free and the case bodies short. Cases that
require the spawn pipeline (5 — parent_id env-read; 8 — env-var
emission) call the real `agent-director spawn`; tmux launches a
`claude` process inside the session but no Claude conversation is
driven.

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-06-discover`
  yourself, inspect the JSON stream case-by-case, confirm each `t2`
  emitted `{"status":"pass"}`.
- The seeded-row cases do not launch any tmux sessions, so they leave
  no `cd-*` sessions to sweep. Cases 5 and 8 do launch real tmux
  sessions; the DB-reset fixture's `cd-*` sweep cleans them between
  cases.

## tmux session sweep

Cases that exercise real tmux (5, 8) leave sessions under the `cd-*`
prefix. The DB-reset fixture kills `cd-*` sessions between cases.
