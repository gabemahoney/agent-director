# release-2-prepublish-rejects-version-skew

Pins SR-4.1 lockstep guard: the umbrella's `prepublishOnly` script
(`scripts/prepublish-guards.ts`) refuses to publish when
`package.json` `version` does not equal `SKILL.md` frontmatter
`version:`. Stage a release-stamped copy with `package.json` bumped
to `0.4.2` (and `optionalDependencies` updated to `^0.4.2` so the
other invariants stay satisfied), but leave SKILL.md frontmatter at
`0.4.1`. Invoke the guard script directly; assert non-zero exit and
the version-skew message names both versions.

## Pass criteria

- `bun run scripts/prepublish-guards.ts` exits non-zero (specifically
  `1` per the script; the case asserts non-zero).
- Stderr contains `version skew` substring.
- Stderr contains `0.4.2` (the package.json version).
- Stderr contains `0.4.1` (the SKILL.md frontmatter version).

## Script

```bash
set -euo pipefail

STAGE=$(mktemp -d /tmp/rv-r2-stage.XXXXXX)

# Stage the umbrella + skill source.
mkdir -p "$STAGE/pkg/ts-bun-client" "$STAGE/skills"
cp -a /work/source/pkg/ts-bun-client/. "$STAGE/pkg/ts-bun-client/"
cp -a /work/source/skills/install-agent-director "$STAGE/skills/"
rm -rf "$STAGE/pkg/ts-bun-client/node_modules"
rm -rf "$STAGE/pkg/ts-bun-client/skills"

# Baseline: stamp BOTH to 0.4.1 (lockstep), then mutate ONE to introduce skew.
jq '.version = "0.4.1"
    | .optionalDependencies["@agent-director/linux-x64"]   = "^0.4.1"
    | .optionalDependencies["@agent-director/darwin-arm64"] = "^0.4.1"' \
    "$STAGE/pkg/ts-bun-client/package.json" > "$STAGE/pkg/ts-bun-client/package.json.tmp"
mv "$STAGE/pkg/ts-bun-client/package.json.tmp" "$STAGE/pkg/ts-bun-client/package.json"
sed -i 's/^version: .*/version: 0.4.1/' "$STAGE/skills/install-agent-director/SKILL.md"

# Now bump only package.json + opt-deps to 0.4.2 — SKILL.md remains at 0.4.1.
jq '.version = "0.4.2"
    | .optionalDependencies["@agent-director/linux-x64"]   = "^0.4.2"
    | .optionalDependencies["@agent-director/darwin-arm64"] = "^0.4.2"' \
    "$STAGE/pkg/ts-bun-client/package.json" > "$STAGE/pkg/ts-bun-client/package.json.tmp"
mv "$STAGE/pkg/ts-bun-client/package.json.tmp" "$STAGE/pkg/ts-bun-client/package.json"

LOG=$(mktemp /tmp/rv-r2-log.XXXXXX)
EXIT=0
(cd "$STAGE/pkg/ts-bun-client" && bun run scripts/prepublish-guards.ts) > "$LOG" 2>&1 || EXIT=$?

if [ "$EXIT" -eq 0 ]; then
    echo "FAIL: prepublish-guards exited 0 — version skew was not detected"
    sed 's/^/  /' "$LOG"
    exit 1
fi
for needle in "version skew" "0.4.2" "0.4.1"; do
    if ! grep -qF "$needle" "$LOG"; then
        echo "FAIL: SR-4.1 skew message missing substring: $needle"
        sed 's/^/  /' "$LOG"
        exit 1
    fi
done

rm -rf "$STAGE" "$LOG"
echo "release-2-prepublish-rejects-version-skew ok: prepublish-guards exit=$EXIT with version-skew message naming 0.4.2 and 0.4.1"
```
