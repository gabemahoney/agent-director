# discover-8: label key with non-alphanumeric chars is normalized in the env

Per SRD §7.2 step 5: label keys are uppercased + non-alphanumerics
replaced by `_` when emitted as env vars. The DB column keeps the
original verbatim. The case spawns with `--label my-key=val` and
verifies both:

- The tmux session env carries `CLAUDE_DIRECTOR_LABEL_MY_KEY=val`.
- The DB row's `labels` column keeps `"my-key":"val"` verbatim.

Real `spawn` is used because the env emission step is the behavior
being pinned.

## Setup

Per-case DB-reset.

## Steps

1. `claude-director spawn --cwd /tmp --label my-key=val` →
   capture id and tmux_session_name.
2. `tmux show-environment -t <session> CLAUDE_DIRECTOR_LABEL_MY_KEY`
   → expect `CLAUDE_DIRECTOR_LABEL_MY_KEY=val`.
3. Query the row's labels column directly → expect `my-key` key
   preserved verbatim.

## Pass criteria

- spawn exits 0 with a non-empty id.
- tmux show-environment returns the normalized env var.
- Direct sqlite3 query shows `my-key` (not `MY_KEY`) in the labels
  JSON.

## Script

```bash
set -euo pipefail
spawn_out=$(claude-director spawn --cwd /tmp --label my-key=val)
id=$(printf '%s' "$spawn_out" | jq -r '.claude_instance_id')
test -n "$id"
sess=$(claude-director get --claude-instance-id "$id" | jq -r '.tmux_session_name')
test -n "$sess"
env_line=$(tmux show-environment -t "$sess" CLAUDE_DIRECTOR_LABEL_MY_KEY)
if [ "$env_line" != "CLAUDE_DIRECTOR_LABEL_MY_KEY=val" ]; then
    echo "FAIL: tmux env line='$env_line'; want CLAUDE_DIRECTOR_LABEL_MY_KEY=val"; exit 1
fi
labels=$(sqlite3 "$HOME/.claude-director/state.db" \
    "SELECT labels FROM spawns WHERE claude_instance_id='$id';")
case "$labels" in
    *'"my-key":"val"'*) ;;
    *) echo "FAIL: labels column='$labels'; does not contain 'my-key':'val' verbatim"; exit 1 ;;
esac
echo "discover-8 ok: env normalized to MY_KEY; DB column keeps my-key verbatim"
```
