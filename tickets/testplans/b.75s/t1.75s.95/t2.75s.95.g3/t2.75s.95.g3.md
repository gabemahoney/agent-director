# Test case: static binary builds with CGO disabled

Maps to Epic 1 AC #1.

## Setup

The repo is checked out at the current directory inside the container.

## Steps

1. Run `make build` from the repo root.
2. Observe the exit code.
3. Stat `./bin/claude-director` for existence and that it is a regular file.
4. On Linux, run `file ./bin/claude-director` and observe whether the output
   indicates a statically-linked ELF (e.g. contains "statically linked" or
   "not a dynamic executable" when run through `ldd`).

## Pass criteria

- `make build` exits 0.
- `./bin/claude-director` exists as a regular file with executable bits set.
- The binary has no dynamic dependencies beyond vDSO/kernel
  (`ldd ./bin/claude-director` prints "not a dynamic executable").

## Notes

The pure-Go SQLite driver (`modernc.org/sqlite`) is what makes this possible
under `CGO_ENABLED=0`. A regression to a cgo-based driver fails this case.
