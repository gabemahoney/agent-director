# install-3: install preserves pre-existing user hooks

The install script's hook injection is strictly additive — user
hooks that pre-exist in `SessionStart` or `SessionEnd` must survive
the install verbatim.

## Setup

Per-case DB-reset. A pre-seeded `~/.claude/settings.json` with a
user-defined SessionStart hook running `/usr/bin/echo user-hook`.

## Steps

1. Pre-write a settings.json with a custom user hook.
2. Run install.sh.
3. Verify the user's hook is still present.
4. Verify agent-director's hook is ALSO present.

## Pass criteria

- User hook (`/usr/bin/echo user-hook`) survives in SessionStart.
- agent-director's hook is also in SessionStart.
- SessionStart has at least 2 entries.

## Script

```bash
set -euo pipefail
TESTHOME=$(mktemp -d)
mkdir -p "$TESTHOME/.claude"
cat > "$TESTHOME/.claude/settings.json" <<'JSON'
{
  "hooks": {
    "SessionStart": [
      {"hooks": [{"type":"command","command":"/usr/bin/echo user-hook"}]}
    ]
  }
}
JSON

HOME="$TESTHOME" /opt/skills/install-agent-director/install.sh \
    --binary /usr/local/bin/agent-director --no-symlink >/dev/null

count=$(jq -r '.hooks.SessionStart | length' "$TESTHOME/.claude/settings.json")
if [ "$count" -lt 2 ]; then
    echo "FAIL: SessionStart count=$count; want >=2"
    cat "$TESTHOME/.claude/settings.json"; exit 1
fi

user_present=$(jq -r '
    any(.hooks.SessionStart[]?; .hooks[]?.command == "/usr/bin/echo user-hook")
' "$TESTHOME/.claude/settings.json")
cd_present=$(jq --arg p "$TESTHOME/.agent-director/bin/agent-director" -r '
    any(.hooks.SessionStart[]?; (.hooks[]?.command // "") | startswith($p))
' "$TESTHOME/.claude/settings.json")

if [ "$user_present" != "true" ] || [ "$cd_present" != "true" ]; then
    echo "FAIL: user=$user_present cd=$cd_present (both want true)"; exit 1
fi
echo "install-3 ok: user hook + agent-director hook both present"
```
