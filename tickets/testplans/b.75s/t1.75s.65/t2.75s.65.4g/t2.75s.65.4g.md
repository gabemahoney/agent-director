# relay-7: send-keys is rejected while relay-on + check_permission

Per SRD §12 + Epic 10 t2.3e: when a Spawn is sitting on a relayed
permission prompt (`relay_mode=on` AND `state=check_permission`),
the permission relay owns the modal answer. A pane-side send-keys
would race the decide() write, so the verb refuses with
ErrSendKeysWhileRelayed.

Epic 4 stubbed the guard; Epic 10 activates the path. The guard
itself was already in place — this case is the integration
regression check.

## Setup

Per-case DB-reset.

## Steps

1. Seed row state=check_permission, relay_mode=on, tmux_session_name=cd-r7.
2. `send-keys --text "x"` → non-zero; err_name=ErrSendKeysWhileRelayed.
3. As a control: change the row's relay_mode to off and re-run; the
   verb should now attempt the tmux call (it'll error on missing
   tmux session — but NOT with ErrSendKeysWhileRelayed; the guard
   is off).

## Pass criteria

- send-keys exits non-zero with ErrSendKeysWhileRelayed under
  relay-on.
- After flipping relay_mode to off, the same send-keys does NOT
  surface ErrSendKeysWhileRelayed.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-r7', 'check_permission', '/tmp', 'cd-r7', 'on');"

stderr_file=$(mktemp)
if agent-director send-keys --claude-instance-id id-r7 --text "x" 2>"$stderr_file" >/dev/null; then
    echo "FAIL: send-keys on relay-on + check_permission exited 0"; exit 1
fi
err=$(jq -r '.err_name' "$stderr_file")
if [ "$err" != "ErrSendKeysWhileRelayed" ]; then
    echo "FAIL: err_name=$err; want ErrSendKeysWhileRelayed"; exit 1
fi

# Control: flip relay_mode off → guard releases.
sqlite3 "$HOME/.agent-director/state.db" \
    "UPDATE spawns SET relay_mode='off' WHERE claude_instance_id='id-r7';"
stderr2=$(mktemp)
agent-director send-keys --claude-instance-id id-r7 --text "x" 2>"$stderr2" >/dev/null || true
err2=$(jq -r '.err_name' "$stderr2" 2>/dev/null || echo "")
if [ "$err2" = "ErrSendKeysWhileRelayed" ]; then
    echo "FAIL: guard still fires after relay_mode flipped off"; exit 1
fi
echo "relay-7 ok: send-keys guard fires under relay-on; releases under relay-off"
```
