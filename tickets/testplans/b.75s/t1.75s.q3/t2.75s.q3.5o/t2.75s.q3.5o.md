# release-8: doc-drift gate clean at release time

Maps to Epic 13 AC #3. The auto-generated reference docs
(cli-reference.md, mcp-reference.md) must match the manifest at
release time. `make generate` is idempotent; running it on a clean
tree should produce no diff.

## Setup

Per-case DB-reset.

## Steps

1. Copy /work/source into a temp dir.
2. cd in; run `make generate`.
3. `git diff --exit-code docs/cli-reference.md docs/mcp-reference.md`
   must exit 0.

## Pass criteria

- make generate exits 0.
- git diff against the two reference docs is empty (exit 0).

## Script

```bash
set -euo pipefail
work=$(mktemp -d)
cp -a /work/source/* "$work/"
# Also copy .git so git diff has a baseline. Bind-mount is read-only
# so we copy.
if [ -d /work/source/.git ]; then
    cp -a /work/source/.git "$work/.git"
fi
cd "$work"

# Run the generator; should be a no-op against the committed state.
make generate >/dev/null

if ! git diff --exit-code docs/cli-reference.md docs/mcp-reference.md >/dev/null 2>&1; then
    echo "FAIL: doc drift detected after make generate"
    git diff docs/cli-reference.md docs/mcp-reference.md | head -40
    exit 1
fi
echo "release-8 ok: no doc drift between manifest and reference docs"
```
