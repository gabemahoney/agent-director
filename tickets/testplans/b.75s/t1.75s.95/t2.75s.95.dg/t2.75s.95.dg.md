# Test case: doc-drift CI gate rejects a hand-edited generated doc

Maps to Epic 1 AC #6.

## Setup

A clean checkout where `docs/cli-reference.md` and `docs/mcp-reference.md`
exist and are up-to-date relative to the manifest. The CI workflow at
`.github/workflows/doc-drift.yml` is the gate under test, but the same
logic can be exercised locally.

## Local reproduction steps

1. Run `make generate` and `git diff --exit-code docs/`. Confirm exit 0
   (no drift on a clean tree).
2. Manually append a line to `docs/cli-reference.md`, e.g.
   `echo 'hand edit' >> docs/cli-reference.md`.
3. Run `make generate`.
4. Run `git diff --exit-code docs/cli-reference.md` and observe exit.
5. Verify the diff shows the hand-edited line removed (generator
   over-wrote it).

## Pass criteria

- Step 1 reports no drift (exit 0).
- Step 3 (`make generate`) exits 0.
- Step 4 (`git diff --exit-code`) exits 0 again — the hand edit is gone,
  the file matches the generator's output.
- The intermediate `git diff` (between step 2's hand edit and the
  post-generate state) shows the manual line being removed.

## CI gate behavior

In CI, the `doc-drift` workflow runs `make generate` then
`git diff --exit-code docs/`. A PR that modifies the manifest without
running `make generate` will fail the gate, with the offending file
visible in the workflow log.

## Notes

This test guards two regressions: (1) someone hand-edits a generated doc
expecting it to stick; (2) someone changes the manifest and forgets to
regenerate. The doc-drift gate fails the build in both cases.
