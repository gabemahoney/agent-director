automation — `t1.3d3.yy`). Each t2 is a plain-shell case the Docker

## Driver mode

All cases use `DRIVER_MODE=shell`. Each t2 body has a single fenced
` ```bash ` block executed by the driver after the per-case DB-reset
fixture. Cases share a fixture cache under
`/tmp/postinstall-skill-copy-fixtures/`: the first case populates it
(builds `libagent_director.so`, stages `platforms/linux-x64/`, stamps
the umbrella + SKILL.md to a fixture version, packs a tarball, and
sets a `.ready` sentinel) and subsequent cases reuse it. The cache is
container-local; it lives only for the lifetime of one
`make test-docker` invocation.

## Postinstall output isolation

The SR-1.6 five-line cap is on the **postinstall**'s combined stdout +
stderr, not on `bun add`'s own progress output. Cases isolate
postinstall output by filtering captured lines to those that begin
with the prefix `agent-director: ` (the postinstall's mandatory line
prefix). Counts and substring assertions run against the filtered set.

## Trust gate

Bun blocks dependency postinstalls by default. Every consumer fixture
in this testplan declares `"trustedDependencies": ["agent-director"]`
in its `package.json` so `bun add` runs the postinstall without prompt.
The b.5 case keeps trust on and uses `--ignore-scripts` to suppress
explicitly — that is the codepath the Pattern B fallback documents
(SRD SR-1.7).

## SR-7 cross-cutting

The pre-existing testplans under `tickets/testplans/b.75s/` are not
affected by this testplan. The pre-existing TS suite
(`bun test pkg/ts-bun-client/`) and Go suite (`go test ./...`) are
exercised by their own per-Epic testplans and remain green.

## Notes for the orchestrator

- Audit: run `make test-docker EPIC=postinstall-skill-copy` and inspect
  the JSON stream case-by-case.
- The fixture-build step inside `ensure_fixtures` runs `make libagent_director`
  inside the container (CGO build). Allow ~30s on first case.
- The fixture pins the umbrella version to a synthetic `0.4.1-rc.fixture`
  before pack so the case-1 "frontmatter version matches package.json
  version" assertion is satisfied without coupling the testplan to whatever
  ephemeral version (`0.0.0`) sits in `pkg/ts-bun-client/package.json` in
  tree. The release-pipeline lockstep guard (Epic 4) bumps both fields in
  lockstep in production.
- The DB-reset fixture is harmless for these cases — none touch
  `~/.agent-director/state.db`. The reset still runs before each case;
  it is a no-op in practice but exercises the isolation contract.

## tmux session sweep

Cases do NOT exercise `agent-director spawn`. No `cd-*` tmux sessions
are created; the DB-reset fixture's sweep is a no-op for these cases.
