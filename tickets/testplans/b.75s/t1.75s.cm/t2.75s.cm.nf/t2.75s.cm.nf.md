# ci-matrix-3-no-floating-tags

Pins Epic 6 AC: "C toolchain versions are pinned per leg; no
`*-latest` floating tags." A `grep` for the canonical floating-tag
forms (`ubuntu-latest`, `macos-latest`, `windows-latest`, and any
`xcode-*-latest`) must return zero hits inside the workflow file.

## Setup

No DB state. Pure read of the committed workflow file.

## Steps

1. `grep -E` for any of the documented floating-tag forms.
2. If any line matches, fail with the matching lines reported so the
   operator sees exactly which pin needs to be tightened.
3. Otherwise emit pass marker.

## Pass criteria

- `grep` reports zero matches.
- Final line matches `ci-matrix-3 ok: ...`.

## Script

```bash
set -euo pipefail

YML="/work/source/.github/workflows/cabi-matrix.yml"
if [[ ! -f "$YML" ]]; then
    echo "FAIL: $YML not found"
    exit 1
fi

# The patterns covered:
#   ubuntu-latest      - drifts whenever GitHub bumps the default
#   macos-latest       - same; also drifts between Intel and Apple Silicon
#   windows-latest     - we don't ship Windows, but a paste-error
#                        could land it; defensive coverage
#   xcode-*-latest     - covers Xcode-version floating tags in case a
#                        setup-xcode action ever uses one
PATTERN='ubuntu-latest|macos-latest|windows-latest|xcode.*-latest'

if HITS=$(grep -nE "$PATTERN" "$YML"); then
    echo "FAIL: floating runner tags found in cabi-matrix.yml:"
    echo "$HITS"
    exit 1
fi

echo "ci-matrix-3 ok: no *-latest floating runner tags in cabi-matrix.yml"
```
