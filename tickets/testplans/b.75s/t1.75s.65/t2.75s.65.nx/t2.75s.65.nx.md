# relay-6: decide with no open PermissionRequest returns ErrNoOpenPermissionRequest

Per SRD §6.2: decide on a Spawn that isn't sitting on a PermissionRequest
hook surfaces the typed error rather than silently writing a row
nothing is reading.

## Setup

Per-case DB-reset.

## Steps

1. Seed row relay_mode=on but DO NOT run the hook (no
   permission_requests row exists).
2. `decide` → non-zero; err_name=ErrNoOpenPermissionRequest.

## Pass criteria

- decide exits non-zero with the typed error.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-r6', 'waiting', '/tmp', 'cd-r6', 'on');"

stderr_file=$(mktemp)
if claude-director decide --claude-instance-id id-r6 --decision allow 2>"$stderr_file" >/dev/null; then
    echo "FAIL: decide on absent request exited 0"; exit 1
fi
err=$(jq -r '.err_name' "$stderr_file")
if [ "$err" != "ErrNoOpenPermissionRequest" ]; then
    echo "FAIL: err_name=$err; want ErrNoOpenPermissionRequest"; exit 1
fi
echo "relay-6 ok: decide with no open request → ErrNoOpenPermissionRequest"
```
