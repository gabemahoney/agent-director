| case | what it pins |

| `release-7-dry-run-notes-templated` | release.sh --dry-run on a clean synthesized repo with a multi-Epic commit history produces notes grouped by Epic ID. |
| `release-8-doc-drift-clean-at-release-time` | `make generate` + `git diff --exit-code docs/cli-reference.md docs/mcp-reference.md` is clean — no drift between the manifest and the regenerated reference docs. |

## Driver mode

All cases are written for `DRIVER_MODE=shell`. Each t2 body has a
single fenced ` ```bash ` block the driver executes after the
per-case DB-reset fixture.

The harness image now ships:
- `go` (1.22.x) — installed in the Dockerfile so
  `make release-binaries` can run inside the container.
- `make` — installed alongside.
- `/work/source` — read-only bind-mount of the host's repo source
  (added to `make test-docker` for this Epic).

Cases that need to mutate the source (e.g. simulating a dirty tree)
copy `/work/source` into a temp dir first; the read-only mount is
preserved for cases that just need to inspect or `go build` against.

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-13-release`
  yourself, inspect the JSON stream case-by-case.
- The first run rebuilds the harness image with the new Go install;
  subsequent runs use the layer cache.
- release-7's synthesized commit history includes commits with
  `Epic 1.t2.foo` and `Epic 5.t2.bar` references; the case asserts
  both `### Epic 1.t2` and `### Epic 5.t2` group headers appear in
  the dry-run output.

## tmux session sweep

These cases do NOT use `agent-director spawn` or any tmux
interaction. No `cd-*` sessions are created; the db-reset fixture's
sweep is a no-op for these cases.
