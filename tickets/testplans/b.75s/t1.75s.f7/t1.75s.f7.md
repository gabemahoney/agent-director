# Epic 12 — `epic-12-install` testplan

t1 collector for Epic 12 (`t1.8op.tx` — Install skill + persistent
help hooks + README rewrite). Each t2 is a plain-English case the
Docker harness executes. The Epic 12 Progression Contract requires
every t2 to pass under `make test-docker EPIC=epic-12-install`.

## Sourced from

- Epic ticket: `t1.8op.tx` (Acceptance Criteria #1–#3).
- SRD §16.2 (install layout, upgrade-safety pattern), §4.3
  (whitespace-free install path), §18.1 (README quickstart).

## Cases

| case | what it pins |
|---|---|
| `install-1-fresh-install` | Run install.sh in a clean $HOME → ~/.claude-director/ is 0700; state.db is 0600; canonical path is a regular file (mode 0755, temp+mv pattern); hooks present in ~/.claude/settings.json. |
| `install-2-idempotent-rerun` | Run install twice → no duplicate hook entries (matched by command string); both runs exit 0. |
| `install-3-preserve-other-hooks` | Pre-seed ~/.claude/settings.json with a custom user SessionStart hook → install → user's hook still present; claude-director's hook appended. |
| `install-4-missing-tmux-rejected` | PATH missing tmux → install aborts with exit 2 and clear error. |
| `install-5-whitespace-path-rejected` | $HOME path containing whitespace → install aborts with exit 2 (SRD §4.3). |
| `install-6-uninstall-default-preserves-dir` | install → uninstall → hooks removed; binary removed; ~/.claude-director/ still exists (state.db preserved). |
| `install-7-uninstall-purge-removes-all` | install → uninstall --purge --force → ~/.claude-director/ gone entirely. |
| `install-8-keep-prior-snapshot` | First install → second install with `--keep-prior` → canonical is a regular file (temp+mv); `.prior` snapshot retained for one-step rollback. |
| `install-9-no-hooks-leaves-settings-untouched` | Pre-seed `~/.claude/settings.json` with sentinel content → install with `--no-hooks` → settings.json sha256 unchanged; no `.bak` sibling created; summary line `hooks   : skipped (--no-hooks)` printed; binary still installed. |

## Driver mode

All cases are written for `DRIVER_MODE=shell` — the orchestrator runs
the harness with `make test-docker EPIC=epic-12-install`. Each t2
body has a single fenced ` ```bash ` block the driver executes after
the per-case DB-reset fixture.

The skill repo lives at `/work/skills/install-claude-director/`
inside the container (the testplans mount also surfaces the skills
tree). The bundled `claude-director` binary at `/usr/local/bin/`
serves as the source; install.sh's `--binary` flag points at it
explicitly so the test doesn't rely on the in-repo `bin/` layout.

Each case runs install.sh fresh against a per-case `$HOME` (via
`HOME=$(mktemp -d) install.sh ...`) so cases don't interfere with
each other.

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-12-install`
  yourself, inspect the JSON stream case-by-case.
- The harness image has `claude`, `tmux`, and `jq` already on PATH —
  the three install pre-flight requirements. install-4 simulates
  the missing-tmux failure by removing tmux from PATH for one
  invocation rather than uninstalling it system-wide.
- install-5 (whitespace-path-rejected) constructs a $HOME with a
  space in the path; the harness's tmpfs is fine with that.

## tmux session sweep

The install cases do NOT use `claude-director spawn` or any tmux
session creation. No `cd-*` sessions are created; the db-reset
fixture's sweep is a no-op for these cases.

The harness mounts the repo's `skills/` directory read-only inside
the container so the testplan can invoke the bundled install
scripts. This requires a small Dockerfile update — see the
testplan's bash blocks for how each case locates the script.
