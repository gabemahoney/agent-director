# relay-5: decide on relay_mode=off Spawn returns ErrRelayModeOff

Per SRD §6.2: decide is meaningful only when the Spawn opted into
relay mode. A relay_mode=off Spawn rejects the verb without writing
anything.

## Setup

Per-case DB-reset.

## Steps

1. Seed row with relay_mode='off' (no hook running — the row alone
   tells the story).
2. `decide` → non-zero; err_name=ErrRelayModeOff.

## Pass criteria

- decide exits non-zero with the typed error.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-r5', 'waiting', '/tmp', 'cd-r5', 'off');"

stderr_file=$(mktemp)
if agent-director decide --claude-instance-id id-r5 --decision allow 2>"$stderr_file" >/dev/null; then
    echo "FAIL: decide on relay-off exited 0"; exit 1
fi
err=$(jq -r '.err_name' "$stderr_file")
if [ "$err" != "ErrRelayModeOff" ]; then
    echo "FAIL: err_name=$err; want ErrRelayModeOff"; exit 1
fi
echo "relay-5 ok: decide on relay-off → ErrRelayModeOff"
```
