# release-3: Linux binaries are statically linked

Per SRD §16.1: CGO_ENABLED=0 + modernc.org/sqlite yields fully
static binaries on Linux. `ldd` should report "not a dynamic
executable".

## Setup

Per-case DB-reset.

## Steps

1. Build via copy-source + make release-binaries.
2. Run `ldd` on each Linux binary; assert "not a dynamic executable".

## Pass criteria

- Both Linux binaries' ldd output contains "not a dynamic executable".

## Script

```bash
set -euo pipefail
work=$(mktemp -d)
cp -a /work/source/* "$work/"
cd "$work"
make release-binaries >/dev/null 2>&1

for arch in amd64 arm64; do
    out="dist/claude-director-linux-${arch}"
    # ldd exits non-zero on a statically-linked binary, which under
    # `set -euo pipefail` would short-circuit the pipeline before
    # grep gets a chance. Capture into a variable first so the exit
    # code is contained.
    ldd_out=$(ldd "$out" 2>&1 || true)
    if ! printf '%s' "$ldd_out" | grep -q "not a dynamic executable"; then
        echo "FAIL: $out is not statically linked"
        printf '%s\n' "$ldd_out" | sed 's/^/  /'
        exit 1
    fi
done
echo "release-3 ok: both Linux binaries statically linked"
```
