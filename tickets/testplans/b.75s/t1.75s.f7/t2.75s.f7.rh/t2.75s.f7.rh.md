# install-8: --keep-prior snapshots the previous binary for rollback

Per SRD §16.2 (post b.43y): the install uses the temp+atomic-mv
pattern (gh/kubectl/terraform style) instead of the version-manager
symlink pattern. The optional `--keep-prior` flag snapshots the
existing binary to `claude-director.prior` before the swap so the
operator can roll back with `mv .prior canonical`.

## Setup

Per-case DB-reset.

## Steps

1. First install with no `--keep-prior`. Confirm the canonical path
   is a regular file and no `.prior` snapshot exists.
2. Capture the sha256 of the first-install canonical.
3. Second install with `--keep-prior`. Confirm:
   - the canonical path is still a regular file (not a symlink),
   - `claude-director.prior` exists at mode 0755,
   - `.prior`'s sha256 matches the first-install canonical's sha256
     (byte-for-byte the previous binary, preserved for rollback).
4. Roll back via `mv .prior canonical`. Confirm the rolled-back
   canonical's sha256 matches the first-install sha256 and that
   `.prior` no longer exists (the rollback consumed it).

## Pass criteria

- After the first install: no `.prior` snapshot.
- After the second install: `.prior` exists, mode 0755, sha256 matches
  the first install's binary.
- After rollback: canonical's sha256 matches the first install; no
  `.prior` left behind.

## Script

```bash
set -euo pipefail
TESTHOME=$(mktemp -d)

# First install. No --keep-prior, so no .prior snapshot.
HOME="$TESTHOME" /opt/skills/install-claude-director/install.sh \
    --binary /usr/local/bin/claude-director --no-symlink >/dev/null

canonical="$TESTHOME/.claude-director/bin/claude-director"
prior="$TESTHOME/.claude-director/bin/claude-director.prior"

if [ ! -f "$canonical" ] || [ -L "$canonical" ]; then
    echo "FAIL: first-install canonical is not a regular file"; exit 1
fi
if [ -e "$prior" ]; then
    echo "FAIL: .prior snapshot exists after first install without --keep-prior"; exit 1
fi
sha1=$(sha256sum "$canonical" | awk '{print $1}')

# Second install WITH --keep-prior. .prior should snapshot the
# first-install binary; the canonical path should be a fresh regular
# file (the new install).
HOME="$TESTHOME" /opt/skills/install-claude-director/install.sh \
    --binary /usr/local/bin/claude-director --no-symlink --keep-prior >/dev/null

if [ ! -f "$canonical" ] || [ -L "$canonical" ]; then
    echo "FAIL: second-install canonical is not a regular file"; exit 1
fi
if [ ! -f "$prior" ]; then
    echo "FAIL: --keep-prior did not create $prior"; exit 1
fi
prior_mode=$(stat -c '%a' "$prior")
if [ "$prior_mode" != "755" ]; then
    echo "FAIL: .prior mode=$prior_mode; want 755"; exit 1
fi
prior_sha=$(sha256sum "$prior" | awk '{print $1}')
if [ "$prior_sha" != "$sha1" ]; then
    echo "FAIL: .prior sha=$prior_sha; want first-install sha=$sha1"; exit 1
fi

# Rollback: mv .prior canonical. The restored binary must match the
# first-install content by sha256, and .prior should be gone.
mv -f "$prior" "$canonical"
restored_sha=$(sha256sum "$canonical" | awk '{print $1}')
if [ "$restored_sha" != "$sha1" ]; then
    echo "FAIL: rolled-back sha=$restored_sha; want first-install sha=$sha1"; exit 1
fi
if [ -e "$prior" ]; then
    echo "FAIL: .prior still present after mv-rollback"; exit 1
fi

echo "install-8 ok: --keep-prior snapshots prior binary; mv-rollback restores it"
```
