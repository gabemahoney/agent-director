# relay-3: deny with empty reason defaults to "Denied by orchestrator"

Per SRD §6.3: on `deny` with empty reason, the envelope's message
field defaults to "Denied by orchestrator" so the TUI always shows
something actionable. The default lives at the envelope encoder
(not the DB).

## Setup

Per-case DB-reset.

## Steps

1. Seed row, background hook.
2. `decide --decision=deny` (no --reason).
3. Envelope's message is the literal "Denied by orchestrator".

## Pass criteria

- behavior=deny, message="Denied by orchestrator".

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-r3', 'waiting', '/tmp', 'cd-r3', 'on');"

hook_out=$(mktemp)
( printf '%s' '{"hook_event_name":"PermissionRequest","tool_name":"Bash"}' \
  | CLAUDE_DIRECTOR_RELAY_MODE=on \
    CLAUDE_DIRECTOR_INSTANCE_ID=id-r3 \
    claude-director hook >"$hook_out" 2>/dev/null ) &
hook_pid=$!

sleep 0.5
claude-director decide --claude-instance-id id-r3 --decision deny >/dev/null

wait "$hook_pid"
behavior=$(jq -r '.hookSpecificOutput.decision.behavior' "$hook_out")
message=$(jq -r '.hookSpecificOutput.decision.message' "$hook_out")

if [ "$behavior" != "deny" ] || [ "$message" != "Denied by orchestrator" ]; then
    echo "FAIL: envelope behavior=$behavior message=$message; want deny/'Denied by orchestrator'"
    cat "$hook_out"; exit 1
fi
echo "relay-3 ok: empty-reason deny defaults to 'Denied by orchestrator'"
```
