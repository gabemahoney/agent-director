# ts-bun-6-publish-blocked-by-placeholder

Pins Epic 5 AC: the `prepublishOnly` guard fires when `package.json` still
contains the `CHANGEME-H3` placeholder scope. Running
`scripts/check-not-placeholder.ts` directly must exit 1 and emit an error
message containing `CHANGEME-H3`. This is the T11 publish-block mechanism
(see `scripts/check-not-placeholder.ts`).

The `npm publish --dry-run` check is also exercised as a secondary assertion;
it is expected to fail because of the placeholder name, though npm's handling
of `prepublishOnly` in dry-run mode is version-dependent. The direct script
check is authoritative.

## Setup

The script copies `/work/source` to a writable temp dir, runs
`bun install --no-progress`, and then invokes the placeholder-check script.
The `pkg/ts-bun-client/package.json` contains `"name":
"@CHANGEME-H3/agent-director"` so the check must fire.

## Steps

1. Assert `bun` is in PATH; fail clearly if not.
2. Create workspace; `cp -a /work/source/.`.
3. `cd pkg/ts-bun-client && bun install --no-progress`.
4. `bun run scripts/check-not-placeholder.ts`; assert exit 1.
5. Assert output contains `CHANGEME-H3`.
6. Also run `npm publish --dry-run`; note exit code (informational, not
   authoritative — some npm versions skip `prepublishOnly` in dry-run).

## Pass criteria

- `bun run scripts/check-not-placeholder.ts` exits 1.
- Output contains `CHANGEME-H3`.
- Script exits 0; final line contains `ts-bun-6-publish-blocked-by-placeholder ok:`.

## Script

```bash
set -euo pipefail

if ! command -v bun >/dev/null 2>&1; then
    echo "FAIL: bun not found in PATH — check test/Dockerfile BUN_VERSION ARG"
    exit 1
fi

export HOME=/tmp/bun-home-tsbun6
mkdir -p "$HOME"

WORK=$(mktemp -d /tmp/tsbun6_XXXXXX)
cp -a /work/source/. "$WORK/"

cd "$WORK/pkg/ts-bun-client"
bun install --no-progress 2>/dev/null

# The authoritative check: check-not-placeholder.ts must exit 1 with CHANGEME-H3
OUT=$(mktemp /tmp/tsbun6_out.XXXXXX)
EXIT=0
bun run scripts/check-not-placeholder.ts > "$OUT" 2>&1 || EXIT=$?

if [[ "$EXIT" -eq 0 ]]; then
    echo "FAIL: check-not-placeholder.ts exited 0 but should exit 1 (package name still has CHANGEME-H3)"
    exit 1
fi

if ! grep -q "CHANGEME-H3" "$OUT"; then
    echo "FAIL: check-not-placeholder.ts exited $EXIT but output does not contain CHANGEME-H3"
    cat "$OUT" >&2
    exit 1
fi

# Secondary: npm publish --dry-run (informational; some npm versions skip prepublishOnly in dry-run)
NPM_OUT=$(mktemp /tmp/tsbun6_npm.XXXXXX)
NPM_EXIT=0
npm publish --dry-run > "$NPM_OUT" 2>&1 || NPM_EXIT=$?
if [[ "$NPM_EXIT" -eq 0 ]]; then
    # npm dry-run passed — this can happen if npm skips prepublishOnly in dry-run mode.
    # The direct bun run check above is the authoritative gate; log this for info only.
    true
fi

echo "ts-bun-6-publish-blocked-by-placeholder ok: check-not-placeholder.ts exited $EXIT with CHANGEME-H3 in output"
```
