# recov-2: degraded-mode guard refuses to write when probe is empty

Maps to Epic 8 AC #4. The cron-user-mismatch scenario: 0 readable
`AGENT_DIRECTOR_INSTANCE_ID` values + ≥1 live row → find-missing
logs a warning and writes nothing. Reproduced by seeding live rows
directly via `sqlite3` (so no matching live process can possibly
exist).

The case must run with an empty probe set — meaning no claude
processes carrying a `AGENT_DIRECTOR_INSTANCE_ID` env var. The
db-reset fixture kills `cd-*` tmux sessions between cases but the
underlying claude children can take a while to actually exit; the
case starts by sending SIGTERM to any straggler and then polling
`/proc` until clean.

## Setup

Per-case DB-reset.

## Steps

1. SIGTERM any process whose environ carries
   `AGENT_DIRECTOR_INSTANCE_ID` (cleanup from prior cases).
2. Poll `/proc` until no such process remains.
3. INSERT 2 live-state rows directly via `sqlite3`.
4. `find-missing` → expect `count=0`, empty `.ids`, exit 0.
5. Both rows remain in their seeded live states.

## Pass criteria

- find-missing exits 0.
- `count=0`; `.ids` is `[]`.
- Both seeded rows remain in their original live state.

## Script

```bash
set -euo pipefail

# Find every claude-tagged pid (its environ carries the marker env
# var) and ask it to exit. The harness runs as a single user inside
# the container so signal delivery should be unrestricted.
for pid in $(ls /proc 2>/dev/null | grep -E '^[0-9]+$'); do
    if [ -r "/proc/$pid/environ" ] && \
        grep -aqz "^AGENT_DIRECTOR_INSTANCE_ID=" "/proc/$pid/environ" 2>/dev/null; then
        kill "$pid" 2>/dev/null || true
    fi
done

# Wait up to 30s for /proc to be free of any straggler. The previous
# case's survivor may take a moment to actually die.
deadline=$(( $(date +%s) + 30 ))
remaining=1
while [ "$(date +%s)" -lt "$deadline" ]; do
    remaining=0
    for pid in $(ls /proc 2>/dev/null | grep -E '^[0-9]+$'); do
        if [ -r "/proc/$pid/environ" ] && \
            grep -aqz "^AGENT_DIRECTOR_INSTANCE_ID=" "/proc/$pid/environ" 2>/dev/null; then
            remaining=1
            kill -9 "$pid" 2>/dev/null || true
        fi
    done
    if [ "$remaining" -eq 0 ]; then
        break
    fi
    sleep 0.5
done
if [ "$remaining" -eq 1 ]; then
    echo "FAIL: leftover AGENT_DIRECTOR_INSTANCE_ID env in /proc even after SIGKILL+30s"; exit 1
fi

sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-dg-1', 'waiting', '/tmp', 'cd-dg-1', 'off'),
          ('id-dg-2', 'working', '/tmp', 'cd-dg-2', 'off');"

res=$(agent-director find-missing)
count=$(printf '%s' "$res" | jq -r '.count')
ids=$(printf '%s' "$res" | jq -c '.ids')
if [ "$count" != "0" ]; then
    echo "FAIL: count=$count; want 0 (degraded-mode guard should refuse to write; res=$res)"; exit 1
fi
if [ "$ids" != "[]" ]; then
    echo "FAIL: ids=$ids; want []"; exit 1
fi

s1=$(sqlite3 "$HOME/.agent-director/state.db" "SELECT state FROM spawns WHERE claude_instance_id='id-dg-1';")
s2=$(sqlite3 "$HOME/.agent-director/state.db" "SELECT state FROM spawns WHERE claude_instance_id='id-dg-2';")
if [ "$s1" != "waiting" ] || [ "$s2" != "working" ]; then
    echo "FAIL: rows mutated despite degraded-mode guard ($s1, $s2)"; exit 1
fi
echo "recov-2 ok: degraded-mode guard refused to write"
```
