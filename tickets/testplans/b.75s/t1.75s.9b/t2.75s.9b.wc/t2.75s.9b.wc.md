# recov-5: default expire window spares young terminal rows

Per SRD §11: default retention is `defaults.expire_retention_days = 31`.
A terminal row with a fresh `ended_at` must survive a `expire` call
without `--older-than`.

## Setup

Per-case DB-reset.

## Steps

1. INSERT an ended row with `ended_at = CURRENT_TIMESTAMP`.
2. `agent-director expire` (no flag) → expect `count=0`.
3. The row still exists.

## Pass criteria

- expire exits 0.
- `count=0`; `.ids=[]`.
- The young terminal row survives.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode, ended_at)
   VALUES ('id-young', 'ended', '/tmp', 'cd-young', 'off', CURRENT_TIMESTAMP);"

res=$(agent-director expire)
count=$(printf '%s' "$res" | jq -r '.count')
ids=$(printf '%s' "$res" | jq -c '.ids')
if [ "$count" != "0" ] || [ "$ids" != "[]" ]; then
    echo "FAIL: count=$count ids=$ids; want 0/[] (default 31d window)"; exit 1
fi

n=$(sqlite3 "$HOME/.agent-director/state.db" \
    "SELECT COUNT(*) FROM spawns WHERE claude_instance_id='id-young';")
if [ "$n" != "1" ]; then
    echo "FAIL: young terminal row was deleted"; exit 1
fi
echo "recov-5 ok: default 31d window left fresh ended row alone"
```
