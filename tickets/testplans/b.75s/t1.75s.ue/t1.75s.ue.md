# Epic 9 — `epic-09-resume` testplan

t1 collector for Epic 9 (`t1.8op.js` — Resume terminated Spawns).
Each t2 is a plain-English case the Docker harness executes. The
Epic 9 Progression Contract requires every t2 to pass under
`make test-docker EPIC=epic-09-resume`.

## Sourced from

- Epic ticket: `t1.8op.js` (Acceptance Criteria #1–#3).
- SRD §7.5 (parent_id derivation), §8.1 (resume verb), §8.2 (JSONL
  slug rule), §13.1 (error catalogue).

## Cases

| case | what it pins |
|---|---|
| `resume-1-happy-path-from-ended` | spawn → synth SessionStart (populates claude_session_id, state=waiting) → seed JSONL file → synth SessionEnd (state=ended) → kill tmux session → `resume` → new tmux session up under the same name; resume returns the same id. |
| `resume-2-happy-path-from-missing` | same setup but flip state to `missing` via find-missing instead of SessionEnd → resume still works. |
| `resume-3-live-state-rejected` | spawn → leave row in waiting → `resume` → `ErrSpawnNotResumable`; no DB or tmux mutation. |
| `resume-4-no-session-id` | seed `ended` row with empty `claude_session_id` → `resume` → `ErrNoSessionId`. |
| `resume-5-jsonl-missing` | seed `ended` row with session id but NO JSONL file on disk → `resume` → `ErrJsonlMissing`. |
| `resume-6-stale-tmux-collision` | spawn → kill its claude but leave the tmux session alive (manual `tmux new-session` under the canonical name AFTER killing the original) → mark row ended → `resume` → `ErrTmuxSessionCreate`; original tmux session untouched. |
| `resume-7-parent-rederivation` | spawn A → spawn B with `AGENT_DIRECTOR_INSTANCE_ID=A` env → end B → `resume` B from a bare shell (no env) → B's `parent_id` is now NULL. |
| `resume-8-ended_at-cleared-on-resurrection` | spawn → end → resume → synth a fresh SessionStart for the resumed Claude → assert `ended_at` is now NULL and state is `waiting`. |

## Driver mode

All cases are written for `DRIVER_MODE=shell`. Each t2 body has a
single fenced ` ```bash ` block the driver executes after the
per-case DB-reset fixture.

The cases use real `spawn` to create the row + tmux session, but
drive state transitions by synthesizing hook payloads via stdin to
`agent-director hook` (the Epic 3 pattern). The JSONL file is hand-
created at the path `JsonlPath(cwd, claude_session_id)` resolves to
so the resume pre-flight Stat succeeds.

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-09-resume`
  yourself, inspect the JSON stream case-by-case.
- After resume, the new tmux session has a real claude trying to
  bootstrap; without an API key it'll fail to do useful work but the
  session itself comes up. The cases only inspect `tmux has-session`
  and the row's `claude_instance_id` — not any conversational state.
- Case 6 deliberately recreates a stale tmux session under the
  canonical name to provoke the collision check. The case cleans up
  the stale session at the end so it doesn't leak into later cases.

## tmux session sweep

Cases that exercise real `spawn` leave `cd-*` sessions; the db-reset
fixture cleans them between cases.
