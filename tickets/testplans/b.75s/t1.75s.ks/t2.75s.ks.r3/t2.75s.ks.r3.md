# pause-2: pause on a working row returns ErrSpawnNotPausable

Per SRD §9: only `waiting` is pausable. `working` / `pending` /
`ask_user` / `check_permission` all reject with the typed error, and
the verb does NOT send `/exit` (the slash would be interpreted as
input text on a working Spawn, silently producing the wrong outcome).

## Setup

Per-case DB-reset. `working` row; no tmux interaction expected.

## Steps

1. Seed `spawns` row at state `working`, session_name `cd-p2`.
2. `agent-director pause --claude-instance-id id-p2`.
3. Inspect stderr for the typed error envelope.

## Pass criteria

- pause exits non-zero.
- stderr JSON envelope's `err_name` is `ErrSpawnNotPausable`.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-p2', 'working', '/tmp', 'cd-p2', 'off');"
stderr_file=$(mktemp)
if agent-director pause --claude-instance-id id-p2 2>"$stderr_file" >/dev/null; then
    echo "FAIL: pause on working row exited 0"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrSpawnNotPausable" ]; then
    echo "FAIL: err_name=$err_name; want ErrSpawnNotPausable"; exit 1
fi
echo "pause-2 ok: working state → ErrSpawnNotPausable"
```
