# discover-6: --cwd filter matches the canonicalized column verbatim

Per SRD §12: cwd filter is exact match against the canonicalized
column. This case seeds rows with two distinct cwds and filters by
one; only the matching subset is returned.

## Setup

Per-case DB-reset. Two rows seeded with different cwds.

## Steps

1. INSERT row in `/tmp`, row in `/opt`.
2. `agent-director list --cwd /tmp` → only the /tmp row.

## Pass criteria

- list exits 0.
- Result contains exactly `id-d6-tmp`.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-d6-tmp', 'waiting', '/tmp', 'cd-d6-t', 'off'),
          ('id-d6-opt', 'waiting', '/opt', 'cd-d6-o', 'off');"
ids=$(agent-director list --cwd /tmp | jq -r '.spawns[].claude_instance_id' | sort | tr '
' ' ')
expected="id-d6-tmp "
if [ "$ids" != "$expected" ]; then
    echo "FAIL: ids='$ids'; want='$expected'"; exit 1
fi
echo "discover-6 ok: --cwd filter exact-matches the canonical column"
```
