# subprocess-client-4-type-surface

Per SRD §SR-10.6: the published declaration files inside the
installed tarball must be byte-identical to the checked-in goldens at
`pkg/ts-bun-client/test/fixtures/public-surface/`. Mirrors
`pkg/ts-bun-client/test/public-surface.test.ts` but compares the
*installed* `.d.ts` files rather than re-emitting via tsc.

Additive new exports from Epic A are permitted because the goldens
already include them; any pre-existing export rename/remove/reshape
fails the diff.

## Setup

- `${HOME}=$(mktemp -d)`.
- `bun install file:///work/source/dist/agent-director-*.tgz` into
  `${HOME}/proj`.

## Steps

1. Locate the installed `.d.ts` files under
   `${HOME}/proj/node_modules/agent-director/dist/`.
2. Compare each tracked file to the matching golden mounted from
   `/work/source/pkg/ts-bun-client/test/fixtures/public-surface/`.

## Pass criteria

- `diff` exits 0 for each of the three tracked files
  (`index.d.ts`, `errors.d.ts`, `types.d.ts`).

## Script

```bash
set -euo pipefail
HOME_DIR="$HOME"
PROJ="$HOME_DIR/proj"
rm -rf "$PROJ"
mkdir -p "$PROJ"
cd "$PROJ"

. /work/source/tickets/testplans/b.eiv/t1.19d.6c/build-tarball.sh

cat > "$PROJ/package.json" <<'EOF'
{
  "name": "subprocess-type-surface-consumer",
  "version": "0.0.0",
  "type": "module"
}
EOF

bun install "file://${TARBALL}" >/dev/null

GOLDEN_DIR=/work/source/pkg/ts-bun-client/test/fixtures/public-surface
INSTALLED_DIR="$PROJ/node_modules/agent-director/dist"

failed=0
for f in index.d.ts errors.d.ts types.d.ts; do
    if ! diff -u "$GOLDEN_DIR/${f}.golden" "$INSTALLED_DIR/$f"; then
        echo "FAIL: $f differs from golden"
        failed=1
    fi
done
if [ "$failed" != "0" ]; then exit 1; fi
echo "subprocess-client-4-type-surface ok"
```
