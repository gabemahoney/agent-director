# Epic pkg/cabi ‚Äî `epic-qe2-2-cabi` testplan

t1 collector for Epic 2 (`t1.qe2.qx` ‚Äî C-ABI shim for pkg/api). Each t2 is a
plain-shell case the Docker harness executes. The Epic 2 Progression Contract
requires every t2 to pass under `make test-docker EPIC=epic-qe2-2-cabi`.

## Sourced from

- Epic ticket: `t1.qe2.qx` (C-ABI shim acceptance criteria, all verbs).
- Task: `t2.qe2.qx.3x` (Docker testplan for pkg/cabi).
- Cases: `cabi-1-build-and-symbols`, `cabi-2-ad-version-envelope`,
  `cabi-3-unknown-handle-error`, `cabi-4-malformed-json-error`,
  `cabi-5-double-close-noop`.

## Cases

| case | what it pins |
|---|---|
| `cabi-1-build-and-symbols` | `nm -gD libagent_director.so` ‚Äî every T symbol is `ad_`-prefixed; zero non-`ad_` T-symbols exported. |
| `cabi-2-ad-version-envelope` | `ad_version("{}")` returns valid JSON with `version` and `commit` fields; no `err_name` present. |
| `cabi-3-unknown-handle-error` | `ad_status` with a never-minted handle returns `err_name == "ErrUnknownHandle"`. |
| `cabi-4-malformed-json-error` | Malformed JSON to `ad_open` returns a parseable `ErrInternal` envelope; process exits 0 (no panic crosses the C boundary). |
| `cabi-5-double-close-noop` | `ad_open` ‚Üí handle; `ad_close(handle)` success; `ad_close(handle)` again success (no-op); `ad_close(unminted)` success. |

## Driver mode

All cases use `DRIVER_MODE=shell`. Each t2 body has a single fenced `bash`
block executed by the driver after the per-case DB-reset fixture.

Cases 1‚Äì5 build `libagent_director.so` from `/work/source` (the read-only
source bind-mount) into a writable temp directory. `CGO_ENABLED=1 go build
-buildmode=c-shared` requires a C compiler at build time. **Before running
this testplan, add `build-essential` and `binutils` to `test/Dockerfile`.**

Cases 2‚Äì5 use a tiny inline C probe compiled with `gcc` to call the `.so`
via `dlopen`/`dlsym` ‚Äî the same pattern as the in-tree `dlopen_*_test.go`
suite. Case 1 copies the freshly built `.so` to `/tmp/ad_cabi_test.so`; cases
2‚Äì5 reuse that path (building fresh if the file is absent), so a full
five-case run pays the CGO build cost exactly once.

## Notes for the orchestrator

- Audit: `make test-docker EPIC=epic-qe2-2-cabi`; inspect the JSON stream
  case-by-case.
- The Dockerfile must include `build-essential` (gcc) and `binutils` (nm).
- None of these cases use `agent-director spawn` or tmux. The db-reset
  fixture's `cd-*` session sweep is a no-op for all five cases.
- Case 5 (`ad_open` ‚Üí `ad_close`) creates a temp SQLite file at
  `/tmp/probe5_state.db` inside the container; it does not touch the
  db-reset-managed store.

## tmux session sweep

No `cd-*` sessions are created. The db-reset fixture's session sweep is a
no-op for these cases.
