# release-5: strict semver rejection (v0.1.0-rc1 → exit 2)

Per SRD §16.4: v1 release skill only accepts strict
MAJOR.MINOR.PATCH semver. Pre-release tags are rejected.

## Setup

Per-case DB-reset.

## Steps

1. Invoke release.sh with VERSION=v0.1.0-rc1 in a clean synthesized
   git repo so the dirty-tree check doesn't fire first.
2. Assert exit code is 2 and stderr mentions "semver".

## Pass criteria

- Exit code is 2.
- Stderr contains "semver".

## Script

```bash
set -euo pipefail
work=$(mktemp -d)
cd "$work"
git init -q
git config user.email test@example.com
git config user.name test
echo "x" > a.txt
git add a.txt
git -c commit.gpgsign=false commit -q -m "init"

stderr_file=$(mktemp)
set +e
/opt/skills/release-agent-director/release.sh v0.1.0-rc1 --dry-run 2>"$stderr_file"
ec=$?
set -e

if [ "$ec" != "2" ]; then
    echo "FAIL: exit=$ec; want 2"; cat "$stderr_file"; exit 1
fi
if ! grep -qi 'semver' "$stderr_file"; then
    echo "FAIL: stderr did not mention semver"; cat "$stderr_file"; exit 1
fi
echo "release-5 ok: v0.1.0-rc1 rejected with semver error"
```
