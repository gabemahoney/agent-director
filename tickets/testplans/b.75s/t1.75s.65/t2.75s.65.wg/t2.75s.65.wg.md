# relay-4: second decide returns ErrAlreadyDecided

Per SRD §6.2: the decide UPDATE is guarded by `decision IS NULL`.
First call wins; second call sees a populated decision column and
the UPDATE no-ops; the verb surfaces ErrAlreadyDecided.

## Setup

Per-case DB-reset.

## Steps

1. Seed row, background hook.
2. First decide --decision allow → exits 0.
3. Second decide --decision deny → non-zero with
   err_name=ErrAlreadyDecided.

## Pass criteria

- Second decide exits non-zero with the typed error.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-r4', 'waiting', '/tmp', 'cd-r4', 'on');"

hook_out=$(mktemp)
( printf '%s' '{"hook_event_name":"PermissionRequest","tool_name":"Bash"}' \
  | CLAUDE_DIRECTOR_RELAY_MODE=on \
    CLAUDE_DIRECTOR_INSTANCE_ID=id-r4 \
    claude-director hook >"$hook_out" 2>/dev/null ) &
hook_pid=$!

sleep 0.5
claude-director decide --claude-instance-id id-r4 --decision allow >/dev/null
wait "$hook_pid"

# Second decide AFTER the hook has exited — first decide already wrote
# the row, so this one trips the IS-NULL guard.
stderr_file=$(mktemp)
if claude-director decide --claude-instance-id id-r4 --decision deny 2>"$stderr_file" >/dev/null; then
    echo "FAIL: second decide exited 0; want ErrAlreadyDecided"; exit 1
fi
err=$(jq -r '.err_name' "$stderr_file")
if [ "$err" != "ErrAlreadyDecided" ]; then
    echo "FAIL: err_name=$err; want ErrAlreadyDecided"; exit 1
fi
echo "relay-4 ok: second decide → ErrAlreadyDecided"
```
