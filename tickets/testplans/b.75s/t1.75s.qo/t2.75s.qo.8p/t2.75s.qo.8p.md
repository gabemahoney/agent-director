# spawn-2: UserPromptSubmit hook drives state to working

Maps to Epic 3 AC #2 (attaching to tmux and typing a prompt flips state
to working). The harness exercises the SRD §5.2 mapping directly via a
synthesized hook payload — no real Claude is needed for the state pin.

## Setup

Per-case DB-reset has wiped the DB.

## Steps

1. `agent-director spawn --cwd /tmp` → capture `claude_instance_id`.
2. Fire SessionStart hook → state should be `waiting`.
3. Fire `UserPromptSubmit` hook → state should be `working`.

## Pass criteria

- After step 2, `status` returns `waiting`.
- After step 3, `status` returns `working`.

## Script

```bash
set -euo pipefail
id="$(agent-director spawn --cwd /tmp | jq -r '.claude_instance_id')"
test -n "$id"
printf '{"hook_event_name":"SessionStart","transcript_path":"/x/abc.jsonl"}' \
  | AGENT_DIRECTOR_INSTANCE_ID="$id" agent-director hook
state="$(agent-director status --claude-instance-id "$id" | jq -r '.state')"
test "$state" = "waiting"
printf '{"hook_event_name":"UserPromptSubmit"}' \
  | AGENT_DIRECTOR_INSTANCE_ID="$id" agent-director hook
state="$(agent-director status --claude-instance-id "$id" | jq -r '.state')"
test "$state" = "working"
echo "spawn-2 ok: id=$id state=working"
```
