# Test case: ~/.claude-director/ + state.db created with correct modes, idempotently

Maps to Epic 1 AC #4.

## Setup

HOME is set to a fresh temp directory `$tmpdir` that does NOT yet contain
`.claude-director`. `./bin/claude-director` exists.

## Steps

1. Run `HOME=$tmpdir ./bin/claude-director help` and capture exit code.
2. `stat -c '%a' $tmpdir/.claude-director` and read the mode.
3. `stat -c '%a' $tmpdir/.claude-director/state.db` and read the mode.
4. Run `HOME=$tmpdir ./bin/claude-director help` a SECOND time.
5. Re-check both modes.

## Pass criteria

- First invocation exits 0.
- After step 1: `.claude-director` mode is `700`, `state.db` mode is `600`.
- Second invocation exits 0.
- After step 4: the modes are UNCHANGED — still 700 and 600.

## Notes

The "second run leaves modes alone" half of the test catches a regression
where someone widens permissions every time `Open` is called (e.g. by
calling `os.Chmod` with a different mode than the original). Both halves
matter.
