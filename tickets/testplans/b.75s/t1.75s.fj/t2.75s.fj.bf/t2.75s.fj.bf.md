# discover-5: parent_id auto-detected from AGENT_DIRECTOR_INSTANCE_ID

Per SRD §7.5: when `spawn` is invoked with `AGENT_DIRECTOR_INSTANCE_ID`
set in its own process env, the new row's `parent_id` is that value.
Maps to Epic 6 AC #2.

Real `spawn` is used here — the parent_id env-read is exactly the
behavior the case is pinning, so the spawn pipeline must run end to
end.

## Setup

Per-case DB-reset. A `/tmp` cwd. Two `spawn` calls: parent first, then
a child with the env var threaded through.

## Steps

1. `agent-director spawn --cwd /tmp` → parent id.
2. `AGENT_DIRECTOR_INSTANCE_ID=<parent> agent-director spawn --cwd /tmp`
   → child id.
3. `agent-director get --claude-instance-id <child>` → parent_id
   field equals `<parent>`.
4. `agent-director list --parent <parent>` → returns only the child.

## Pass criteria

- Both spawns exit 0 with non-empty ids.
- Child row's `parent_id` matches the parent's id.
- `list --parent` returns exactly one row (the child).

## Script

```bash
set -euo pipefail
parent=$(agent-director spawn --cwd /tmp | jq -r '.claude_instance_id')
test -n "$parent"
child=$(AGENT_DIRECTOR_INSTANCE_ID="$parent" agent-director spawn --cwd /tmp \
        | jq -r '.claude_instance_id')
test -n "$child"
got_parent=$(agent-director get --claude-instance-id "$child" | jq -r '.parent_id')
if [ "$got_parent" != "$parent" ]; then
    echo "FAIL: child.parent_id='$got_parent'; want '$parent'"; exit 1
fi
listed=$(agent-director list --parent "$parent" | jq -r '.spawns[].claude_instance_id' | sort | tr '
' ' ')
expected="$child "
if [ "$listed" != "$expected" ]; then
    echo "FAIL: list --parent returned '$listed'; want '$expected'"; exit 1
fi
echo "discover-5 ok: parent_id auto-detected; list --parent returns child"
```
