# resume-5: resume with JSONL missing returns ErrJsonlMissing

Per SRD §8.1: resume Stats the JSONL transcript pre-flight. If the
file doesn't exist (hand-deleted, never written, etc.) the verb
surfaces ErrJsonlMissing rather than letting `claude --resume` fail
opaquely.

## Setup

Per-case DB-reset. Seed an `ended` row with a populated session_id
but DO NOT create the JSONL file on disk.

## Steps

1. INSERT row with state=ended, claude_session_id='no-jsonl-here'.
2. `resume` → non-zero; `err_name=ErrJsonlMissing`.

## Pass criteria

- resume exits non-zero with the typed error.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode, claude_session_id, ended_at)
   VALUES ('id-no-jsonl', 'ended', '/tmp', 'cd-no-jsonl', 'off', 'no-jsonl-here', CURRENT_TIMESTAMP);"

# Defensive: ensure no JSONL file with this name exists anywhere we'd
# resolve to. The slug for /tmp is `-tmp`.
rm -f "$HOME/.claude/projects/-tmp/no-jsonl-here.jsonl"

stderr_file=$(mktemp)
if agent-director resume --claude-instance-id id-no-jsonl 2>"$stderr_file" >/dev/null; then
    echo "FAIL: resume with missing JSONL exited 0"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrJsonlMissing" ]; then
    echo "FAIL: err_name=$err_name; want ErrJsonlMissing"; exit 1
fi
echo "resume-5 ok: missing JSONL → ErrJsonlMissing"
```
