# Test case: insert a spawn row (precondition for smoke-3)

Maps to Subtask 2.3 acceptance criteria: paired case A. Inserts a row into
the `spawns` table. The next case (smoke-3) asserts that the DB-reset
fixture wiped this row between cases.

## Setup

The per-case DB-reset fixture has run; `~/.agent-director/state.db` is
fresh and contains the schema-v1 tables with zero rows.

## Steps

1. Confirm `sqlite3` is on PATH inside the container.
2. Confirm `~/.agent-director/state.db` exists and `PRAGMA user_version`
   returns 1 (schema v1, set by `agent-director help`'s setupStore call).
3. Insert one row into `spawns` with a known `claude_instance_id`
   (`smoke-leftover`).
4. Confirm exactly one row is now in `spawns`.

## Pass criteria

`COUNT(*)` from `spawns` returns `1` after the insert; exit 0.

```bash
set -euo pipefail
DB="$HOME/.agent-director/state.db"
test -r "$DB"
version=$(sqlite3 "$DB" 'PRAGMA user_version')
test "$version" = "1"
sqlite3 "$DB" "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode) VALUES ('smoke-leftover', 'running', '/tmp', 'cd-smoke-2', 'passthrough');"
count=$(sqlite3 "$DB" 'SELECT COUNT(*) FROM spawns;')
test "$count" = "1"
echo "smoke-2 ok: 1 row in spawns (leftover for smoke-3 to detect)"
```
