# subprocess-coverage-6-store-path-default

Per bug **b.32k**: the TS Client must NOT mimic the CLI's default
resolution for `storePath`. When the consumer omits `storePath` from
`new Client({...})`, the Client must forward no `--store-path` flag in
argv, letting the CLI's own three-tier resolution (flag → config.toml
`[store] db_path` → hardcoded `~/.agent-director/state.db`) determine
the actual store location. Conversely, when the consumer supplies
`storePath`, it must appear verbatim in subprocess argv as
`--store-path <path>`.

This case proves both directions in a single fixture. Each direction
exercises a stub CLI binary that captures its own `$@` into the result
envelope so the driver can assert argv-shape from the consumer side.
The stub also echoes its inherited `HOME` so the omitted-path branch
proves the CLI sees the consumer's HOME (i.e. the surface against
which the CLI's default would resolve).

## Setup

- `${HOME}=$(mktemp -d)` (the t2 driver inherits this from the harness).
- `bun install file://...tgz` into `${HOME}/proj`.
- Stage a stub CLI binary that prints `{"version", "commit", "argv",
  "home"}` — `argv` is the JSON-encoded `"$@"` array, `home` is `$HOME`.

## Steps

1. Driver constructs `new Client({})` (NO `storePath` field) and calls
   `version({})`. Asserts the stub's echoed `argv` array contains NO
   `--store-path` token, AND the stub's echoed `HOME` equals the
   consumer's `process.env.HOME` (proving the CLI's default-resolution
   surface is the consumer's HOME, untouched).
2. Driver constructs `new Client({ storePath: "/tmp/explicit.db" })`
   and calls `version({})`. Asserts the stub's echoed `argv` array
   contains the two adjacent tokens `--store-path` then
   `/tmp/explicit.db`.

## Pass criteria

- Omit case: `argv` array contains no element equal to `--store-path`;
  `home` field equals `process.env.HOME` of the driver.
- Provide case: `argv` array contains `--store-path` immediately
  followed by `/tmp/explicit.db`.
- Driver prints `OK store-path-omit` and `OK store-path-provide` on
  stdout; the shell wrapper greps for both before declaring pass.

## Script

```bash
set -euo pipefail
HOME_DIR="$HOME"
PROJ="$HOME_DIR/proj"
rm -rf "$PROJ"
mkdir -p "$PROJ"
cd "$PROJ"

. /work/source/tickets/testplans/b.eiv/t1.19d.6c/build-tarball.sh

cat > "$PROJ/package.json" <<'EOF'
{
  "name": "subprocess-coverage-sp-consumer",
  "version": "0.0.0",
  "type": "module"
}
EOF

bun install "file://${TARBALL}" >/dev/null

DEST="$PROJ/node_modules/@agent-director/linux-x64"
mkdir -p "$DEST/bin"
cp "$PLATFORM_PKG_DIR/package.json" "$DEST/package.json"

# Stub CLI: echo its argv (positional args) and HOME back through a valid
# version envelope. The driver parses the JSON envelope and inspects the
# argv array to assert presence / absence of --store-path.
cat > "$DEST/bin/agent-director" <<'STUB_EOF'
#!/usr/bin/env bash
# Build a JSON array literal from "$@" with shell-only quoting (test
# inputs are plain ASCII paths, so a simple double-quote wrap is safe).
argv_json="["
first=1
for a in "$@"; do
    if [ $first -eq 1 ]; then first=0; else argv_json="${argv_json},"; fi
    argv_json="${argv_json}\"${a}\""
done
argv_json="${argv_json}]"
printf '{"version":"0.0.0-stub","commit":"stub","argv":%s,"home":"%s"}\n' \
    "$argv_json" "${HOME:-}"
STUB_EOF
chmod +x "$DEST/bin/agent-director"

cp /work/source/tickets/testplans/b.eiv/t1.eiv.cv/t2.eiv.cv.sp/driver.mjs "$PROJ/driver.mjs"

out=$(bun run "$PROJ/driver.mjs" 2>&1)
echo "$out"
grep -q "OK store-path-omit"    <<<"$out" || { echo "FAIL: store-path-omit assertion missing"; exit 1; }
grep -q "OK store-path-provide" <<<"$out" || { echo "FAIL: store-path-provide assertion missing"; exit 1; }
echo "subprocess-coverage-6-store-path-default ok"
```
