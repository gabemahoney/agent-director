# pause-4: pause on stuck waiting Spawn times out

Per SRD §9: if the row never reaches `ended` within
`pause.timeout_seconds`, the verb returns `ErrPauseTimeout`. The
harness writes a 2-second timeout to config.toml and confirms the
verb both errors out and waits the full deadline (no premature
short-circuit).

## Setup

Per-case DB-reset. A `cat` pane under `cd-p4` (so `/exit` has
somewhere to land). A scratch `~/.agent-director/config.toml`
pinning `pause.timeout_seconds = 2`.

## Steps

1. Start `tmux new-session -d -s cd-p4 -- cat`.
2. Seed `spawns` row at state `waiting`, session_name `cd-p4`.
3. Write `~/.agent-director/config.toml` with
   `[pause] timeout_seconds = 2`.
4. `agent-director pause --claude-instance-id id-p4` — no state
   flip occurs; the deadline must trip.
5. Inspect stderr for the typed envelope.

## Pass criteria

- pause exits non-zero.
- stderr JSON envelope's `err_name` is `ErrPauseTimeout`.
- Elapsed wall-clock ≥ 2s and ≤ 5s (proves the verb honored the
  deadline rather than short-circuiting or hanging).

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-p4 -- cat
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-p4', 'waiting', '/tmp', 'cd-p4', 'off');"
mkdir -p "$HOME/.agent-director"
printf '[pause]
timeout_seconds = 2
' > "$HOME/.agent-director/config.toml"
stderr_file=$(mktemp)
start=$(date +%s)
if agent-director pause --claude-instance-id id-p4 2>"$stderr_file" >/dev/null; then
    echo "FAIL: pause on stuck waiting row exited 0"; exit 1
fi
elapsed=$(( $(date +%s) - start ))
err_name=$(jq -r '.err_name' "$stderr_file")
rm -f "$HOME/.agent-director/config.toml"
if [ "$err_name" != "ErrPauseTimeout" ]; then
    echo "FAIL: err_name=$err_name; want ErrPauseTimeout"; exit 1
fi
if [ "$elapsed" -lt 2 ] || [ "$elapsed" -gt 5 ]; then
    echo "FAIL: elapsed=${elapsed}s; want 2-5s"; exit 1
fi
echo "pause-4 ok: stuck waiting → ErrPauseTimeout after ${elapsed}s"
```
