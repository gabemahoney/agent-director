# pkg-api-5-callable-verbs-set

Pins E1 AC: `manifest.CallableVerbs()` returns the expected set — exactly
the 15 verbs in canonical order, with `serve`, `hook`, and `help`
excluded.

The `b.qe2.md` tracking note says "`help` included" but the landed code
marks `help` as `Callable: false` because help is rendered cmd-side as
informational output (the function's own doc comment in
`pkg/api/manifest/manifest.go` enumerates `help, serve, hook` as the
excluded triad, and `manifest_test.go::TestCallableVerbsExcludesNonCallable`
asserts the same). This case pins what the code actually does.

Approach: write a tiny Go program that imports the manifest package,
prints `CallableVerbs()` names one per line, and compare against the
expected ordered list. We use a stand-alone program (not `go test`) so
the case has no overlap with what case 1 already covers.

## Setup

No DB state needed. The script copies `/work/source` to a writable temp
dir (so `go run` can use its module cache without writing into the
read-only mount).

## Steps

1. Assert `go` and `diff` are in PATH; fail with a clear message if not.
2. Create a writable workspace under `/tmp/` and `cp -a /work/source/.` into it.
3. Write a tiny driver Go program at `<workspace>/tools/list-callable-verbs/main.go`
   that imports `pkg/api/manifest`, iterates `manifest.CallableVerbs()`,
   and prints each verb's `Name` on its own line.
4. `go run ./tools/list-callable-verbs` from inside the workspace; capture
   stdout.
5. `diff` against the expected fixed list. Assert empty diff.
6. Also assert `serve`, `hook`, `help` are absent (defensive — diff
   already catches it, but the dedicated line gives a clearer failure
   message).

## Pass criteria

- `go run ./tools/list-callable-verbs` exits 0.
- Output is exactly the 15 canonical callable-verb names in the canonical
  order (`spawn` first, `version` last).
- None of `serve`, `hook`, `help` appear in the output.
- Script exits 0; final line matches `pkg-api-5-callable-verbs-set ok: ...`.

## Script

```bash
set -euo pipefail

if ! command -v go >/dev/null 2>&1; then
    echo "FAIL: go not found in PATH"
    exit 1
fi
if ! command -v diff >/dev/null 2>&1; then
    echo "FAIL: diff not found in PATH"
    exit 1
fi

WORK=$(mktemp -d /tmp/pkgapi5_XXXXXX)
cp -a /work/source/. "$WORK/"

DRIVER_DIR="$WORK/tools/list-callable-verbs"
mkdir -p "$DRIVER_DIR"
cat > "$DRIVER_DIR/main.go" <<'GO'
package main

import (
	"fmt"

	"github.com/gabemahoney/agent-director/pkg/api/manifest"
)

func main() {
	for _, v := range manifest.CallableVerbs() {
		fmt.Println(v.Name)
	}
}
GO

cd "$WORK"

RUN_OUT=$(mktemp /tmp/pkgapi5_run.XXXXXX)
RUN_EXIT=0
go run ./tools/list-callable-verbs > "$RUN_OUT" 2>/tmp/pkgapi5_run.err || RUN_EXIT=$?
if [[ "$RUN_EXIT" -ne 0 ]]; then
    echo "FAIL: 'go run ./tools/list-callable-verbs' exited with code $RUN_EXIT"
    cat /tmp/pkgapi5_run.err >&2 || true
    exit 1
fi

# Canonical CallableVerbs() ordering — keep in sync with
# pkg/api/manifest/manifest_test.go::TestCallableVerbsOrder.
EXPECTED=$(mktemp /tmp/pkgapi5_expected.XXXXXX)
cat > "$EXPECTED" <<'EOF'
spawn
status
get
send-keys
read-pane
kill
decide
resume
find-missing
expire
delete
make-template
list
pause
version
EOF

if ! diff -u "$EXPECTED" "$RUN_OUT" > /tmp/pkgapi5_diff 2>&1; then
    echo "FAIL: CallableVerbs() output does not match expected canonical list"
    cat /tmp/pkgapi5_diff >&2
    exit 1
fi

# Defensive: confirm the excluded triad is absent. The diff above would
# already catch any of these appearing, but a dedicated check gives a
# clearer failure message if someone flips Callable on help/serve/hook.
for forbidden in serve hook help; do
    if grep -qxE "$forbidden" "$RUN_OUT"; then
        echo "FAIL: CallableVerbs() unexpectedly contains '$forbidden' — it should be Callable: false"
        exit 1
    fi
done

COUNT=$(wc -l < "$RUN_OUT" | tr -d ' ')
echo "pkg-api-5-callable-verbs-set ok: CallableVerbs() returned $COUNT verbs in canonical order; serve/hook/help excluded"
```
