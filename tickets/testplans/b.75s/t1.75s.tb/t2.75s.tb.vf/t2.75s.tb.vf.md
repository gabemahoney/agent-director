# ts-bun-5-error-subclass-drift

Pins Epic 5 AC: the catalog-drift regression gate (`errors-catalog-drift.test.ts`)
detects when a new entry is added to `pkg/api/errnames/catalog.json` without a
matching TypeScript subclass. Injecting a synthetic `ErrFake` entry causes the
test to fail with a message identifying `ErrFake` as "in catalog but not in TS".

This is the inverse of the allow-list path: the test must actively fail when
the catalog grows beyond what the TS error module exports.

## Setup

The script copies `/work/source` to a writable temp dir, uses `jq` to append a
fake entry `{"name":"ErrFake","package":"fake","description":"injected"}` to
`pkg/api/errnames/catalog.json`, then runs
`bun test test/errors-catalog-drift.test.ts` and asserts the exit code is
non-zero and the output mentions `ErrFake`.

## Steps

1. Assert `bun` and `jq` are in PATH; fail clearly if not.
2. Create workspace; `cp -a /work/source/.`.
3. Append `ErrFake` entry to `<workspace>/pkg/api/errnames/catalog.json` with `jq`.
4. `cd pkg/ts-bun-client && bun install --no-progress`.
5. `bun test test/errors-catalog-drift.test.ts`; assert exit NON-ZERO.
6. Assert output contains `ErrFake` or `in catalog but not in TS`.

## Pass criteria

- `bun test test/errors-catalog-drift.test.ts` exits non-zero (drift detected).
- Output contains `ErrFake` or `in catalog but not in TS`.
- Script exits 0; final line contains `ts-bun-5-error-subclass-drift ok:`.

## Script

```bash
set -euo pipefail

if ! command -v bun >/dev/null 2>&1; then
    echo "FAIL: bun not found in PATH — check test/Dockerfile BUN_VERSION ARG"
    exit 1
fi
if ! command -v jq >/dev/null 2>&1; then
    echo "FAIL: jq not found in PATH"
    exit 1
fi

export HOME=/tmp/bun-home-tsbun5
mkdir -p "$HOME"

WORK=$(mktemp -d /tmp/tsbun5_XXXXXX)
cp -a /work/source/. "$WORK/"

CATALOG="$WORK/pkg/api/errnames/catalog.json"
if [[ ! -f "$CATALOG" ]]; then
    echo "FAIL: catalog.json not found at $CATALOG"
    exit 1
fi

# Inject a synthetic entry that has no TS counterpart
jq '. += [{"name":"ErrFake","package":"fake","description":"injected for drift regression test"}]' \
    "$CATALOG" > /tmp/tsbun5_catalog_modified.json
mv /tmp/tsbun5_catalog_modified.json "$CATALOG"

# Verify injection
if ! jq -e '.[] | select(.name == "ErrFake")' "$CATALOG" >/dev/null 2>&1; then
    echo "FAIL: ErrFake was not successfully injected into catalog.json"
    exit 1
fi

cd "$WORK/pkg/ts-bun-client"
bun install --no-progress 2>/dev/null

# Run the drift test — it MUST fail because ErrFake is in catalog but not in TS
OUT=$(mktemp /tmp/tsbun5_out.XXXXXX)
EXIT=0
bun test test/errors-catalog-drift.test.ts > "$OUT" 2>&1 || EXIT=$?

if [[ "$EXIT" -eq 0 ]]; then
    echo "FAIL: errors-catalog-drift.test.ts should have failed with injected ErrFake but exited 0"
    cat "$OUT" >&2
    exit 1
fi

# Confirm the failure mentions ErrFake or the drift description
if ! grep -qE "ErrFake|in catalog but not in TS" "$OUT"; then
    echo "FAIL: drift test failed but output does not mention ErrFake or 'in catalog but not in TS'"
    cat "$OUT" >&2
    exit 1
fi

echo "ts-bun-5-error-subclass-drift ok: catalog injection of ErrFake detected (exit=$EXIT, ErrFake mentioned in output)"
```
