# interact-6: read-pane --ansi returns raw bytes

Per Epic 4 testplan: `read-pane --ansi` (the opt-in raw mode) returns
the pane bytes verbatim, including ANSI escape sequences. The opposite
of interact-5: where the default strips escapes, `--ansi` preserves
them so a future TUI viewer can re-render the color layout.

## Setup

Per-case DB-reset. Use the same printf-into-bash trick as interact-5
to produce a deterministic ANSI corpus.

## Steps

1. Start `tmux new-session -d -s cd-i6 -- bash`.
2. Seed `spawns` row.
3. tmux send-keys a printf that emits SGR red + text.
4. `agent-director read-pane --claude-instance-id id-i6 --ansi` →
   parse the `.pane` JSON value and confirm an ESC byte (0x1B) survives.

## Pass criteria

- Output contains the ESC byte (in JSON form, that's ``).

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-i6 -- bash
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i6', 'waiting', '/tmp', 'cd-i6', 'off');"
tmux send-keys -t cd-i6:0.0 $'printf "\\033[31mRED\\033[0m END\
"' Enter
sleep 0.3
raw_json="$(agent-director read-pane --claude-instance-id id-i6 --ansi --n-lines 50)"
# The JSON string field carries the escape byte as .
if ! echo "$raw_json" | grep -q '\\u001b\['; then
    echo "FAIL: --ansi did not preserve escape codes"
    echo "$raw_json"
    exit 1
fi
echo "interact-6 ok: raw bytes preserved under --ansi"
```
