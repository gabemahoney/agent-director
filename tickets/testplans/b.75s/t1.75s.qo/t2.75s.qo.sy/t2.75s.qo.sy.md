# spawn-6: CLAUDE_DIRECTOR_* env key is rejected

Maps to SRD §7.2 step 4 / §14.4: `extra_env` keys matching the
`CLAUDE_DIRECTOR_*` prefix are reserved. Auth env vars
(`ANTHROPIC_API_KEY`, `CLAUDE_CODE_OAUTH_TOKEN`) do NOT carry the
prefix and are not reserved (see spawn-8).

## Setup

Per-case DB-reset has wiped the DB.

## Steps

1. `claude-director spawn --cwd /tmp --extra-env CLAUDE_DIRECTOR_FOO=bar`
   → non-zero exit.
2. Stderr JSON envelope with `err_name = ErrReservedEnvKey`.
3. Zero spawns rows.

## Pass criteria

- Non-zero exit.
- err_name == ErrReservedEnvKey.
- Zero spawns rows.

## Script

```bash
set -euo pipefail
stderr_file=$(mktemp)
trap 'rm -f "$stderr_file"' EXIT
if claude-director spawn --cwd /tmp --extra-env CLAUDE_DIRECTOR_FOO=bar 2>"$stderr_file"; then
    echo "spawn unexpectedly succeeded"; exit 1
fi
err_name="$(jq -r '.err_name' <"$stderr_file")"
test "$err_name" = "ErrReservedEnvKey"
n="$(sqlite3 "$HOME/.claude-director/state.db" 'SELECT COUNT(*) FROM spawns;')"
test "$n" = "0"
echo "spawn-6 ok: ErrReservedEnvKey; no row"
```
