# mcp-1: initialize handshake

Per SRD §3.3: the server is long-lived. `initialize` is the first
request it expects; the response carries protocolVersion + serverInfo
+ capabilities. This case pins the handshake bytes-on-the-wire.

## Setup

Per-case DB-reset.

## Steps

1. Pipe an initialize request into `claude-director serve --stdio`.
2. Parse stdout's first line as JSON.
3. Assert protocolVersion, serverInfo.name, capabilities.tools.

## Pass criteria

- Response has `result.protocolVersion = "2024-11-05"`.
- Response has `result.serverInfo.name = "claude-director"`.
- Response advertises a `tools` capability.

## Script

```bash
set -euo pipefail
out=$(printf '%s
' '{"jsonrpc":"2.0","id":1,"method":"initialize"}' \
    | claude-director serve --stdio 2>/dev/null)
pv=$(printf '%s' "$out" | head -1 | jq -r '.result.protocolVersion')
sn=$(printf '%s' "$out" | head -1 | jq -r '.result.serverInfo.name')
tools=$(printf '%s' "$out" | head -1 | jq -e '.result.capabilities.tools' >/dev/null && echo ok || echo missing)
if [ "$pv" != "2024-11-05" ] || [ "$sn" != "claude-director" ] || [ "$tools" != "ok" ]; then
    echo "FAIL: pv=$pv sn=$sn tools=$tools"; printf '%s' "$out"; exit 1
fi
echo "mcp-1 ok: initialize handshake (pv=$pv server=$sn)"
```
