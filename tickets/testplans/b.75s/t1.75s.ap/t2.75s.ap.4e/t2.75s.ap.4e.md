# pkg-api-4-errnames-catalog-present

Pins E1 AC: `pkg/api/errnames/catalog.json` exists, parses as JSON, and
contains the documented err_name set — the SRD §13.1 surface that
`pkg/api/manifest/manifest_test.go` pins per verb. The catalog is the
generated, machine-readable mirror of the source-of-truth Go catalog
(`pkg/api/errnames/catalog.go` + `//go:generate go run ./pkg/api/errnames/`),
so a drift between the two would mean the published library ships a
catalog that omits errors the manifest advertises.

We assert:

1. The file exists and parses as a JSON array of objects with `name`
   string fields.
2. A representative set of expected err_names (drawn from the verb-by-verb
   pins in `pkg/api/manifest/manifest_test.go`) is present.

The representative set covers every verb-family that advertises errors
in the manifest tests: cwd-validation (spawn), relay/env/instance
(spawn), tmux transport (interact verbs), state preconditions
(send-keys, pause), list, and pause-specific.

## Setup

No DB state needed. Read-only inspection of
`/work/source/pkg/api/errnames/catalog.json`.

## Steps

1. Assert `jq` is in PATH; fail with a clear message if not.
2. Assert the catalog file exists at the expected path.
3. `jq '.'` over the file; assert exit 0 (file is valid JSON).
4. Extract the set of `name` fields via `jq -r '.[].name'`.
5. For each expected name in the documented set, assert presence; record
   misses.
6. Fail with the full miss list if any expected name is absent.

## Pass criteria

- `pkg/api/errnames/catalog.json` exists.
- Parses as valid JSON (top-level array).
- Contains every name in the expected documented set.
- Script exits 0; final line matches `pkg-api-4-errnames-catalog-present ok: ...`.

## Script

```bash
set -euo pipefail

if ! command -v jq >/dev/null 2>&1; then
    echo "FAIL: jq not found in PATH"
    exit 1
fi

CATALOG=/work/source/pkg/api/errnames/catalog.json

if [[ ! -f "$CATALOG" ]]; then
    echo "FAIL: catalog file does not exist: $CATALOG"
    exit 1
fi

if ! jq -e '.' "$CATALOG" >/dev/null 2>&1; then
    echo "FAIL: catalog file is not valid JSON: $CATALOG"
    jq '.' "$CATALOG" 2>&1 | head -20 >&2 || true
    exit 1
fi

# Top-level must be an array of objects each with a string "name" field.
if ! jq -e 'type == "array" and (length > 0) and (all(.[]; type == "object" and (.name | type == "string") and (.name | length > 0)))' "$CATALOG" >/dev/null 2>&1; then
    echo "FAIL: catalog must be a non-empty array of objects with non-empty string .name fields"
    exit 1
fi

# Documented err_name set — drawn from pkg/api/manifest/manifest_test.go
# verb-by-verb error pins (SRD §13.1 surface). Keep in sync with that
# file when new ErrorNames entries are added there.
EXPECTED=(
    ErrCwdMissing
    ErrCwdNotAPath
    ErrCwdNotFound
    ErrCwdNotADirectory
    ErrRelayModeInvalid
    ErrSpawnDeniedFlag
    ErrReservedEnvKey
    ErrInstanceIdCollision
    ErrTmuxNotAvailable
    ErrTmuxSessionCreate
    ErrTmuxSendKeys
    ErrTmuxCaptureFailed
    ErrSpawnNotFound
    ErrSpawnNotInteractive
    ErrSendKeysWhileRelayed
    ErrListInvalidLabel
    ErrPauseTimeout
    ErrSpawnNotPausable
)

mapfile -t HAVE < <(jq -r '.[].name' "$CATALOG")

# Build a presence set for O(1) lookups.
declare -A HAVE_SET=()
for n in "${HAVE[@]}"; do
    HAVE_SET[$n]=1
done

MISSES=()
for want in "${EXPECTED[@]}"; do
    if [[ -z "${HAVE_SET[$want]:-}" ]]; then
        MISSES+=("$want")
    fi
done

if [[ "${#MISSES[@]}" -ne 0 ]]; then
    echo "FAIL: ${#MISSES[@]} documented err_name(s) missing from catalog.json:"
    printf '  - %s\n' "${MISSES[@]}" >&2
    exit 1
fi

echo "pkg-api-4-errnames-catalog-present ok: catalog has ${#HAVE[@]} entries, all ${#EXPECTED[@]} documented err_names present"
```
