# release-pipeline-4-matrix-red-halt

Pins Epic 7 AC: the release halts in the build phase when the
cabi-matrix run on the release commit is red, with the message
`CI matrix is red on $COMMIT; fix before releasing`. No partial
release ships against a red matrix.

The case stubs `gh` so `gh run list` (unfiltered) returns a single
run with `conclusion=failure` while `gh run list --status=success`
returns an empty list. That is exactly the state that produces
the "matrix-red" branch in `collect_cabi_artifacts`.

## Setup

Writable copy of `/work/source`. `gh` is stubbed on PATH; no real
network calls.

## Pass criteria

- Exit code 3.
- Output contains `[build] CI matrix is red on` (the `$COMMIT`
  literal in the message is a shell escape, so the visible text
  is the literal `$COMMIT`).
- No `[tag]`, `[publish]`, or `[gh-release]` line.
- Final line matches `release-pipeline-4 ok: ...`.

## Script

```bash
set -euo pipefail

SRC=/work/source
WORK=$(mktemp -d)
OUT=$(mktemp)
STUB_DIR=$(mktemp -d)
trap 'rm -rf "$WORK" "$OUT" "$STUB_DIR"' EXIT

cp -a "$SRC/." "$WORK/"
cd "$WORK"
rm -rf .git
git init -q .
git config user.email "test@example.com"
git config user.name  "Test"
git checkout -q -b main
git add -A
git commit -q -m "release-pipeline test baseline"
git tag v0.0.1 HEAD

cat > "$STUB_DIR/gh" <<'GHEOF'
#!/usr/bin/env bash
case "$1" in
    run)
        case "$2" in
            list)
                # If --status=success appears anywhere in args, return [].
                # Otherwise return one run with conclusion=failure.
                want_success=0
                for a in "$@"; do
                    if [[ "$a" == "--status=success" ]]; then want_success=1; fi
                done
                if [[ "$want_success" -eq 1 ]]; then
                    printf '%s' '[]'
                else
                    printf '%s' '[{"databaseId":99,"headSha":"abc","conclusion":"failure","status":"completed"}]'
                fi
                exit 0
                ;;
        esac
        ;;
    repo) echo '{"nameWithOwner":"gabemahoney/agent-director"}'; exit 0 ;;
esac
GHEOF
chmod +x "$STUB_DIR/gh"

set +e
PATH="$STUB_DIR:$PATH" \
    VERSION=v0.99.0 \
    bash skills/release-agent-director/release.sh \
    --dry-run --branch main --no-build > "$OUT" 2>&1
RC=$?
set -e

if [[ "$RC" -ne 3 ]]; then
    echo "FAIL: expected exit code 3 (build-phase halt), got $RC"
    sed 's/^/    /' "$OUT"
    exit 1
fi

if ! grep -q '\[build\] CI matrix is red on' "$OUT"; then
    echo "FAIL: missing 'CI matrix is red on \$COMMIT' message"
    sed 's/^/    /' "$OUT"
    exit 1
fi

if grep -qE '^\[(tag|publish|gh-release)\]' "$OUT"; then
    echo "FAIL: phases past build ran despite red matrix"
    sed 's/^/    /' "$OUT"
    exit 1
fi

echo "release-pipeline-4 ok: build halted on red cabi-matrix run"
```
