# C-ABI shim (`cabi-shim`) testplan

t1 collector for the C-ABI shim (`pkg/cabi` — Epic 2 of `b.qe2`). Each t2 is a
plain-shell case the Docker harness executes. Runs under
`make test-docker EPIC=cabi-shim`; the driver resolves the slug by
case-insensitive substring match against the t1 frontmatter title (see
`test/driver/run-testplan.sh`), so the canonical substring `cabi-shim` must
remain in the title verbatim.

## Sourced from

- Source Epic: `t1.qe2.qx` (C-ABI shim acceptance criteria, all verbs).
- Testplan-adding Task: `t2.qe2.qx.3x`.

## Tests included

| t2 | what it pins |
|---|---|
| `t2.75s.nv.5x` — `cabi-1-build-and-symbols` | `nm -gD libagent_director.so` — every exported text symbol is `ad_`-prefixed; zero non-`ad_` T-symbols leak into the public ABI. |
| `t2.75s.nv.9w` — `cabi-2-ad-version-envelope` | `ad_version("{}")` returns a valid success envelope (`version`, `commit`); no `err_name` present. |

| `t2.75s.nv.tt` — `cabi-5-double-close-noop` | `ad_open` → handle; `ad_close(handle)` succeeds; `ad_close(handle)` again succeeds (no-op); `ad_close(unminted)` also succeeds. |

## Driver mode

All cases use `DRIVER_MODE=shell`. Each t2 body has a single fenced `bash`
block executed by the driver after the per-case DB-reset fixture.

Cases 1–5 build `libagent_director.so` from `/work/source` (the read-only
source bind-mount) into a writable temp directory. `CGO_ENABLED=1 go build
-buildmode=c-shared` requires a C compiler at build time. **Before running
this testplan, add `build-essential` and `binutils` to `test/Dockerfile`.**

Cases 2–5 use a tiny inline C probe compiled with `gcc` to call the `.so`
via `dlopen`/`dlsym` — the same pattern as the in-tree `dlopen_*_test.go`
suite. Case 1 copies the freshly built `.so` to `/tmp/ad_cabi_test.so`; cases
2–5 reuse that path (building fresh if the file is absent), so a full
five-case run pays the CGO build cost exactly once.

## Notes for the orchestrator

- Audit: `make test-docker EPIC=cabi-shim`; inspect the JSON stream
  case-by-case.
- The Dockerfile must include `build-essential` (gcc) and `binutils` (nm).
- None of these cases use `agent-director spawn` or tmux. The db-reset
  fixture's `cd-*` session sweep is a no-op for all five cases.
- Case 5 (`ad_open` → `ad_close`) creates a temp SQLite file at
  `/tmp/probe5_state.db` inside the container; it does not touch the
  db-reset-managed store.

## tmux session sweep

No `cd-*` sessions are created. The db-reset fixture's session sweep is a
no-op for these cases.
