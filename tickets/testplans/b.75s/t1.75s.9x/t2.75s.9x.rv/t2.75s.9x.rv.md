# interact-9: send-keys with keysym-text payload stays literal

Pins the bug fix for `b.btj`: a `text` payload containing a token that
exactly matches a tmux keysym (`Enter`, `Tab`, `BSpace`, …) must be sent
literally — `tmux send-keys -l <text>` — and the trailing submit must
remain a real Enter keystroke. Without `-l`, the first matching token
would fire as a real keysym event and submit the buffer prematurely (a
"Spawn-driving Claude composes a multi-token payload" scenario).

## Setup

Per-case DB-reset. A `cat` pane echoes whatever the verb types.

## Steps

1. Start `tmux new-session -d -s cd-i9 -- cat`.
2. Seed a `spawns` row at state `waiting` with `tmux_session_name=cd-i9`
   and `claude_instance_id=id-i9`.
3. `claude-director send-keys --claude-instance-id id-i9 --text 'Enter the password'`.
4. Read the pane; assert the literal line `Enter the password` appears
   intact (echoed back by cat).
5. Assert NO bare `the password` line appears on its own — a bare-rest
   line would prove the leading `Enter` fired as a keysym and submitted
   `Enter` alone, then `the password` was typed and echoed as a second
   line.

## Pass criteria

- send-keys exit 0.
- Pane contains the literal line `Enter the password`.
- Pane does NOT contain a bare `the password` line (i.e. with no
  preceding `Enter `) — the negative assertion that pins the bug.

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-i9 -- cat
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i9', 'waiting', '/tmp', 'cd-i9', 'off');"
claude-director send-keys --claude-instance-id id-i9 --text 'Enter the password' >/dev/null
sleep 0.2
pane="$(claude-director read-pane --claude-instance-id id-i9 --n-lines 100 | jq -r '.pane')"
echo "$pane" | grep -qE '^Enter the password$'
# Negative: a bare "the password" line on its own would mean the
# leading "Enter" fired as a keysym and the rest typed afterward.
n_bare="$(echo "$pane" | grep -cE '^the password$' || true)"
test "$n_bare" = "0"
echo "interact-9 ok: keysym-text payload stayed literal"
```
