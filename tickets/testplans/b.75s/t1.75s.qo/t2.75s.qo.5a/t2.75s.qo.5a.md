# spawn-7: cwd canonicalization

Maps to SRD §7.2 step 1: `filepath.EvalSymlinks` canonicalizes the
input cwd before INSERT. Two callers spawning into `/tmp` and `/tmp/./`
end up with the same `cwd` value in the DB.

## Setup

Per-case DB-reset has wiped the DB.

## Steps

1. spawn into `/tmp` → capture `id1`.
2. spawn into `/tmp/./` → capture `id2`.
3. `get` each → both rows should report the same canonical cwd.

## Pass criteria

- Two distinct claude_instance_ids.
- Both rows' `cwd` are byte-identical (the canonical form of `/tmp`).

## Script

```bash
set -euo pipefail
id1="$(agent-director spawn --cwd /tmp | jq -r '.claude_instance_id')"
id2="$(agent-director spawn --cwd '/tmp/./' | jq -r '.claude_instance_id')"
test "$id1" != "$id2"
cwd1="$(agent-director get --claude-instance-id "$id1" | jq -r '.cwd')"
cwd2="$(agent-director get --claude-instance-id "$id2" | jq -r '.cwd')"
test "$cwd1" = "$cwd2"
canonical="$(readlink -f /tmp)"
test "$cwd1" = "$canonical"
echo "spawn-7 ok: both rows have cwd=$cwd1"
```
