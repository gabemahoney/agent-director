# release-4: host-arch binary runs `help` cleanly

The harness container is linux/amd64; the freshly-built
agent-director-linux-amd64 should execute and return a non-empty
verbs list when asked for `help`.

## Setup

Per-case DB-reset.

## Steps

1. Build via copy-source + make release-binaries.
2. Exec dist/agent-director-linux-amd64 help into jq.
3. Assert the result has a non-empty `.verbs` array.

## Pass criteria

- The binary exits 0.
- jq reads `.verbs | length > 0` truthily.

## Script

```bash
set -euo pipefail
work=$(mktemp -d)
cp -a /work/source/* "$work/"
cd "$work"
make release-binaries >/dev/null 2>&1

# Set HOME to a temp dir so the help warmup doesn't trample our
# real $HOME's state.db.
HOME=$(mktemp -d) ./dist/agent-director-linux-amd64 help \
    | jq -e '.verbs | length > 0' >/dev/null

n=$(HOME=$(mktemp -d) ./dist/agent-director-linux-amd64 help | jq '.verbs | length')
echo "release-4 ok: linux-amd64 help returned $n verbs"
```
