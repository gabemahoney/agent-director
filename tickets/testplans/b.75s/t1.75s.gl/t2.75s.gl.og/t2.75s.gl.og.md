# go-client-2-smoke-all-callable-verbs

Pins Epic 4 AC: `go test ./test/smoke/go/...` is green and
`TestSmokeAllVerbs` reports a `--- PASS:` line for every entry returned by
`manifest.CallableVerbs()` (currently 15 verbs — see
`pkg/api/manifest/manifest_test.go`). The smoke test is the consumer-grade
end-to-end check that every callable verb is reachable through the
`pkg/api` surface and returns a result that matches its manifest shape.

## Setup

No DB state needed in the host store — the smoke test builds its own
per-subtest temp stores via `internal/testsupport/storefix`. `HOME` is
redirected by the smoke `TestMain` so this case does not need a pre-seeded
`~/.agent-director/` directory. The script copies `/work/source` to a
writable temp dir because `go test` writes to its module cache and may
write coverage / test-binary artifacts.

## Steps

1. Assert `go` is available in PATH; fail with a clear message if not.
2. Create a writable workspace under `/tmp/` and `cp -a /work/source/.` into it.
3. `cd <workspace> && go test -v ./test/smoke/go/...` and capture both
   exit code and stdout.
4. Assert exit code is 0.
5. Count `=== RUN   TestSmokeAllVerbs/` lines in the captured output;
   assert the count is ≥ 15 (one per callable verb in the manifest).
6. Assert every matching `--- PASS: TestSmokeAllVerbs/` line is present
   and there are no `--- FAIL:` lines from any TestSmokeAllVerbs subtest.

## Pass criteria

- `go test -v ./test/smoke/go/...` exits 0.
- Output contains `--- PASS: TestSmokeAllVerbs` (the parent test passed).
- At least 15 `TestSmokeAllVerbs/<verb>` subtests ran.
- No `--- FAIL:` lines in the captured output.
- Script exits 0; final line matches `go-client-2-smoke-all-callable-verbs ok: ...`.

## Script

```bash
set -euo pipefail

if ! command -v go >/dev/null 2>&1; then
    echo "FAIL: go not found in PATH"
    exit 1
fi

WORK=$(mktemp -d /tmp/goclient2_XXXXXX)
cp -a /work/source/. "$WORK/"

cd "$WORK"

OUT_FILE=$(mktemp /tmp/goclient2_out.XXXXXX)
EXIT=0
go test -v ./test/smoke/go/... > "$OUT_FILE" 2>&1 || EXIT=$?

if [[ "$EXIT" -ne 0 ]]; then
    echo "FAIL: go test ./test/smoke/go/... exited with code $EXIT"
    tail -200 "$OUT_FILE" >&2
    exit 1
fi

# Count subtests. Each callable verb gets its own t.Run, emitting a
# `=== RUN   TestSmokeAllVerbs/<verb>` line.
SUBTEST_COUNT=$(grep -cE '^=== RUN   TestSmokeAllVerbs/' "$OUT_FILE" || true)
if [[ "$SUBTEST_COUNT" -lt 15 ]]; then
    echo "FAIL: expected >=15 TestSmokeAllVerbs subtests (one per CallableVerbs() entry); got $SUBTEST_COUNT"
    grep -E '^=== RUN   TestSmokeAllVerbs/' "$OUT_FILE" >&2 || true
    exit 1
fi

# Defensive: surface any FAIL lines even though a non-zero exit would have
# tripped earlier. This catches the (unlikely) case where t.Errorf was
# called without failing the parent test, which shouldn't happen but is
# cheap to assert.
if grep -E '^--- FAIL: TestSmokeAllVerbs' "$OUT_FILE" >/dev/null; then
    echo "FAIL: at least one TestSmokeAllVerbs subtest reported --- FAIL:"
    grep -E '^--- FAIL: TestSmokeAllVerbs' "$OUT_FILE" >&2
    exit 1
fi

# Confirm the parent test passed.
if ! grep -E '^--- PASS: TestSmokeAllVerbs ' "$OUT_FILE" >/dev/null; then
    echo "FAIL: did not observe --- PASS: TestSmokeAllVerbs in output"
    tail -50 "$OUT_FILE" >&2
    exit 1
fi

echo "go-client-2-smoke-all-callable-verbs ok: TestSmokeAllVerbs passed with $SUBTEST_COUNT verb subtests"
```
