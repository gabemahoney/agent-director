# Epic 8 — `epic-08-recovery` testplan

t1 collector for Epic 8 (`t1.8op.1w` — Crash recovery & DB hygiene:
find-missing + expire + delete). Each t2 is a plain-English case the
Docker harness executes. The Epic 8 Progression Contract requires
every t2 to pass under `make test-docker EPIC=epic-08-recovery`.

## Sourced from

- Epic ticket: `t1.8op.1w` (Acceptance Criteria #1–#4).
- SRD §4.4 (probe layer), §5.2 (state-table), §12 (verb shapes),
  §14.6 (degraded-mode guard).

## Cases

| case | what it pins |
|---|---|
| `recov-1-find-missing-reconciles-killed` | spawn a Claude; `tmux kill-session` externally; `find-missing` reports count=1; the killed row's state is now `missing` with `ended_at` populated. |
| `recov-2-find-missing-degraded-guard` | seed live rows directly + ensure the prober sees ZERO Claude processes → `find-missing` returns `count=0`, marks NOTHING missing, logs a warning. |
| `recov-3-find-missing-noop-when-all-alive` | spawn 2 Claudes; `find-missing` returns count=0, no state changes. |
| `recov-4-expire-zero-reaps-all-terminal` | seed terminal-state rows + live row → `expire --older-than 0d` removes only the terminal rows, returns their IDs. |
| `recov-5-expire-default-window-spares-young` | seed a terminal row with a fresh `ended_at` → `expire` (default 31d) returns count=0; the row survives. |
| `recov-6-delete-per-row-result-map` | seed two rows + one bogus id → `delete --claude-instance-id A --claude-instance-id A2 --claude-instance-id bogus` returns `{A:ok, A2:ok, bogus:ErrSpawnNotFound}`; A and A2 are gone. |
| `recov-7-delete-bypasses-state-guards` | seed live-state row → `delete --claude-instance-id <id>` returns `{<id>:ok}` and the row is gone; no tmux interaction asserted. |

## Driver mode

All cases are written for `DRIVER_MODE=shell` — the orchestrator runs
the harness with `make test-docker EPIC=epic-08-recovery`. Each t2
body has a single fenced ` ```bash ` block the driver executes after
the per-case DB-reset fixture.

The find-missing cases (1, 2, 3) exercise the real prober's Linux
`/proc/<pid>/environ` walk. Case 1 uses the full spawn pipeline so a
real tmux session + claude process exist; the test then kills the
session via `tmux kill-session` (the operator's externally-driven
kill path, NOT `agent-director kill`) and runs `find-missing`.
Case 2 seeds live rows directly via `sqlite3` so no matching process
exists at all — the degraded-mode guard MUST trip. Case 3 spawns
two Claudes and confirms `find-missing` doesn't false-positive when
the probe set covers every DB id.

The expire and delete cases (4, 5, 6, 7) seed rows directly via
`sqlite3` for speed and isolation; the verbs are pure DB cleanup and
do not require any tmux interaction.

## Notes for the orchestrator

- Audit standard (SRD §17): run `make test-docker EPIC=epic-08-recovery`
  yourself, inspect the JSON stream case-by-case.
- The degraded-mode case (2) deliberately ensures no `AGENT_DIRECTOR_INSTANCE_ID`
  env var exists anywhere by NOT spawning anything and seeding rows
  via raw INSERT. This is the only valid way to reproduce the
  "wrong cron user" scenario inside the harness, where the harness
  user owns every process.
- The `tmux kill-session` in case 1 is the EXTERNAL kill path (the
  operator typing it manually), distinct from `agent-director kill`
  which the kill_cli tests already cover. find-missing's job is to
  reconcile both paths the same way.

## tmux session sweep

Cases 1 and 3 launch real `agent-director spawn`. The
`cd-*` prefix matches the DB-reset fixture's sweep so cases don't
leak sessions to each other.
