# tmpl-8: malformed template (unknown top-level key) surfaces ErrTemplateMalformed

Per SRD §10.2: LoadTemplate rejects unknown top-level keys at parse
time. Hand-editing a template to add a stray field exercises the
strict-decode path and the error surfaces on `spawn --template`.

## Setup

Per-case DB-reset. Hand-write a TOML file directly into the
templates dir.

## Steps

1. Write `~/.agent-director/templates/bad.toml` with an unknown key
   like `mystery_field = "wat"`.
2. `spawn --template bad --cwd /tmp` → exit non-zero;
   `err_name=ErrTemplateMalformed`.

## Pass criteria

- spawn exits non-zero with `err_name=ErrTemplateMalformed`.
- No spawn row is created (the error happens before INSERT).

## Script

```bash
set -euo pipefail
mkdir -p "$HOME/.agent-director/templates"
cat > "$HOME/.agent-director/templates/bad.toml" <<'TOML'
cwd = "/tmp"
mystery_field = "wat"
TOML

stderr_file=$(mktemp)
if agent-director spawn --template bad --cwd /tmp 2>"$stderr_file" >/dev/null; then
    echo "FAIL: spawn with malformed template exited 0"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrTemplateMalformed" ]; then
    echo "FAIL: err_name=$err_name; want ErrTemplateMalformed"; exit 1
fi
n=$(sqlite3 "$HOME/.agent-director/state.db" "SELECT COUNT(*) FROM spawns;")
if [ "$n" != "0" ]; then
    echo "FAIL: $n row(s) inserted despite ErrTemplateMalformed"; exit 1
fi
echo "tmpl-8 ok: unknown key → ErrTemplateMalformed; no row inserted"
```
