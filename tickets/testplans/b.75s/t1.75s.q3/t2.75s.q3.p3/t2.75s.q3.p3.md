# release-7: dry-run produces release notes grouped by Epic ID

The release-notes templater walks `git log <prev-tag>..HEAD` and
groups entries by Epic ID. This case synthesizes a fresh repo with
multi-Epic commit messages and confirms the groups appear in the
templated notes.

## Setup

Per-case DB-reset.

## Steps

1. Init a clean repo on a `main` branch.
2. Make 4 commits with messages referencing Epic 1.t2.foo,
   Epic 5.t2.bar, Epic 5.t2.baz, and an unreferenced message.
3. Run release.sh v0.2.0 --dry-run.
4. Assert dist/release-notes.md contains the expected `### Epic 1.t2`,
   `### Epic 5.t2`, and `### Other` group headers.

## Pass criteria

- release.sh --dry-run exits 0.
- The generated notes file contains all expected group headers.

## Script

```bash
set -euo pipefail
work=$(mktemp -d)
cd "$work"
git init -q -b main
git config user.email test@example.com
git config user.name test

commit() {
    echo "$1" > "f$(date +%N).txt"
    git add .
    git -c commit.gpgsign=false commit -q -m "$1"
}
commit "Add the foo verb (Epic 1.t2.foo)"
commit "Activate the bar guard (Epic 5.t2.bar)"
commit "Fix the baz polling loop (Epic 5.t2.baz)"
commit "Tidy README"  # Unreferenced → "Other" group

/opt/skills/release-claude-director/release.sh v0.2.0 --dry-run >/dev/null

notes="dist/release-notes.md"
if [ ! -f "$notes" ]; then
    echo "FAIL: $notes missing"; exit 1
fi
for header in '### Epic 1.t2' '### Epic 5.t2' '### Other'; do
    if ! grep -qF "$header" "$notes"; then
        echo "FAIL: notes missing $header"
        cat "$notes"
        exit 1
    fi
done
echo "release-7 ok: notes grouped by Epic (incl. Other for unreferenced)"
```
