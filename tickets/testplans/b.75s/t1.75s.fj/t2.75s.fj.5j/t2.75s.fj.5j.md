# discover-4: multiple --label flags AND together

Per SRD §12: filters AND. Two `--label` flags require both labels
present and matching on the row. The case proves the AND, not OR:
only the row carrying both labels is returned.

## Setup

Per-case DB-reset. Two rows seeded directly; first has both labels,
second only the first label.

## Steps

1. INSERT row-A with `{"project":"foo","env":"dev"}`.
2. INSERT row-B with `{"project":"foo"}`.
3. `agent-director list --label project=foo --label env=dev` →
   expect only row-A.

## Pass criteria

- list exits 0.
- Result contains exactly `id-d4-both`.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode, labels)
   VALUES ('id-d4-both', 'waiting', '/tmp', 'cd-d4-a', 'off', '{\"project\":\"foo\",\"env\":\"dev\"}'),
          ('id-d4-one',  'waiting', '/tmp', 'cd-d4-b', 'off', '{\"project\":\"foo\"}');"
ids=$(agent-director list --label project=foo --label env=dev | jq -r '.spawns[].claude_instance_id' | sort | tr '
' ' ')
expected="id-d4-both "
if [ "$ids" != "$expected" ]; then
    echo "FAIL: ids='$ids'; want='$expected'"; exit 1
fi
echo "discover-4 ok: multiple --label AND together"
```
