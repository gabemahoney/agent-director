# ts-bun-1-install-resolves-only-host-platform

Pins Epic 5 AC: `bun install` inside `pkg/ts-bun-client/` resolves the
`linux-x64` sub-package correctly on the linux container. The
`node_modules/@agent-director/linux-x64/` directory must exist and
contain `libagent_director.so`. The `darwin-arm64` sub-package may be
present as a directory (bun installs all `file:` optional deps regardless
of os/cpu) but must NOT have a `.dylib` binary — no cross-platform
native binary lands on the linux container.

v1 platform set is `{linux-x64, darwin-arm64}`; `darwin-x64` was dropped
on 2026-05-24 and the `platforms/darwin-x64/` directory no longer exists
in the worktree. This case actively asserts that the `darwin-x64`
sub-package is **absent from `optionalDependencies`** so the drop cannot
regress.

This mirrors the assertions in the in-tree
`test/install-resolution.test.ts` (T11).

## Setup

No DB state needed. The script copies `/work/source` to a writable temp dir,
runs `bun install --no-progress` from `pkg/ts-bun-client/`, and inspects
`node_modules/@agent-director/`.

## Steps

1. Assert `bun` is available in PATH; fail with a clear message if not.
2. Create a writable workspace under `/tmp/` and `cp -a /work/source/.` into it.
3. `cd <workspace>/pkg/ts-bun-client && bun install --no-progress`.
4. Assert `node_modules/@agent-director/linux-x64` directory exists.
5. Assert `node_modules/@agent-director/linux-x64/libagent_director.so` exists.
6. Assert `node_modules/@agent-director/darwin-arm64/libagent_director.dylib` does NOT exist.
7. Assert `node_modules/@agent-director/darwin-x64/` does NOT exist (dropped 2026-05-24).
8. Assert the umbrella `package.json` does NOT declare `@agent-director/darwin-x64` in `optionalDependencies`.

## Pass criteria

- `bun install` exits 0.
- `linux-x64` sub-package directory and `.so` binary are present.
- `darwin-arm64` sub-package directory may exist but `.dylib` binary is absent.
- `darwin-x64` is absent from both `node_modules` and `optionalDependencies`.
- Script exits 0; final line contains `ts-bun-1-install-resolves-only-host-platform ok:`.

## Script

```bash
set -euo pipefail

if ! command -v bun >/dev/null 2>&1; then
    echo "FAIL: bun not found in PATH — check test/Dockerfile BUN_VERSION ARG"
    exit 1
fi

export HOME=/tmp/bun-home-tsbun1
mkdir -p "$HOME"

WORK=$(mktemp -d /tmp/tsbun1_XXXXXX)
cp -a /work/source/. "$WORK/"

cd "$WORK/pkg/ts-bun-client"

# Wipe any node_modules carried in by `cp -a` so this case exercises a
# fresh `bun install` resolution against the current package.json, not
# whatever was lying around in the host worktree. Without this, a stale
# `@agent-director/darwin-x64` left over from a prior dev `bun install`
# could survive into the test even though the optionalDependencies
# entry is gone.
rm -rf node_modules

INSTALL_OUT=$(mktemp /tmp/tsbun1_install.XXXXXX)
INSTALL_EXIT=0
bun install --no-progress > "$INSTALL_OUT" 2>&1 || INSTALL_EXIT=$?
if [[ "$INSTALL_EXIT" -ne 0 ]]; then
    echo "FAIL: bun install exited with code $INSTALL_EXIT"
    cat "$INSTALL_OUT" >&2
    exit 1
fi

NM_BASE="$WORK/pkg/ts-bun-client/node_modules/@agent-director"

# linux-x64 directory must exist
if [[ ! -d "$NM_BASE/linux-x64" ]]; then
    echo "FAIL: node_modules/@agent-director/linux-x64 does not exist after bun install"
    exit 1
fi

# linux-x64 .so binary must exist
LINUX_SO="$NM_BASE/linux-x64/libagent_director.so"
if [[ ! -f "$LINUX_SO" ]]; then
    echo "FAIL: libagent_director.so not found inside @agent-director/linux-x64 (run prepare-platforms first?)"
    exit 1
fi

# darwin-arm64 .dylib must NOT exist on linux host (v1 platform set
# is {linux-x64, darwin-arm64}; the bun install resolved both file:
# optional deps but only the host-matching binary should be present).
DARWIN_ARM64_DYLIB="$NM_BASE/darwin-arm64/libagent_director.dylib"
if [[ -f "$DARWIN_ARM64_DYLIB" ]]; then
    echo "FAIL: darwin-arm64 libagent_director.dylib should not be present on linux host"
    exit 1
fi

# darwin-x64 was dropped from v1 on 2026-05-24 — the sub-package
# directory and the optionalDependencies entry must both be gone.
if [[ -e "$NM_BASE/darwin-x64" ]]; then
    echo "FAIL: @agent-director/darwin-x64 should not be installed (dropped from v1 on 2026-05-24)"
    exit 1
fi
if grep -q '"@agent-director/darwin-x64"' "$WORK/pkg/ts-bun-client/package.json"; then
    echo "FAIL: @agent-director/darwin-x64 must not appear in optionalDependencies (dropped from v1 on 2026-05-24)"
    exit 1
fi

SO_SIZE=$(stat -c '%s' "$LINUX_SO")
echo "ts-bun-1-install-resolves-only-host-platform ok: linux-x64 .so present (${SO_SIZE} bytes), darwin-arm64 dylib absent, darwin-x64 fully removed"
```
