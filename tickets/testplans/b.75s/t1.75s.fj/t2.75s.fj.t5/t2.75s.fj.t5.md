# discover-3: --label filter matches exact key=value via json_extract

Per SRD §4.2: label match is exact JSON-value via `json_extract`.
This case seeds 3 rows with different `project` label values and
filters for `project=foo`; only the matching row is returned.

## Setup

Per-case DB-reset. Seed 3 rows via sqlite3, inserting the labels JSON
as the literal column value.

## Steps

1. INSERT 3 rows with labels `project=foo`, `project=bar`, no label.
2. `claude-director list --label project=foo` → expect just `id-d3-foo`.

## Pass criteria

- list exits 0.
- Result contains exactly `id-d3-foo`.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode, labels)
   VALUES ('id-d3-foo', 'waiting', '/tmp', 'cd-d3-f', 'off', '{\"project\":\"foo\"}'),
          ('id-d3-bar', 'waiting', '/tmp', 'cd-d3-b', 'off', '{\"project\":\"bar\"}'),
          ('id-d3-no',  'waiting', '/tmp', 'cd-d3-n', 'off', '{}');"
ids=$(claude-director list --label project=foo | jq -r '.spawns[].claude_instance_id' | sort | tr '
' ' ')
expected="id-d3-foo "
if [ "$ids" != "$expected" ]; then
    echo "FAIL: ids='$ids'; want='$expected'"; exit 1
fi
echo "discover-3 ok: --label project=foo returns only matching row"
```
