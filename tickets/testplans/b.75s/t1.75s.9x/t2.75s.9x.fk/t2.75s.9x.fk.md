# interact-4: \r bytes stripped pre-send

Per SRD §4.3 + `reference/send-keys-research.md` "CR caveat": a literal
CR (0x0D) byte in the `text` payload would submit the buffer at the
position the CR appears — splitting one logical message into multiple
submissions. The verb strips CR bytes before invoking tmux.

## Setup

Per-case DB-reset. A `cat` pane.

## Steps

1. Start `tmux new-session -d -s cd-i4 -- cat`.
2. Seed `spawns` row at state `waiting`.
3. `agent-director send-keys --claude-instance-id id-i4 --text $'ab\rcd'`
   (literal CR between `ab` and `cd`).
4. Read pane; assert the composed line `abcd` appears (terminal input
   echo and/or `cat` stdout) AND no orphan line `ab` ever appears —
   the latter would prove CR submitted the partial buffer before the
   `cd` bytes were typed.

## Pass criteria

- send-keys exit 0.
- Pane contains at least one occurrence of the echoed line `abcd`
  (cat-on-a-tty produces both an input-echo line and a stdout-echo
  line; the test does not pin the exact count).
- Zero bare `ab` lines on their own (which would indicate CR triggered
  a mid-payload submit).

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-i4 -- cat
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i4', 'waiting', '/tmp', 'cd-i4', 'off');"
agent-director send-keys --claude-instance-id id-i4 --text $'ab\rcd' >/dev/null
sleep 0.2
pane="$(agent-director read-pane --claude-instance-id id-i4 | jq -r '.pane')"
n_abcd=$(echo "$pane" | grep -cE '^abcd$' || true)
test "$n_abcd" -ge 1
# Make sure there is no orphan "ab" line on its own (would mean CR
# submitted prematurely).
n_ab=$(echo "$pane" | grep -cE '^ab$' || true)
test "$n_ab" = "0"
echo "interact-4 ok: CR stripped; composed 'abcd' visible ($n_abcd time(s)); no orphan 'ab'"
```
