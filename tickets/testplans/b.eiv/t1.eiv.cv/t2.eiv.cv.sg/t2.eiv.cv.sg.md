# subprocess-coverage-1-signal

Per SRD §SR-5.2 / §SR-5.4 / §SR-3.3: when the in-flight CLI subprocess
is killed by an OS signal, the call must reject with `ErrConsumerSignal`
carrying the signal name, and the Client's serialization queue must
remain operable — a subsequent call must succeed.

The installed-tarball Client is exercised against a stub CLI binary
that, in "signal" mode, sends SIGTERM to itself; the bash interpreter
running the stub has no SIGTERM trap and dies from the default signal
action. Bun's `subprocess.signalCode` surfaces "SIGTERM", and the
Client maps that to `ErrConsumerSignal`.

## Setup

- `${HOME}=$(mktemp -d)`.
- `bun install file://...tgz` into `${HOME}/proj`.
- Stage a stub CLI binary into
  `${HOME}/proj/node_modules/@agent-director/linux-x64/bin/agent-director`.
  The stub branches on a mode-file at `/tmp/sub-cov-sg-mode`:
  - `signal` → `kill -TERM $$` (the stub bash process dies from SIGTERM).
  - any other value → emit a valid `version` JSON envelope on stdout.

## Steps

1. Write `signal` into the mode-file.
2. `client.version({})` — expect rejection with `ErrConsumerSignal`,
   with `.signal === "SIGTERM"`.
3. Write `ok` into the mode-file.
4. `client.version({})` — expect a normal `{version, commit}` envelope.

## Pass criteria

- Step 2's promise rejects with `e instanceof ErrConsumerSignal`.
- Step 4's promise resolves with a `string` `version` field.
- Driver exits 0.

## Script

```bash
set -euo pipefail
HOME_DIR="$HOME"
PROJ="$HOME_DIR/proj"
rm -rf "$PROJ"
mkdir -p "$PROJ"
cd "$PROJ"

. /work/source/tickets/testplans/b.eiv/t1.19d.6c/build-tarball.sh

cat > "$PROJ/package.json" <<'EOF'
{
  "name": "subprocess-coverage-sg-consumer",
  "version": "0.0.0",
  "type": "module"
}
EOF

bun install "file://${TARBALL}" >/dev/null

DEST="$PROJ/node_modules/@agent-director/linux-x64"
mkdir -p "$DEST/bin"
cp "$PLATFORM_PKG_DIR/package.json" "$DEST/package.json"

# Stub CLI binary. Reads /tmp/sub-cov-sg-mode and either self-kills with
# SIGTERM or emits a valid version envelope.
cat > "$DEST/bin/agent-director" <<'STUB_EOF'
#!/usr/bin/env bash
mode="$(cat /tmp/sub-cov-sg-mode 2>/dev/null || echo ok)"
if [ "$mode" = "signal" ]; then
    # No SIGTERM trap → bash terminates from the default signal action,
    # and the wait status carries WIFSIGNALED with SIGTERM. Bun reports
    # this as subprocess.signalCode === "SIGTERM".
    kill -TERM $$
    # Block long enough for the signal handler to fire and tear down bash
    # before this branch can fall through.
    sleep 30
fi
printf '{"version":"0.0.0-stub","commit":"stub-commit"}\n'
STUB_EOF
chmod +x "$DEST/bin/agent-director"

cp /work/source/tickets/testplans/b.eiv/t1.eiv.cv/t2.eiv.cv.sg/driver.mjs "$PROJ/driver.mjs"

out=$(bun run "$PROJ/driver.mjs" 2>&1)
echo "$out"
grep -q "OK signal-kill" <<<"$out" || { echo "FAIL: ErrConsumerSignal assertion missing"; exit 1; }
grep -q "OK follow-up"   <<<"$out" || { echo "FAIL: queue-not-wedged assertion missing"; exit 1; }
echo "subprocess-coverage-1-signal ok"
```
