# release-pipeline-2-h3-halt-in-live-mode

Pins Epic 7 AC: live (`--release`) run halts cleanly when H3 is
unresolved, before any `npm publish` runs, with a clear error
message naming H3 and listing every offending `package.json` file.

The placeholder name `@CHANGEME-H3/agent-director*` is still
present in the committed sources (H3 is operator-gated), so the
live-mode behavior is testable today with no setup.

## Setup

Same writable copy of `/work/source` as case 1. Tests halt path so
no `npm publish` actually runs (the script exits before that
step).

## Pass criteria

- Exit code 6 (publish-phase failure).
- Output contains `[publish] H3 unresolved: claim npm name first`.
- Output names all four placeholder package.jsons.
- Output does NOT contain `+ @CHANGEME-H3/agent-director` (the
  marker `npm publish` would emit if it had actually run).
- Final line matches `release-pipeline-2 ok: ...`.

## Script

```bash
set -euo pipefail

SRC=/work/source
WORK=$(mktemp -d)
ORIGIN=$(mktemp -d)
OUT=$(mktemp)
trap 'rm -rf "$WORK" "$ORIGIN" "$OUT"' EXIT
cp -a "$SRC/." "$WORK/"
cd "$WORK"
rm -rf .git
git init -q .
git config user.email "test@example.com"
git config user.name  "Test"
git checkout -q -b main
git add -A
git commit -q -m "release-pipeline test baseline"
git tag v0.0.1 HEAD

# Live mode needs a real `origin` to push the tag to: stand up a
# local bare repo and wire it in. (The tag phase runs BEFORE the
# publish phase; without a working origin the script would fail
# at exit 7 in tag-phase, never reaching the H3 publish halt
# we're trying to pin.)
git init -q --bare "$ORIGIN"
git remote add origin "$ORIGIN"

# Live mode requires NPM_TOKEN; supply a dummy so we exercise the
# H3 halt rather than the missing-token preflight halt.
set +e
NPM_TOKEN=dummy VERSION=v0.99.0 AD_RELEASE_SKIP_CABI=1 \
    bash skills/release-agent-director/release.sh \
    --release --branch main --no-build > "$OUT" 2>&1
RC=$?
set -e

if [[ "$RC" -ne 6 ]]; then
    echo "FAIL: expected exit code 6 (publish-phase H3 halt), got $RC"
    sed 's/^/    /' "$OUT"
    exit 1
fi

if ! grep -q '\[publish\] H3 unresolved: claim npm name first' "$OUT"; then
    echo "FAIL: missing '[publish] H3 unresolved' message"
    sed 's/^/    /' "$OUT"
    exit 1
fi

# All four package.jsons must be named in the error output.
for f in \
    pkg/ts-bun-client/package.json \
    pkg/ts-bun-client/platforms/linux-x64/package.json \
    pkg/ts-bun-client/platforms/darwin-x64/package.json \
    pkg/ts-bun-client/platforms/darwin-arm64/package.json; do
    if ! grep -qF "$f" "$OUT"; then
        echo "FAIL: H3 halt did not name $f"
        sed 's/^/    /' "$OUT"
        exit 1
    fi
done

# Belt-and-suspenders: no actual `npm publish` line.
if grep -q '^\+ @CHANGEME-H3/agent-director' "$OUT"; then
    echo "FAIL: npm publish ran despite H3 placeholder"
    sed 's/^/    /' "$OUT"
    exit 1
fi

echo "release-pipeline-2 ok: live run halted on H3 before any npm publish"
```
