# mcp-5: tools/call error response carries typed err_name

Per SRD §13.1: failure path surfaces a typed err_name. tools/call
on a verb that errors → MCP error response with
`error.data.err_name` set to the canonical name.

## Setup

Per-case DB-reset.

## Steps

1. (No row seeded.)
2. tools/call name=status with unknown id.
3. Assert error.code is -32000 and error.data.err_name is
   `ErrSpawnNotFound`.

## Pass criteria

- Response has an `error` object (no `result`).
- `error.data.err_name = "ErrSpawnNotFound"`.

## Script

```bash
set -euo pipefail
out=$(printf '%s
%s
' \
    '{"jsonrpc":"2.0","id":1,"method":"initialize"}' \
    '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"status","arguments":{"claude_instance_id":"absent"}}}' \
    | agent-director serve --stdio 2>/dev/null)

second=$(printf '%s' "$out" | sed -n '2p')
err_code=$(printf '%s' "$second" | jq -r '.error.code')
err_name=$(printf '%s' "$second" | jq -r '.error.data.err_name')

if [ "$err_code" != "-32000" ] || [ "$err_name" != "ErrSpawnNotFound" ]; then
    echo "FAIL: error code=$err_code name=$err_name (want -32000 / ErrSpawnNotFound)"
    printf '%s' "$second"; exit 1
fi
echo "mcp-5 ok: error envelope carries err_name=ErrSpawnNotFound"
```
