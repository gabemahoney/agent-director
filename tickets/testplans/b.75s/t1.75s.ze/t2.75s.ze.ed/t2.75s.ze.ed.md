# tmpl-4: per-call --label merges with template labels

Maps to Epic 7 AC #3. Template's `project=foo` survives; per-call
adds `other=bar`. Both end up on the row.

## Setup

Per-case DB-reset.

## Steps

1. `make-template --name tmpl4 --cwd /tmp --label project=foo`.
2. `spawn --template tmpl4 --label other=bar`.
3. `get` → row labels contain BOTH `project=foo` and `other=bar`.

## Pass criteria

- Row labels include both keys with the expected values.

## Script

```bash
set -euo pipefail
claude-director make-template --name tmpl4 --cwd /tmp --label project=foo >/dev/null
id=$(claude-director spawn --template tmpl4 --label other=bar | jq -r '.claude_instance_id')
test -n "$id"
row=$(claude-director get --claude-instance-id "$id")
p=$(printf '%s' "$row" | jq -r '.labels.project // empty')
o=$(printf '%s' "$row" | jq -r '.labels.other // empty')
if [ "$p" != "foo" ] || [ "$o" != "bar" ]; then
    echo "FAIL: labels = $(printf '%s' "$row" | jq -c '.labels'); want project=foo + other=bar"; exit 1
fi
echo "tmpl-4 ok: labels merged from both sources"
```
