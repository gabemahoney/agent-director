# kill-1: kill on a live row removes the tmux session

Per Epic 5 AC #3 (`Spawn → kill → tmux session gone`). The verb
invokes `tmux kill-session` against the row's `tmux_session_name`;
real tmux removes the session; the verb returns 0.

## Setup

Per-case DB-reset. A `cat` pane under the `cd-k1` session.

## Steps

1. Start `tmux new-session -d -s cd-k1 -- cat`.
2. Seed `spawns` row at state `waiting`, session_name `cd-k1`.
3. `agent-director kill --claude-instance-id id-k1`.
4. Confirm `tmux has-session -t cd-k1` exits non-zero (session is gone).

## Pass criteria

- kill exits 0.
- `tmux has-session -t cd-k1` returns non-zero after the kill.

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-k1 -- cat
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-k1', 'waiting', '/tmp', 'cd-k1', 'off');"
agent-director kill --claude-instance-id id-k1 >/dev/null
if tmux has-session -t cd-k1 2>/dev/null; then
    echo "FAIL: tmux session cd-k1 still exists after kill"; exit 1
fi
echo "kill-1 ok: live row → kill → session gone"
```
