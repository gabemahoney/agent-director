# discover-2: --state filter ORs within the slice, ANDs with other filters

Per SRD §12: `--state waiting,working` returns rows in either state.
The `ended` row is excluded — proves the comma-list is a closed set,
not a wildcard.

## Setup

Per-case DB-reset. Seed 3 rows: one waiting, one working, one ended.

## Steps

1. INSERT the 3 rows.
2. `agent-director list --state waiting,working` → expect the two
   live rows; not the ended one.

## Pass criteria

- list exits 0.
- Result contains exactly `id-d2-wait` and `id-d2-work`.
- Result does NOT contain `id-d2-end`.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-d2-wait', 'waiting', '/tmp', 'cd-d2-w', 'off'),
          ('id-d2-work', 'working', '/tmp', 'cd-d2-r', 'off'),
          ('id-d2-end',  'ended',   '/tmp', 'cd-d2-e', 'off');"
ids=$(agent-director list --state waiting,working | jq -r '.spawns[].claude_instance_id' | sort | tr '
' ' ')
expected="id-d2-wait id-d2-work "
if [ "$ids" != "$expected" ]; then
    echo "FAIL: ids='$ids'; want='$expected'"; exit 1
fi
echo "discover-2 ok: --state filter excludes ended"
```
