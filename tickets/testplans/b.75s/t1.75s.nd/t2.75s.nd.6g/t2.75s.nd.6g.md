# mcp-6: 30 sequential calls in one server lifetime

Per SRD §3.3: the MCP server is long-lived. This case streams 30
sequential help calls in one server process and verifies every
response is well-formed.

## Setup

Per-case DB-reset.

## Steps

1. Generate 30 JSON-RPC tools/call (name=help) requests with ids 1..30.
2. Pipe them into one `claude-director serve --stdio` invocation,
   plus a leading initialize.
3. Collect stdout lines; assert there are 31 lines (1 init + 30
   calls) and each parses cleanly with the expected id.

## Pass criteria

- 31 well-formed JSON lines on stdout.
- Each response's `id` matches its request's `id`.
- No response has an `error` field.

## Script

```bash
set -euo pipefail

{
    printf '%s
' '{"jsonrpc":"2.0","id":0,"method":"initialize"}'
    for i in $(seq 1 30); do
        printf '{"jsonrpc":"2.0","id":%d,"method":"tools/call","params":{"name":"help","arguments":{}}}
' "$i"
    done
} | claude-director serve --stdio 2>/dev/null > /tmp/mcp6.out

n=$(wc -l < /tmp/mcp6.out)
if [ "$n" != "31" ]; then
    echo "FAIL: $n response lines; want 31"; exit 1
fi

# Each response parses as JSON-RPC with the matching id and no error.
expected_id=0
while IFS= read -r line; do
    id=$(printf '%s' "$line" | jq -r '.id')
    err=$(printf '%s' "$line" | jq -r '.error // empty')
    if [ "$id" != "$expected_id" ]; then
        echo "FAIL: line id=$id; want $expected_id"; exit 1
    fi
    if [ -n "$err" ]; then
        echo "FAIL: id=$id carries error: $err"; exit 1
    fi
    expected_id=$(( expected_id + 1 ))
done < /tmp/mcp6.out

echo "mcp-6 ok: 31 sequential requests in one server lifetime"
```
