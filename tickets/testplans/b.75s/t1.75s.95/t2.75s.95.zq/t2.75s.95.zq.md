# Test case: help emits manifest-derived JSON on stdout, exit 0

Maps to Epic 1 AC #2.

## Setup

`./bin/agent-director` exists (run the build test case first).
HOME is set to a fresh, writable temp directory.

## Steps

1. Run `HOME=$tmpdir ./bin/agent-director help` and capture stdout, stderr,
   exit code.
2. Parse stdout as JSON.
3. Inspect the top-level shape — it must contain a `verbs` array.
4. Confirm at least one entry exists with `name == "help"` and a non-empty
   `description`.

## Pass criteria

- Exit code is 0.
- stderr is empty.
- The first non-whitespace byte of stdout is `{`.
- stdout parses as JSON without error.
- The parsed object's `verbs` array contains an entry with
  `name == "help"` and non-empty `description`.

## Notes

The descriptive text on the `help` verb is sourced from the manifest in
`internal/api/manifest/manifest.go`. If a future PR edits that file but
forgets to run `make generate`, the doc-drift gate (see separate t2 case)
catches the drift independently — the JSON shape will still be valid here.
