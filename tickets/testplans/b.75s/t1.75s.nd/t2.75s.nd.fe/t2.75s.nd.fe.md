# mcp-8: tools/list inputSchema is well-formed JSON Schema

Per SRD §4.1: each tool advertises a JSON Schema describing its
arguments. This case pins that the spawn tool's schema has the
expected shape: type=object, properties.cwd.type=string, properties
includes labels as a map.

## Setup

Per-case DB-reset.

## Steps

1. tools/list.
2. Pick the spawn entry.
3. Assert inputSchema.type = "object" and properties.cwd.type = "string".

## Pass criteria

- spawn entry exists.
- inputSchema.type = "object".
- properties.cwd.type = "string".
- properties.cwd has a non-empty description.

## Script

```bash
set -euo pipefail
out=$(printf '%s
%s
' \
    '{"jsonrpc":"2.0","id":1,"method":"initialize"}' \
    '{"jsonrpc":"2.0","id":2,"method":"tools/list"}' \
    | claude-director serve --stdio 2>/dev/null)

spawn_tool=$(printf '%s' "$out" | sed -n '2p' | jq -c '.result.tools[] | select(.name=="spawn")')
if [ -z "$spawn_tool" ]; then
    echo "FAIL: no spawn tool in tools/list"; exit 1
fi
schema_type=$(printf '%s' "$spawn_tool" | jq -r '.inputSchema.type')
cwd_type=$(printf '%s' "$spawn_tool" | jq -r '.inputSchema.properties.cwd.type')
cwd_desc=$(printf '%s' "$spawn_tool" | jq -r '.inputSchema.properties.cwd.description // ""')

if [ "$schema_type" != "object" ] || [ "$cwd_type" != "string" ] || [ -z "$cwd_desc" ]; then
    echo "FAIL: schema_type=$schema_type cwd_type=$cwd_type cwd_desc='$cwd_desc'"
    printf '%s' "$spawn_tool"; exit 1
fi
echo "mcp-8 ok: spawn inputSchema is well-formed (object, cwd:string)"
```
