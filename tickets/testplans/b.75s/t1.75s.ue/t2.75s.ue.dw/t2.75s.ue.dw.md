# resume-6: stale tmux session under the canonical name triggers ErrTmuxSessionCreate

Per SRD §8.1: resume refuses to launch when the canonical tmux session
name is already in use (resume does NOT auto-kill stale sessions —
the operator cleans up manually). The wrapped sentinel is
tmux.ErrTmuxSessionCreate.

## Setup

Per-case DB-reset.

## Steps

1. Seed an `ended` row directly via sqlite3 with a populated
   claude_session_id, JSONL on disk, and tmux_session_name = `cd-r6`.
2. Manually start a stale tmux session under `cd-r6` (running cat).
3. `resume` → non-zero; `err_name=ErrTmuxSessionCreate`.
4. The stale session is untouched (still alive).

## Pass criteria

- resume exits non-zero with the typed collision error.
- The pre-existing stale tmux session is NOT killed.

## Script

```bash
set -euo pipefail
session_id="r6-session-uuid"
mkdir -p "$HOME/.claude/projects/-tmp"
echo '{}' > "$HOME/.claude/projects/-tmp/${session_id}.jsonl"

sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode, claude_session_id, ended_at)
   VALUES ('id-r6', 'ended', '/tmp', 'cd-r6', 'off', '${session_id}', CURRENT_TIMESTAMP);"

# Pre-create the stale session under the canonical name.
tmux new-session -d -s cd-r6 -- cat

stderr_file=$(mktemp)
if claude-director resume --claude-instance-id id-r6 2>"$stderr_file" >/dev/null; then
    echo "FAIL: resume with stale tmux session exited 0"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrTmuxSessionCreate" ]; then
    echo "FAIL: err_name=$err_name; want ErrTmuxSessionCreate"; exit 1
fi

# Stale session must still be alive.
if ! tmux has-session -t cd-r6 2>/dev/null; then
    echo "FAIL: resume killed the stale session (should have left it alone)"; exit 1
fi

# Clean up our own stale session so it doesn't bleed into later cases.
tmux kill-session -t cd-r6 2>/dev/null || true
echo "resume-6 ok: stale tmux session → ErrTmuxSessionCreate; original untouched"
```
