# Epic 4 (`b.3d3`) — `release-postinstall-verify` testplan

t1 collector for Epic 4 of Plan Bee `b.3d3` (release pipeline
postinstall verify + prepublishOnly guards — `t1.3d3.8c`). Runs under
`make test-docker EPIC=release-postinstall-verify`; the driver
resolves the slug by case-insensitive substring match against this t1
title (`test/driver/run-testplan.sh`), so `release-postinstall-verify`
must remain in the title verbatim.

## Driver mode

`DRIVER_MODE=shell`. Each t2 body carries a single fenced ` ```bash `
block executed by the driver after the per-case DB-reset fixture.
Each case stages from `/work/source` into its own writable temp dir
under `/tmp/` so cases do not interfere; no live npm publish ever
runs (SR-6.1).

## Coverage outside this testplan

- Live publish flow (`npm publish` to the real registry) is operator-
  driven outside CI per SR-6.1; this testplan asserts the local
  gates that PRECEDE the publish, not the publish itself.

## Notes for the orchestrator

- Audit: run `make test-docker EPIC=release-postinstall-verify` and
  inspect the JSON stream.
- All three cases need `bun pm pack` + `bun install` to work inside
  a temp staging dir; the harness image has bun + go + file(1) +
  jq pinned in `test/Dockerfile`.
- Case r1 builds `libagent_director.so` via `make libagent_director`
  inside the staging dir so the umbrella's optional FFI sub-package
  resolves; cases r2 and r3 do not require the .so since they exit
  before any actual publish attempt.
- The fixture release tag the cases stamp is `0.4.1` (case r1),
  `0.4.2` (case r2's bumped umbrella), `0.4.1` (case r3's
  baseline). These are synthetic per-case values and do not couple
  the testplan to any specific Plan-Bee version.
