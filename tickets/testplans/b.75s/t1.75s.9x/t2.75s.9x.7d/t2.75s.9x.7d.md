# interact-7: send-keys on ended Spawn → ErrSpawnNotInteractive

Per SRD §4.3 + the state precondition: live states only. An `ended`
Spawn has nothing to type into; the verb returns the typed envelope
without contacting tmux.

## Setup

Per-case DB-reset. NO tmux session is created — the case proves the
state guard fires before the transport layer.

## Steps

1. Seed a `spawns` row at state `ended` (no tmux session needed; the
   guard returns before tmux is invoked).
2. `claude-director send-keys --claude-instance-id id-i7 --text "hi"` →
   non-zero exit; stderr envelope `err_name=ErrSpawnNotInteractive`.

## Pass criteria

- Non-zero exit.
- `err_name == ErrSpawnNotInteractive`.

## Script

```bash
set -euo pipefail
sqlite3 "$HOME/.claude-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i7', 'ended', '/tmp', 'cd-i7-absent', 'off');"
stderr_file=$(mktemp)
trap 'rm -f "$stderr_file"' EXIT
if claude-director send-keys --claude-instance-id id-i7 --text "hi" 2>"$stderr_file"; then
    echo "FAIL: send-keys unexpectedly succeeded on ended Spawn"; exit 1
fi
err_name="$(jq -r '.err_name' <"$stderr_file")"
test "$err_name" = "ErrSpawnNotInteractive"
echo "interact-7 ok: ErrSpawnNotInteractive on ended state"
```
