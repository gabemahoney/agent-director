# Bootstrap (Epic 1) — Test plan collector

Plain-English test cases (`t2`s under this collector) for Epic 1's behavior.
Per the Epic body, Epic 1's gate is the **degraded form** — `go test ./...`
+ doc-drift CI — because the Docker test harness does not yet exist (it's
what Epic 2 builds). These `t2`s document what the Docker-level tests
**will** look like once the harness lands, so the Epic 2 worker can wire
them up directly without re-deriving acceptance criteria.

## Sourced from

- Epic ticket: `t1.8op.3f` (the seven AC numbered there)
- SRD §15.2 (Docker integration tests)

## Notes for the Epic 2 worker

- Each `t2` body is intended to be readable verbatim by the harness's driver
  Claude, then translated into shell commands inside the container. Keep
  acceptance criteria observable (exit code, file modes, byte-level diff,
  JSON shape) so the driver has clear signals.
- The harness pins a Claude Code version; that pinning is **not** required
  to run any Epic 1 test — Bootstrap is pre-Claude-integration.
- Until the harness exists, none of these run in CI. They are spec, not
  fixtures.
