# ts-bun-6-publish-guard-still-fires-on-placeholder

Pins Epic 5 AC: the `prepublishOnly` guard (`scripts/check-not-placeholder.ts`)
remains a working tripwire against re-introducing the `CHANGEME-H3` placeholder
in any of the four published `package.json` files. H3 was resolved 2026-05-24;
this case keeps the guard's behavior under regression test.

It asserts two halves:

1. **Current names are clean.** The four real `package.json` files
   (umbrella + 3 per-platform) MUST NOT contain `CHANGEME-H3` in their
   `name` field. The guard run from each of those directories must exit 0.
2. **Guard still fires on a placeholder.** When the guard is run from a
   directory whose `package.json` name contains `@CHANGEME-H3/...`, it must
   exit 1 and emit a message containing `CHANGEME-H3` so the operator sees
   the breach.

## Setup

The script copies `/work/source` to a writable temp dir, runs
`bun install --no-progress`, then exercises the guard in both directions
described above. The placeholder case uses a synthetic temp `package.json`
so the worktree's real names are never mutated.

## Steps

1. Assert `bun` is in PATH; fail clearly if not.
2. Create workspace; `cp -a /work/source/.`.
3. `cd pkg/ts-bun-client && bun install --no-progress`.
4. For each of the four real `package.json` directories, run
   `bun run /work/source/pkg/ts-bun-client/scripts/check-not-placeholder.ts`;
   assert exit 0.
5. Create a synthetic temp dir with `package.json` `{"name": "@CHANGEME-H3/foo"}`.
   Run the guard from there; assert exit 1 and stderr contains `CHANGEME-H3`.

## Pass criteria

- All four real `package.json` directories: guard exits 0.
- Synthetic placeholder dir: guard exits 1 with `CHANGEME-H3` in stderr.
- Script exits 0; final line contains `ts-bun-6-publish-guard-still-fires-on-placeholder ok:`.

## Script

```bash
set -euo pipefail

if ! command -v bun >/dev/null 2>&1; then
    echo "FAIL: bun not found in PATH — check test/Dockerfile BUN_VERSION ARG"
    exit 1
fi

export HOME=/tmp/bun-home-tsbun6
mkdir -p "$HOME"

WORK=$(mktemp -d /tmp/tsbun6_XXXXXX)
cp -a /work/source/. "$WORK/"

cd "$WORK/pkg/ts-bun-client"
bun install --no-progress 2>/dev/null

GUARD="$WORK/pkg/ts-bun-client/scripts/check-not-placeholder.ts"

# (1) Real package.jsons must all pass the guard (exit 0).
for d in \
    "$WORK/pkg/ts-bun-client" \
    "$WORK/pkg/ts-bun-client/platforms/linux-x64" \
    "$WORK/pkg/ts-bun-client/platforms/darwin-x64" \
    "$WORK/pkg/ts-bun-client/platforms/darwin-arm64"; do
    OUT=$(mktemp /tmp/tsbun6_real.XXXXXX)
    EXIT=0
    (cd "$d" && bun run "$GUARD") > "$OUT" 2>&1 || EXIT=$?
    if [[ "$EXIT" -ne 0 ]]; then
        echo "FAIL: guard exited $EXIT in real package dir $d (post-H3-resolution names should be clean)"
        cat "$OUT" >&2
        exit 1
    fi
done

# (2) Synthetic placeholder must trip the guard (exit 1 + stderr contains CHANGEME-H3).
SYN=$(mktemp -d /tmp/tsbun6_syn.XXXXXX)
printf '{"name": "@CHANGEME-H3/foo"}\n' > "$SYN/package.json"
OUT=$(mktemp /tmp/tsbun6_syn_out.XXXXXX)
EXIT=0
(cd "$SYN" && bun run "$GUARD") > "$OUT" 2>&1 || EXIT=$?

if [[ "$EXIT" -eq 0 ]]; then
    echo "FAIL: guard exited 0 on synthetic placeholder name — tripwire is broken"
    cat "$OUT" >&2
    exit 1
fi

if ! grep -q "CHANGEME-H3" "$OUT"; then
    echo "FAIL: guard exited $EXIT but output does not contain CHANGEME-H3"
    cat "$OUT" >&2
    exit 1
fi

echo "ts-bun-6-publish-guard-still-fires-on-placeholder ok: clean on real pkgs, exits $EXIT on synthetic placeholder"
```
