# spawn-3: PreToolUse(AskUserQuestion) drives state to ask_user

Maps to Epic 3 AC #3 (AskUserQuestion → state = ask_user). Verifies the
SRD §5.2 tool-name carve-out: PreToolUse with `tool_name=AskUserQuestion`
maps to `ask_user`, any other tool maps to `working`.

## Setup

Per-case DB-reset has wiped the DB.

## Steps

1. spawn → SessionStart hook → state should be `waiting`.
2. PreToolUse(AskUserQuestion) hook → state should be `ask_user`.
3. PreToolUse(Bash) hook → state should be `working` (non-AUQ branch).

## Pass criteria

- All three states observed in order.

## Script

```bash
set -euo pipefail
id="$(agent-director spawn --cwd /tmp | jq -r '.claude_instance_id')"
printf '{"hook_event_name":"SessionStart"}' \
  | AGENT_DIRECTOR_INSTANCE_ID="$id" agent-director hook
printf '{"hook_event_name":"PreToolUse","tool_name":"AskUserQuestion"}' \
  | AGENT_DIRECTOR_INSTANCE_ID="$id" agent-director hook
state="$(agent-director status --claude-instance-id "$id" | jq -r '.state')"
test "$state" = "ask_user"
printf '{"hook_event_name":"PreToolUse","tool_name":"Bash"}' \
  | AGENT_DIRECTOR_INSTANCE_ID="$id" agent-director hook
state="$(agent-director status --claude-instance-id "$id" | jq -r '.state')"
test "$state" = "working"
echo "spawn-3 ok: AUQ -> ask_user, Bash -> working"
```
