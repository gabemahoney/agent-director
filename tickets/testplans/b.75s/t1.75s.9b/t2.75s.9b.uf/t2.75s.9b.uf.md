# recov-1: find-missing reconciles externally-killed Spawn

Maps to Epic 8 AC #1. Spawn TWO Claudes, externally kill ONE via
`tmux kill-session` (the manual operator path, NOT
`claude-director kill`), then `find-missing` reports count=1 and the
killed row's state flips to `missing` with `ended_at` populated.

The "two Claudes" framing matters: the degraded-mode guard
(SRD §14.6) refuses to write when the probe returns zero IDs but
the DB has live rows. We need at least one survivor so the probe
sees SOMETHING — which is also the realistic operator scenario
(find-missing exists for fleets, not single-Spawn setups).

## Setup

Per-case DB-reset. The harness image has `claude` + `tmux` + `jq` +
`sqlite3` on PATH.

## Steps

1. Spawn two Claudes; capture both ids and the first's tmux session
   name.
2. Poll `/proc/*/environ` until BOTH `CLAUDE_DIRECTOR_INSTANCE_ID`
   values are visible (the same surface the prober uses).
3. `tmux kill-session -t <name>` — external kill on Claude #1.
4. Poll until the killed id is no longer visible in `/proc`.
5. `find-missing` → expect `count=1` and Claude #1's id in `.ids`.
6. `get` on Claude #1 → state is `missing`, `ended_at` populated.
7. `get` on Claude #2 → unchanged.

## Pass criteria

- find-missing exits 0.
- Result envelope: `{"count":1,"ids":["<killed-id>"]}`.
- Killed row's state transitions to `missing`; `ended_at` is non-null.
- Survivor row is unchanged.

## Script

```bash
set -euo pipefail

# proc_has scans /proc/*/environ for a literal NUL-separated assignment.
# This is the same surface the prober walks; matching here guarantees
# the verb run after this point will see the same set we just observed.
proc_has() {
    local id="$1"
    for pid in $(ls /proc 2>/dev/null | grep -E '^[0-9]+$'); do
        if [ -r "/proc/$pid/environ" ] && \
            grep -aqz "CLAUDE_DIRECTOR_INSTANCE_ID=$id" "/proc/$pid/environ" 2>/dev/null; then
            return 0
        fi
    done
    return 1
}

killed=$(claude-director spawn --cwd /tmp | jq -r '.claude_instance_id')
survivor=$(claude-director spawn --cwd /tmp | jq -r '.claude_instance_id')
test -n "$killed" && test -n "$survivor"

sess=$(claude-director get --claude-instance-id "$killed" | jq -r '.tmux_session_name')
test -n "$sess"

# Wait until both claude children's env vars are visible.
deadline=$(( $(date +%s) + 10 ))
while [ "$(date +%s)" -lt "$deadline" ]; do
    if proc_has "$killed" && proc_has "$survivor"; then
        break
    fi
    sleep 0.2
done
if ! proc_has "$killed" || ! proc_has "$survivor"; then
    echo "FAIL: prober prerequisite — env vars never became visible in /proc"; exit 1
fi

# External kill on the first spawn.
tmux kill-session -t "$sess"

# Wait until the killed env disappears from /proc (process actually reaped).
deadline=$(( $(date +%s) + 10 ))
while [ "$(date +%s)" -lt "$deadline" ]; do
    if ! proc_has "$killed"; then
        break
    fi
    sleep 0.2
done
if proc_has "$killed"; then
    echo "FAIL: killed claude env still visible after kill-session + 10s"; exit 1
fi

res=$(claude-director find-missing)
count=$(printf '%s' "$res" | jq -r '.count')
ids=$(printf '%s' "$res" | jq -r '.ids | join(",")')
if [ "$count" != "1" ]; then
    echo "FAIL: count=$count; want 1 (res=$res)"; exit 1
fi
if [ "$ids" != "$killed" ]; then
    echo "FAIL: ids=$ids; want $killed"; exit 1
fi

killed_state=$(claude-director get --claude-instance-id "$killed" | jq -r '.state')
if [ "$killed_state" != "missing" ]; then
    echo "FAIL: killed state=$killed_state; want missing"; exit 1
fi
ended=$(sqlite3 "$HOME/.claude-director/state.db" \
    "SELECT ended_at FROM spawns WHERE claude_instance_id='$killed';")
if [ -z "$ended" ]; then
    echo "FAIL: ended_at not populated after find-missing"; exit 1
fi

survivor_state=$(claude-director get --claude-instance-id "$survivor" | jq -r '.state')
if [ "$survivor_state" = "missing" ]; then
    echo "FAIL: survivor was wrongly marked missing"; exit 1
fi
echo "recov-1 ok: external kill on 1 of 2 → find-missing marks just the killed row"
```
