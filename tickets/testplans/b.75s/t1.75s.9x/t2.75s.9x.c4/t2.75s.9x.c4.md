# interact-2: multiline composed as one buffer

Maps to Epic 4 AC #2 (`send-keys --text $'line1
line2'` produces a
single composed message visible as two lines). Pins the SRD §4.3
contract that LF (0x0A) bytes in the text payload compose newlines in
the input box rather than triggering submission.

## Setup

Per-case DB-reset. A `cat` pane provides a deterministic echo target.

## Steps

1. Start `tmux new-session -d -s cd-i2 -- cat`.
2. Seed a `spawns` row at state `waiting`.
3. `agent-director send-keys --claude-instance-id id-i2 --text $'line1
line2'`.
4. Read the pane; assert both `line1` and `line2` appear.
5. Inspect that exactly one trailing Enter was emitted (cat echoes two
   lines if the buffer was one composed message; cat would echo each
   line separately on its own pre-Enter chunk regardless, so the LF
   preservation is asserted at the byte level by reading the pane).

## Pass criteria

- send-keys exit 0.
- Both `line1` and `line2` are in the pane.
- The bytes between them include LF (not stripped).

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-i2 -- cat
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i2', 'waiting', '/tmp', 'cd-i2', 'off');"
agent-director send-keys --claude-instance-id id-i2 --text $'line1
line2' >/dev/null
sleep 0.2
pane="$(agent-director read-pane --claude-instance-id id-i2 --n-lines 100 | jq -r '.pane')"
echo "$pane" | grep -q "line1"
echo "$pane" | grep -q "line2"
# Multiline preservation: line1 and line2 must occupy separate display
# lines in the captured pane buffer.
lines="$(echo "$pane" | grep -c -E '^line[12]$' || true)"
test "$lines" -ge 2
echo "interact-2 ok: both lines composed"
```
