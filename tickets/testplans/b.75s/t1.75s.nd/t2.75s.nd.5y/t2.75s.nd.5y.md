# mcp-2: tools/list count matches manifest minus filtered

Per SRD §4.1: tools/list is generated from internal/api/manifest.Verbs,
filtering out `hook` (internal) and `serve` (self-referential). This
case pins the drift-by-construction invariant: a new manifest verb
that isn't filtered must appear in tools/list, and the count must
stay synchronized.

## Setup

Per-case DB-reset.

## Steps

1. Pipe initialize + tools/list into the server.
2. Parse stdout's second line; count tools.
3. Compare against the manifest count via `agent-director help`,
   subtracting the two filtered verbs.

## Pass criteria

- `len(result.tools)` equals `(help.verbs count) - 2`.

## Script

```bash
set -euo pipefail
out=$(printf '%s
%s
' \
    '{"jsonrpc":"2.0","id":1,"method":"initialize"}' \
    '{"jsonrpc":"2.0","id":2,"method":"tools/list"}' \
    | agent-director serve --stdio 2>/dev/null)
n_tools=$(printf '%s' "$out" | sed -n '2p' | jq -r '.result.tools | length')
n_verbs=$(agent-director help | jq -r '.verbs | length')
expected=$(( n_verbs - 2 ))
if [ "$n_tools" != "$expected" ]; then
    echo "FAIL: tools=$n_tools verbs=$n_verbs expected=$expected (verbs - hook - serve)"
    exit 1
fi
echo "mcp-2 ok: tools/list count = $n_tools = manifest($n_verbs) - 2 filtered"
```
