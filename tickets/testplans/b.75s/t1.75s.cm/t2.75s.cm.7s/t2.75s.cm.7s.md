# ci-matrix-4-runbook-seven-sections

Pins Epic 6 AC: "`docs/self-hosted-runner-setup.md` exists and covers
all seven sections enumerated above (Prerequisites + the six core
sections)." Each section's level-2 heading must be present verbatim
in the runbook.

## Setup

No DB state. Pure read of the committed runbook file.

## Steps

1. Assert the runbook exists at `/work/source/docs/self-hosted-runner-setup.md`.
2. For each of the seven required `##`-level headings, grep for the
   heading line.
3. If any heading is missing, fail with the list of missing headings
   so the operator sees exactly which section to add.
4. Otherwise emit pass marker.

## Pass criteria

- All seven section headings are present in the runbook.
- Final line matches `ci-matrix-4 ok: ...`.

## Script

```bash
set -euo pipefail

MD="/work/source/docs/self-hosted-runner-setup.md"
if [[ ! -f "$MD" ]]; then
    echo "FAIL: runbook not found at $MD"
    exit 1
fi

# Each heading is matched as a leading-line literal so we don't get
# fooled by the same phrase appearing in body prose. The seven
# headings are the ones the Epic body enumerates:
#   0. Prerequisites
#   1. Download
#   2. Register
#   3. Install as a launchd service
#   4. Security configuration (CRITICAL)
#   5. Verify end-to-end
#   6. Operations
REQUIRED=(
    "## 0. Prerequisites"
    "## 1. Download"
    "## 2. Register the runner"
    "## 3. Install as a launchd service"
    "## 4. Security configuration (CRITICAL)"
    "## 5. Verify end-to-end"
    "## 6. Operations"
)

MISSING=()
for heading in "${REQUIRED[@]}"; do
    if ! grep -qxF "$heading" "$MD"; then
        MISSING+=("$heading")
    fi
done

if [[ ${#MISSING[@]} -gt 0 ]]; then
    echo "FAIL: runbook missing required section heading(s):"
    for h in "${MISSING[@]}"; do
        echo "  - $h"
    done
    exit 1
fi

echo "ci-matrix-4 ok: all 7 required sections present in runbook"
```
