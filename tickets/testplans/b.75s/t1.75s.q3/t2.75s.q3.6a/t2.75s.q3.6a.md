# release-6: dirty working tree rejection

Per SRD §16.4: release.sh refuses to release dirty state. The check
must fire BEFORE the build step.

## Setup

Per-case DB-reset. A synthesized git repo with one committed file
plus one uncommitted file.

## Steps

1. Init a clean repo with an initial commit.
2. Make an uncommitted change.
3. Invoke release.sh --dry-run v0.1.0.
4. Assert exit 2 and stderr mentions "dirty".

## Pass criteria

- Exit code is 2.
- Stderr contains "dirty".

## Script

```bash
set -euo pipefail
work=$(mktemp -d)
cd "$work"
git init -q
git config user.email test@example.com
git config user.name test
echo "x" > a.txt
git add a.txt
git -c commit.gpgsign=false commit -q -m "init"
# Dirty the tree
echo "y" >> a.txt

stderr_file=$(mktemp)
set +e
/opt/skills/release-claude-director/release.sh v0.1.0 --dry-run 2>"$stderr_file"
ec=$?
set -e

if [ "$ec" != "2" ]; then
    echo "FAIL: exit=$ec; want 2"; cat "$stderr_file"; exit 1
fi
if ! grep -qi 'dirty' "$stderr_file"; then
    echo "FAIL: stderr did not mention dirty"; cat "$stderr_file"; exit 1
fi
echo "release-6 ok: dirty working tree rejected"
```
