# go-client-4-no-home-store-touched

Pins Epic 4 AC: the Go smoke run never touches the real
`~/.agent-director/state.db`. The in-process `TestMain` canary in
`test/smoke/go/main_test.go` already snapshots the real home and fails
the suite from inside Go on any write; this Docker case is the
external observable check — it seeds a known-content `state.db` file
under the container's `$HOME`, records its size + sha256, runs the smoke
test, and asserts the file is byte-identical afterwards.

## Setup

The container `$HOME` (typically `/root`) usually has no
`~/.agent-director/` directory. This script creates one with a known seed
file (`state.db` containing a fixed byte sequence) BEFORE the smoke run so
the byte-identical assertion has something to compare. Without the seed
the snapshot would be nil both before and after and the test would
trivially pass. The script copies `/work/source` to a writable temp dir
for `go test`'s scratch needs.

## Steps

1. Assert `go` and `sha256sum` are in PATH; fail with a clear message if not.
2. Create `$HOME/.agent-director/` and write a known seed
   (`state.db` with deterministic content).
3. Snapshot the seed file: record `stat -c '%s'`, `sha256sum`, and
   `stat -c '%Y'` (mtime).
4. Create a writable workspace under `/tmp/` and `cp -a /work/source/.` into it.
5. `cd <workspace> && go test ./test/smoke/go/...`; assert exit 0.
6. Re-snapshot the seed file.
7. Assert size and sha256 are byte-identical. mtime is reported as a
   softer signal but does not fail the case on its own.

## Pass criteria

- `go test ./test/smoke/go/...` exits 0.
- `$HOME/.agent-director/state.db` has the same size before and after.
- `$HOME/.agent-director/state.db` has the same sha256 before and after.
- Script exits 0; final line matches `go-client-4-no-home-store-touched ok: ...`.

## Script

```bash
set -euo pipefail

if ! command -v go >/dev/null 2>&1; then
    echo "FAIL: go not found in PATH"
    exit 1
fi
if ! command -v sha256sum >/dev/null 2>&1; then
    echo "FAIL: sha256sum not found in PATH"
    exit 1
fi

# Seed a known-content state.db under the container's real HOME. We use
# user.Current().HomeDir's equivalent — `getent passwd "$USER"` field 6 —
# rather than $HOME so the seed lands at the path the smoke test's canary
# inspects via user.Current() (which reads /etc/passwd and is unaffected
# by $HOME). For the container's `root` user this is /root.
REAL_HOME=$(getent passwd "$(id -un)" | cut -d: -f6)
if [[ -z "$REAL_HOME" ]]; then
    echo "FAIL: could not resolve real home directory for user $(id -un)"
    exit 1
fi

AD_DIR="$REAL_HOME/.agent-director"
SEED="$AD_DIR/state.db"

mkdir -p "$AD_DIR"
# Deterministic seed content. 4 KiB of a fixed pattern so a partial
# overwrite would change both size and sha.
printf 'go-client-4-no-home-store-touched canary seed\n' > "$SEED"
dd if=/dev/zero bs=1 count=4000 status=none >> "$SEED"

BEFORE_SIZE=$(stat -c '%s' "$SEED")
BEFORE_SHA=$(sha256sum "$SEED" | cut -d' ' -f1)
BEFORE_MTIME=$(stat -c '%Y' "$SEED")

WORK=$(mktemp -d /tmp/goclient4_XXXXXX)
cp -a /work/source/. "$WORK/"

cd "$WORK"

OUT_FILE=$(mktemp /tmp/goclient4_out.XXXXXX)
EXIT=0
go test ./test/smoke/go/... > "$OUT_FILE" 2>&1 || EXIT=$?

if [[ "$EXIT" -ne 0 ]]; then
    echo "FAIL: go test ./test/smoke/go/... exited with code $EXIT"
    tail -200 "$OUT_FILE" >&2
    exit 1
fi

if [[ ! -f "$SEED" ]]; then
    echo "FAIL: seed file $SEED disappeared during smoke run"
    exit 1
fi

AFTER_SIZE=$(stat -c '%s' "$SEED")
AFTER_SHA=$(sha256sum "$SEED" | cut -d' ' -f1)
AFTER_MTIME=$(stat -c '%Y' "$SEED")

if [[ "$BEFORE_SIZE" != "$AFTER_SIZE" ]]; then
    echo "FAIL: seed size changed: $BEFORE_SIZE -> $AFTER_SIZE bytes"
    exit 1
fi

if [[ "$BEFORE_SHA" != "$AFTER_SHA" ]]; then
    echo "FAIL: seed sha256 changed: $BEFORE_SHA -> $AFTER_SHA"
    exit 1
fi

# mtime is a softer signal — note any change but don't fail on it.
MTIME_NOTE=""
if [[ "$BEFORE_MTIME" != "$AFTER_MTIME" ]]; then
    MTIME_NOTE=" (mtime changed: $BEFORE_MTIME -> $AFTER_MTIME — bytes unchanged so still ok)"
fi

echo "go-client-4-no-home-store-touched ok: $SEED unchanged (size=$AFTER_SIZE sha256=${AFTER_SHA:0:12}...)$MTIME_NOTE"
```
