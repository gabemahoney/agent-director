# resume-3: resume on a live row returns ErrSpawnNotResumable

Per SRD §8.1: resume only resurrects terminal-state rows. A `pending`
or `waiting` row means the original Claude is still running; calling
resume is a caller-side error.

## Setup

Per-case DB-reset.

## Steps

1. `spawn --cwd /tmp` → captures id. State is `pending` immediately.
2. `resume --claude-instance-id <id>` → expect non-zero exit; stderr
   envelope `err_name=ErrSpawnNotResumable`.
3. Confirm row state is still `pending` (no DB mutation).

## Pass criteria

- resume exits non-zero with the typed error.
- Row state unchanged.

## Script

```bash
set -euo pipefail
id=$(claude-director spawn --cwd /tmp | jq -r '.claude_instance_id')
test -n "$id"

stderr_file=$(mktemp)
if claude-director resume --claude-instance-id "$id" 2>"$stderr_file" >/dev/null; then
    echo "FAIL: resume on live row exited 0"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrSpawnNotResumable" ]; then
    echo "FAIL: err_name=$err_name; want ErrSpawnNotResumable"; exit 1
fi

state=$(claude-director get --claude-instance-id "$id" | jq -r '.state')
if [ "$state" = "ended" ] || [ "$state" = "missing" ]; then
    echo "FAIL: row state mutated to terminal: $state"; exit 1
fi
echo "resume-3 ok: live row → ErrSpawnNotResumable; row unchanged"
```
