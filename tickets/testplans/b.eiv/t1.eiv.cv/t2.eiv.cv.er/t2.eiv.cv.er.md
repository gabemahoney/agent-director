# subprocess-coverage-5-error-catalog

Per SRD §SR-4.1 / §SR-4.2 / §SR-4.4: every `err_name` in
`pkg/api/errnames/catalog.json` (34 entries as of v0.5.0) must round
trip through the subprocess Client as an `instanceof Err<Name>`
throw. `subprocess-client-1-smoke` (sibling t1) samples ~6 of these
names; this case asserts the complete catalog.

The Client's `errorMap` is built at module load by reflecting over
the catalog and looking up the matching constructor in `errors.ts`.
This case proves both halves of that pairing for every entry:
the catalog → constructor wiring (`errorMap.get`) and the
constructor → name fidelity (`e instanceof Err<Name>`,
`e.errName === <Name>`).

## Setup

- `${HOME}=$(mktemp -d)`.
- `bun install file://...tgz` into `${HOME}/proj`.
- Stage a stub CLI binary that reads `STUB_ERR_NAME` from its env,
  writes a `{err_name, err_description}` envelope to stderr, and
  exits 1.

## Steps

1. Driver loads the catalog from `/work/source/pkg/api/errnames/catalog.json`
   (mounted read-only).
2. For each entry, the driver sets `process.env.STUB_ERR_NAME` to
   the entry's name, calls `client.version({})`, and asserts:
   - the call rejected;
   - the rejection is `instanceof` the matching `Err<Name>` from
     `agent-director`'s default exports;
   - `e.errName` is the expected name.
3. Driver also asserts a non-catalog name produces `ErrUnknownErrorName`.

## Pass criteria

- All 34 catalog entries pass `instanceof Err<Name>` + name check.
- A synthetic `ErrNotInCatalog` entry surfaces as `ErrUnknownErrorName`.

## Script

```bash
set -euo pipefail
HOME_DIR="$HOME"
PROJ="$HOME_DIR/proj"
rm -rf "$PROJ"
mkdir -p "$PROJ"
cd "$PROJ"

. /work/source/tickets/testplans/b.eiv/t1.19d.6c/build-tarball.sh

cat > "$PROJ/package.json" <<'EOF'
{
  "name": "subprocess-coverage-er-consumer",
  "version": "0.0.0",
  "type": "module"
}
EOF

bun install "file://${TARBALL}" >/dev/null

DEST="$PROJ/node_modules/@agent-director/linux-x64"
mkdir -p "$DEST/bin"
cp "$PLATFORM_PKG_DIR/package.json" "$DEST/package.json"

# Stub CLI: emit a typed-error envelope on stderr and exit non-zero. The
# subprocessClient scans stderr lines (last-to-first) for a line starting
# with `{` that parses as an error envelope, then re-throws it as the
# matching Err<Name> subclass.
cat > "$DEST/bin/agent-director" <<'STUB_EOF'
#!/usr/bin/env bash
name="${STUB_ERR_NAME:-ErrCwdMissing}"
desc="${STUB_ERR_DESC:-stub error for ${name}}"
printf '{"err_name":"%s","err_description":"%s"}\n' "$name" "$desc" >&2
exit 1
STUB_EOF
chmod +x "$DEST/bin/agent-director"

cp /work/source/tickets/testplans/b.eiv/t1.eiv.cv/t2.eiv.cv.er/driver.mjs "$PROJ/driver.mjs"

export CATALOG_PATH=/work/source/pkg/api/errnames/catalog.json

out=$(bun run "$PROJ/driver.mjs" 2>&1)
echo "$out"
grep -q "OK error-catalog" <<<"$out" || { echo "FAIL: error-catalog assertion missing"; exit 1; }
grep -q "OK unknown-name"  <<<"$out" || { echo "FAIL: unknown-name assertion missing"; exit 1; }
echo "subprocess-coverage-5-error-catalog ok"
```
