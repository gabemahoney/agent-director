# interact-5: read-pane default strips ANSI but preserves unicode

Per Epic 4 testplan + `reference/pane-output-research.md`: default
`read-pane` returns ANSI-free pane bytes (SGR colors / cursor moves
gone) while leaving unicode TUI glyphs (`❯`, `⎿`, `🐝`, box-drawing)
intact. The orchestrator reads glyphs as state signal; ASCII-mapping
them would destroy that.

## Setup

Per-case DB-reset. Use `printf` to emit a controlled mix of ANSI
escapes and unicode glyphs into the pane.

## Steps

1. Start a tmux session running `bash` so we can `printf` into its
   pane.
2. Seed `spawns` row.
3. Use tmux send-keys to type a printf line that emits SGR + glyphs to
   the pane's stdout, then read-pane (default).
4. Assert the captured pane contains the glyph `❯` and does NOT
   contain any `\x1b[` escape.

## Pass criteria

- Captured pane contains the unicode prompt glyph.
- Captured pane does not contain `\x1b[`.

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-i5 -- bash
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i5', 'waiting', '/tmp', 'cd-i5', 'off');"
# Send a printf that emits a colored ❯ glyph followed by " hi".
# Use direct tmux send-keys (this bash IS the shell inside the pane).
tmux send-keys -t cd-i5:0.0 $'printf "\\033[31m\\xe2\\x9d\\xaf\\033[0m hi\
"' Enter
sleep 0.3
pane="$(agent-director read-pane --claude-instance-id id-i5 --n-lines 50 | jq -r '.pane')"
# Glyph must survive.
echo "$pane" | grep -q "❯"
# Escape codes must be stripped.
if echo "$pane" | grep -q $'\x1b\['; then
    echo "FAIL: ANSI escape leaked into default read-pane output"; exit 1
fi
echo "interact-5 ok: ANSI stripped, ❯ preserved"
```
