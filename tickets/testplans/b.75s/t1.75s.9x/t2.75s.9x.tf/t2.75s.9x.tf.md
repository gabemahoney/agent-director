# interact-8: send-keys with relay_mode=on + check_permission → ErrSendKeysWhileRelayed

Per SRD §4.3: when a Spawn is sitting on a relayed permission prompt
(`relay_mode=on` AND `state=check_permission`), the permission relay
(Epic 10) owns the modal answer. A pane-side send-keys would race the
decide() write. Epic 4 stubs this guard so the precondition is correct
from day one — the actual relay path activates in Epic 10.

## Setup

Per-case DB-reset. No tmux session needed (guard returns before tmux).

## Steps

1. Seed `spawns` row at state `check_permission` with `relay_mode=on`.
2. `agent-director send-keys --claude-instance-id id-i8 --text "1"` →
   non-zero exit; `err_name=ErrSendKeysWhileRelayed`.
3. (Negative control) Update the row to `relay_mode=off`. Re-run
   send-keys against a live tmux session — succeeds.

## Pass criteria

- Step 2 non-zero exit with the relay guard envelope.
- Step 3 succeeds (proves the guard is conditional on relay_mode, not
  on the state alone).

## Script

```bash
set -euo pipefail
DB="$HOME/.agent-director/state.db"
sqlite3 "$DB" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i8', 'check_permission', '/tmp', 'cd-i8', 'on');"

stderr_file=$(mktemp)
trap 'rm -f "$stderr_file"' EXIT
if agent-director send-keys --claude-instance-id id-i8 --text "1" 2>"$stderr_file"; then
    echo "FAIL: send-keys unexpectedly succeeded with relay_mode=on"; exit 1
fi
err_name="$(jq -r '.err_name' <"$stderr_file")"
test "$err_name" = "ErrSendKeysWhileRelayed"

# Negative control: relay_mode=off with the same state should succeed.
sqlite3 "$DB" "UPDATE spawns SET relay_mode='off' WHERE claude_instance_id='id-i8';"
tmux new-session -d -s cd-i8 -- cat
agent-director send-keys --claude-instance-id id-i8 --text "1" >/dev/null
echo "interact-8 ok: relay guard fires under on; clears under off"
```
