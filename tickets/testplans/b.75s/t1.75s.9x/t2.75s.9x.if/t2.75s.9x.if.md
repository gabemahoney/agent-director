# interact-1: single-line send-keys + read-pane roundtrip

Maps to Epic 4 AC #1 (`send-keys --text "what is 2+2?"` lands; `read-pane`
shows the answer). The harness exercises the verb surface end-to-end via
real `tmux` inside the Docker image — no Claude required (a `cat`
process holds the pane open, providing a deterministic pane buffer the
test can drive and read back).

## Setup

Per-case DB-reset has wiped `~/.agent-director/state.db` and killed
`cd-*` sessions. The container has `agent-director`, `tmux`, `sqlite3`,
and `jq` on PATH.

## Steps

1. Start a tmux session `cd-i1` running `cat` so the pane stays alive
   and echoes whatever is typed into it.
2. Seed a `spawns` row at state `waiting` with `tmux_session_name=cd-i1`
   and `claude_instance_id=id-i1`.
3. `agent-director send-keys --claude-instance-id id-i1 --text "hello"`.
4. Allow a short settle (cat echoes back).
5. `agent-director read-pane --claude-instance-id id-i1 | jq -r '.pane'`
   and assert the captured pane contains `hello`.

## Pass criteria

- Step 3 exit 0; empty `{}` on stdout.
- Step 5 captures the line containing `hello`.
- Default `read-pane` returns ANSI-free text (cat echoes are plain
  ASCII, but the assertion is symmetric: no `\x1b[` escapes).

## Script

```bash
set -euo pipefail
tmux new-session -d -s cd-i1 -- cat
sqlite3 "$HOME/.agent-director/state.db" \
  "INSERT INTO spawns (claude_instance_id, state, cwd, tmux_session_name, relay_mode)
   VALUES ('id-i1', 'waiting', '/tmp', 'cd-i1', 'off');"
out="$(agent-director send-keys --claude-instance-id id-i1 --text 'hello')"
test "$(echo "$out" | jq -c '.')" = "{}"
sleep 0.2
pane="$(agent-director read-pane --claude-instance-id id-i1 | jq -r '.pane')"
echo "$pane" | grep -q "hello"
test -z "$(echo "$pane" | grep -F $'\x1b[')" || (echo "pane contains escape codes"; exit 1)
echo "interact-1 ok: pane=${pane}"
```
