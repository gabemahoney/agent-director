# Epic 3 (`b.3d3`) — `installsh-preflight-gates` testplan

t1 collector for Epic 3 of Plan Bee `b.3d3` (install.sh preflight
OS/CPU gate + `--binary` architecture probe — `t1.3d3.eh`). Runs under
`make test-docker EPIC=installsh-preflight-gates`; the driver resolves
the slug by case-insensitive substring match against this t1 title
(`test/driver/run-testplan.sh`), so `installsh-preflight-gates` must
remain in the title verbatim.

## Sourced from

- Plan Bee: `b.3d3`.
- Epic ticket: `t1.3d3.eh` (Acceptance Criteria around install.sh's
  two new preflight gates).
- SRD: `t1.fg3.7i` §SR-2.1 / §SR-2.2 / §SR-2.3 (preflight ordering),
  §SR-6.3 (per-Epic Docker testplans).

## Tests included

| t2 | what it pins |
|---|---|
| `t2.75s.pg.b7` — `bootstrap-7-installsh-arch-probe` | On the linux/x86_64 harness, invoke `install.sh --binary <wrong-arch>` where the binary is a synthetic Mach-O ARM64 file (no network fetch). install.sh exits `2` with stderr containing the SR-2.2 mismatch message naming the binary, the detected architecture excerpt, and the host pair. `${HOME}/.agent-director/` is not created; no `~/.local/bin/agent-director` symlink. |
| `t2.75s.pg.b8` — `bootstrap-8-installsh-uname-gate` | On the linux/x86_64 harness, invoke `install.sh` with `uname` spoofed via a wrapper on PATH that returns a rejected tuple (`Linux/aarch64`). install.sh exits `2` with stderr containing the SR-2.1 unsupported-host message naming the supported set and referencing `b.fg3`. `${HOME}/.agent-director/` is not created. |

## Driver mode

`DRIVER_MODE=shell`. Each t2 body carries a single fenced ` ```bash `
block executed by the driver after the per-case DB-reset fixture.
Each case uses a per-case `HOME=$(mktemp -d)` so cases do not interfere.

## Notes for the orchestrator

- Audit: run `make test-docker EPIC=installsh-preflight-gates` and
  inspect the JSON stream.
- The harness image installs `file(1)` (the SR-2.2 probe dependency)
  via `test/Dockerfile`'s apt-get list. Both new preflight gates
  surface a self-explanatory message on failure; the testplan asserts
  the exact SR-2.1 / SR-2.2 substrings.
- Case 7 generates a tiny synthetic Mach-O ARM64 file (~56 bytes) via
  `python3` inside the bash block; this avoids cross-building a real
  darwin binary inside the linux container while still exercising
  `file(1)`'s real classifier.
- Case 8's `uname` wrapper PATH-prepends a temp dir containing an
  executable `uname` script that returns `Linux` / `aarch64` for the
  `-s` / `-m` flags respectively.

## tmux session sweep

The cases do NOT exercise `agent-director spawn`. No `cd-*` sessions
are created; the DB-reset fixture's sweep is a no-op.

## SR-7 cross-cutting

install.sh's existing flag surface, exit-code contract, and happy-path
behavior are preserved. Epic 1's `postinstall-skill-copy` and Epic 2's
`postinstall-platform-refusal` testplans are unaffected by the
install.sh edits.
