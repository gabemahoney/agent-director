# tmpl-7: unsafe template name is rejected

Per SRD §10.3: empty / `.` / `..` / leading-dot / path-separator
names are rejected before any disk write. The `../escape` form is
the canonical path-traversal attempt.

## Setup

Per-case DB-reset.

## Steps

1. `make-template --name '../escape' --cwd /tmp` → exit non-zero;
   `err_name=ErrTemplateNameUnsafe`.
2. Confirm no `escape.toml` file was written outside the templates
   dir (defense in depth — name validation already blocks it, but
   we verify the disk side too).

## Pass criteria

- make-template exits non-zero with `err_name=ErrTemplateNameUnsafe`.
- No `escape.toml` file lands in `~/.agent-director/` (one dir above
  the templates dir, which is where `../escape.toml` would resolve to
  absent the name check).

## Script

```bash
set -euo pipefail
stderr_file=$(mktemp)
if agent-director make-template --name '../escape' --cwd /tmp 2>"$stderr_file" >/dev/null; then
    echo "FAIL: unsafe name accepted"; exit 1
fi
err_name=$(jq -r '.err_name' "$stderr_file")
if [ "$err_name" != "ErrTemplateNameUnsafe" ]; then
    echo "FAIL: err_name=$err_name; want ErrTemplateNameUnsafe"; exit 1
fi
if [ -e "$HOME/.agent-director/escape.toml" ]; then
    echo "FAIL: ../escape.toml landed at $HOME/.agent-director/"; exit 1
fi
echo "tmpl-7 ok: unsafe name rejected; nothing leaked above templates dir"
```
