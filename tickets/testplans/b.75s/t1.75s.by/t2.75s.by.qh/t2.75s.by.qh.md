# smoke-4: broken db-reset surfaces as a fail case with a clear reason

Pins the bug fix for `b.emq`: when the per-case `db-reset.sh` returns
non-zero, the driver MUST report the case as `fail` with the reason
`"db-reset failed"`. Pre-fix, `run-testplan.sh` swallowed the failure
with `|| true`, so the case body would run against unreset state and
the case might still report pass — masking real harness breakage.

## Setup

Per-case DB-reset has wiped the real DB. The container has the harness
driver at `/opt/driver/`. The case body builds a synthetic harness
sandbox under `mktemp -d` so it can inject a deliberately-broken
db-reset without touching the real one (which would compromise the
remaining cases in this very testplan run).

## Steps

1. Copy `run-testplan.sh` and `prompt.md` from `/opt/driver/` into a
   temp dir, plus a `db-reset.sh` that prints to stderr and exits 7.
2. Build a one-case mini-testplan in the same temp tree.
3. Run the harness against the mini-testplan; capture its JSONL
   stream.
4. Assert: the JSONL contains a `fail` case whose `details` mention
   `db-reset`, and the summary tallies at least one failure.

## Pass criteria

- Synthesized harness run reports the synthetic case as `fail`.
- The fail reason mentions `db-reset` (the regression signal).
- The summary line records at least one fail.

## Script

```bash
set -euo pipefail
tmpdir=$(mktemp -d)
trap 'rm -rf "$tmpdir"' EXIT

# 1. Sandbox a copy of the driver with a broken db-reset.
cp /opt/driver/run-testplan.sh "$tmpdir/"
cp /opt/driver/prompt.md "$tmpdir/"
cat > "$tmpdir/db-reset.sh" <<'BROKEN_DBRESET'
#!/bin/bash
echo "deliberately-broken db-reset" >&2
exit 7
BROKEN_DBRESET
chmod +x "$tmpdir/db-reset.sh"

# 2. Build a minimal one-case testplan. Fence variable avoids embedding
#    inner ```bash markers that the outer awk extractor would otherwise
#    collide with.
fence="$(printf '%s' '`'; printf '%s' '`'; printf '%s' '`')"
mkdir -p "$tmpdir/tp/b.X/t1.X.Y/t2.X.Y.Z"
cat > "$tmpdir/tp/b.X/t1.X.Y/t1.X.Y.md" <<'T1_EOF'
---
id: t1.X.Y
title: Epic smoke-4 broken-db-reset probe
parent: b.X
children:
- t2.X.Y.Z
---
# probe
T1_EOF
{
    echo '---'
    echo 'id: t2.X.Y.Z'
    echo 'parent: t1.X.Y'
    echo '---'
    echo '# probe case'
    echo
    echo "${fence}bash"
    echo 'echo "should never execute since db-reset is broken"'
    echo 'exit 0'
    echo "${fence}"
} > "$tmpdir/tp/b.X/t1.X.Y/t2.X.Y.Z/t2.X.Y.Z.md"

# 3. Run the synthetic harness. Match-by-title resolution finds the
#    sandbox testplan via the "smoke-4 broken-db-reset probe" slug.
out="$(TESTPLAN_ROOT="$tmpdir/tp" EPIC="smoke-4 broken-db-reset probe" DRIVER_MODE=shell bash "$tmpdir/run-testplan.sh" 2>/dev/null || true)"

# 4. Assertions. The case must be reported fail, the details must
#    mention db-reset, and the summary must record at least one fail.
#    Use -s (slurp) so jq -e doesn't trip on inputs that didn't match
#    the select filter (jq -e returns 4 if any input produces no
#    output, even when other inputs produced true).
echo "$out" | jq -sce 'any(.case == "t2.X.Y.Z" and .status == "fail")' >/dev/null
echo "$out" | jq -sce 'any(.case == "t2.X.Y.Z" and (.details | test("db-reset")))' >/dev/null
echo "$out" | jq -sce 'any(.summary.fail >= 1)' >/dev/null

echo "smoke-4 ok: broken db-reset surfaced as fail with the expected reason"
```
