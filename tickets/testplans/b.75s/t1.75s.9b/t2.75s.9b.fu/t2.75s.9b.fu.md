# recov-3: find-missing is a no-op when every live row is alive

The negative case: real `spawn` produces real processes with their
env vars set; `find-missing` sees both ids in the probe and
transitions nothing.

## Setup

Per-case DB-reset.

## Steps

1. Spawn two Claudes.
2. `find-missing` → expect `count=0`, `.ids=[]`.
3. Inspect both rows: still in their original live states.

## Pass criteria

- find-missing exits 0 with `count=0`.
- Both rows' states are unchanged.

## Script

```bash
set -euo pipefail
id1=$(claude-director spawn --cwd /tmp | jq -r '.claude_instance_id')
id2=$(claude-director spawn --cwd /tmp | jq -r '.claude_instance_id')
test -n "$id1" && test -n "$id2"

# Give the kernel a moment to expose /proc/<pid>/environ for both
# child claude processes. Without this the prober race-loses and
# would flag a still-bootstrapping pid as missing.
deadline=$(( $(date +%s) + 5 ))
while [ "$(date +%s)" -lt "$deadline" ]; do
    if pgrep -f "CLAUDE_DIRECTOR_INSTANCE_ID=$id1" >/dev/null 2>&1 \
        && pgrep -f "CLAUDE_DIRECTOR_INSTANCE_ID=$id2" >/dev/null 2>&1; then
        break
    fi
    sleep 0.2
done

res=$(claude-director find-missing)
count=$(printf '%s' "$res" | jq -r '.count')
ids=$(printf '%s' "$res" | jq -c '.ids')
if [ "$count" != "0" ] || [ "$ids" != "[]" ]; then
    echo "FAIL: count=$count ids=$ids; want 0/[]"; exit 1
fi

# Neither row should be missing.
for id in "$id1" "$id2"; do
    state=$(claude-director get --claude-instance-id "$id" | jq -r '.state')
    if [ "$state" = "missing" ]; then
        echo "FAIL: row $id wrongly marked missing"; exit 1
    fi
done
echo "recov-3 ok: find-missing left both live rows alone"
```
