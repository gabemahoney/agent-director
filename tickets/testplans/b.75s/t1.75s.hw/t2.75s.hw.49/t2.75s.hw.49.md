# env-diff-1-all-callable-verbs-match

Pins Epic 3 (envelope-diff harness) AC: running `go test ./test/envelope-diff/...`
against a clean baseline exits 0, meaning every callable verb's CLI subprocess
envelope and in-process Client envelope are structurally identical after
normalization and nondeterministic-field suppression.

## Setup

No database pre-seeding required beyond what the test harness seeds per-verb
internally. The test binary needs a built CLI binary — the harness calls
`buildCLI(t)` to compile it from source. The source tree is available at
`/work/source` (read-only bind-mount); a writable working copy is created by the
script.

## Steps

1. Assert `go` is available in PATH.
2. Copy `/work/source` to `WORK=$(mktemp -d /tmp/envdiff1_XXXXXX)`.
3. `cd "$WORK"` and run `go test -v ./test/envelope-diff/...` to capture per-subtest `--- PASS` lines.
4. Capture exit code; assert it is 0.
5. Assert that stdout/stderr contains at least one `PASS` line.
6. Emit pass marker.

## Pass criteria

- `go test ./test/envelope-diff/...` exits 0.
- No `FAIL` lines appear in the test output.
- The test output names ≥15 callable verb subtest lines of the form `--- PASS: TestEnvelopeDiff_Success/<verb>`.

## Script

```bash
set -euo pipefail

if ! command -v go >/dev/null 2>&1; then
    echo "FAIL: go not found in PATH"
    exit 1
fi

WORK=$(mktemp -d /tmp/envdiff1_XXXXXX)
trap 'rm -rf "$WORK"' EXIT

cp -a /work/source/. "$WORK/"
cd "$WORK"

OUTPUT=$(go test -v ./test/envelope-diff/... 2>&1) || {
    echo "FAIL: go test -v ./test/envelope-diff/... exited non-zero"
    echo "$OUTPUT"
    exit 1
}

if echo "$OUTPUT" | grep -q '^--- FAIL'; then
    echo "FAIL: one or more subtests failed"
    echo "$OUTPUT"
    exit 1
fi

VERB_COUNT=$(echo "$OUTPUT" | grep -c 'PASS: TestEnvelopeDiff_Success/' || true)
if [ "$VERB_COUNT" -lt 15 ]; then
    echo "FAIL: expected >=15 verb subtests for callable verbs, got $VERB_COUNT"
    exit 1
fi

echo "env-diff-1-all-callable-verbs-match ok: all callable verbs passed ($VERB_COUNT verb subtests)"
```
