# env-diff-2-injected-drift-fails

Pins Epic 3 (envelope-diff harness) AC: the harness detects envelope drift.
When the CLI marshal path is perturbed to emit a stray JSON field that the
in-process Client does not produce, `TestEnvelopeDiff_Success` exits non-zero
and its output names the affected verb and the diverging field path. This case
proves the diff machinery catches regressions.

## Setup

A writable copy of the source tree is required so that source files can be
mutated and restored. A `trap` ensures restoration even on failure. No DB
pre-seeding is required.

## Steps

1. Copy `/work/source` to `WORK=$(mktemp -d /tmp/envdiff2_XXXXXX)`.
2. Register a `trap` to restore the patched file and clean up `$WORK` on exit.
3. Inject a stray field into `writeJSON` in `$WORK/cmd/agent-director/spawn_cmd.go`
   using Python in-place string replacement: before the `fmt.Fprintln` call,
   append `,"_stray_cli_field":"injected"` to the JSON bytes so the CLI emits
   an extra key the Client does not.
4. `cd "$WORK"` and run `go test -run TestEnvelopeDiff_Success ./test/envelope-diff/`.
5. Capture exit code; assert it is **non-zero** (the harness must fail).
6. Assert that the test output contains `_stray_cli_field` (the injected field name appears in the diff).
7. Emit pass marker confirming the harness caught the injected drift.

## Pass criteria

- `go test -run TestEnvelopeDiff_Success ./test/envelope-diff/` exits **non-zero**.
- The test output (stderr) contains the string `_stray_cli_field`.
- After the trap fires, the source file is restored to its original content.

## Script

```bash
set -euo pipefail

WORK=$(mktemp -d /tmp/envdiff2_XXXXXX)
TARGET_FILE="$WORK/cmd/agent-director/spawn_cmd.go"
ORIG_HASH=""

cleanup() {
    rm -rf "$WORK"
}
trap cleanup EXIT

cp -a /work/source/. "$WORK/"
cd "$WORK"

ORIG_HASH=$(sha256sum "$TARGET_FILE" | awk '{print $1}')

# Inject a stray field into writeJSON's output path.
# We find the line that calls fmt.Fprintln(w, string(b)) and prepend a
# byte-mangling step that appends ,"_stray_cli_field":"injected"} before the
# closing brace of the marshaled JSON.
python3 - <<'PYEOF'
import sys

path = 'cmd/agent-director/spawn_cmd.go'
code = open(path).read()
old = '	if _, err := fmt.Fprintln(w, string(b)); err != nil {'
new = (
    '\t// DRIFT-INJECT: stray field — removed by test cleanup\n'
    '\tb = append(b[:len(b)-1], []byte(`,"_stray_cli_field":"injected"}`)...)\n'
    '\tif _, err := fmt.Fprintln(w, string(b)); err != nil {'
)
if old not in code:
    print('FAIL: injection target not found in spawn_cmd.go', file=sys.stderr)
    sys.exit(1)
code = code.replace(old, new, 1)
open(path, 'w').write(code)
PYEOF

# Verify injection happened
if ! grep -q '_stray_cli_field' "$TARGET_FILE"; then
    echo "FAIL: injection did not land in $TARGET_FILE"
    exit 1
fi

# Run the success harness — it must fail because the CLI now emits a stray field.
TEST_OUT=$(go test -run TestEnvelopeDiff_Success ./test/envelope-diff/... 2>&1) && {
    echo "FAIL: expected go test to exit non-zero after drift injection, but it exited 0"
    exit 1
}

# The failure output must name the injected field.
if ! echo "$TEST_OUT" | grep -q '_stray_cli_field'; then
    echo "FAIL: test output does not mention '_stray_cli_field'; drift not surfaced"
    echo "$TEST_OUT"
    exit 1
fi

echo "env-diff-2-injected-drift-fails ok: harness caught injected stray field '_stray_cli_field'"
```
