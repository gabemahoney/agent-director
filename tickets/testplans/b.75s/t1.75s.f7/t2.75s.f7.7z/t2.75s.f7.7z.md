# install-2: re-running install is idempotent

Per SRD §16.2: re-running the install script is a no-op for the
operator. Second run must not add duplicate hook entries; the
versioned binary stays in place (or a fresh versioned file is added
without breaking the symlink).

## Setup

Per-case DB-reset.

## Steps

1. Run install.sh once.
2. Run install.sh again.
3. Verify SessionStart + SessionEnd each have EXACTLY one
   claude-director-prefixed hook entry.

## Pass criteria

- Both install runs exit 0.
- Hook entry counts stay at 1 each.

## Script

```bash
set -euo pipefail
TESTHOME=$(mktemp -d)

HOME="$TESTHOME" /opt/skills/install-claude-director/install.sh \
    --binary /usr/local/bin/claude-director --no-symlink >/dev/null
HOME="$TESTHOME" /opt/skills/install-claude-director/install.sh \
    --binary /usr/local/bin/claude-director --no-symlink >/dev/null

prefix="$TESTHOME/.claude-director/bin/claude-director"
ss=$(jq --arg p "$prefix" \
    '[.hooks.SessionStart[]?
      | select(any(.hooks[]?; (.command // "") | startswith($p)))] | length' \
    "$TESTHOME/.claude/settings.json")
se=$(jq --arg p "$prefix" \
    '[.hooks.SessionEnd[]?
      | select(any(.hooks[]?; (.command // "") | startswith($p)))] | length' \
    "$TESTHOME/.claude/settings.json")
if [ "$ss" != "1" ] || [ "$se" != "1" ]; then
    echo "FAIL: idempotency broken — SessionStart=$ss SessionEnd=$se (both want 1)"
    cat "$TESTHOME/.claude/settings.json" | jq
    exit 1
fi
echo "install-2 ok: re-run is idempotent (each event has 1 claude-director hook)"
```
