# subprocess-coverage-2-timeout

Per SRD §SR-6.1 / §SR-6.2 / §SR-6.5: when a subprocess fails to exit
within `callTimeoutMs`, the Client must send SIGTERM, wait up to 2 s
for a graceful exit, then send SIGKILL, and reject the call with
`ErrCallTimeout` carrying `verb`, `elapsedMs`, and `timeoutMs`.

The installed-tarball Client is exercised against a stub CLI binary
that unconditionally `sleep 5`s. With `callTimeoutMs=500`, the Client
sends SIGTERM at ~500 ms; `sleep` does not trap SIGTERM, so the
subprocess exits almost immediately and the SIGKILL fallback path is
not reached (the grace timer is cancelled by `proc.exited`). The
timeout codepath still throws `ErrCallTimeout` because `timedOut` is
checked before `signalCode`.

## Setup

- `${HOME}=$(mktemp -d)`.
- `bun install file://...tgz` into `${HOME}/proj`.
- Stage a stub CLI binary at
  `${HOME}/proj/node_modules/@agent-director/linux-x64/bin/agent-director`
  that runs `exec sleep 5`.

## Steps

1. Construct `new Client({storePath, createIfMissing:true, callTimeoutMs:500})`.
2. `client.version({})`. Expect rejection with `ErrCallTimeout`.
3. Assert `e.verb === "version"`, `e.timeoutMs === 500`, and
   `e.elapsedMs >= 400` (sanity bound — the timer fired) and
   `e.elapsedMs <= 5000` (we did not have to wait the full sleep).

## Pass criteria

- The call rejects within ~3 s (allowing for the 2 s SIGKILL grace as
  an upper bound) with `instanceof ErrCallTimeout`.
- `verb`, `timeoutMs`, `elapsedMs` fields are populated as specified.

exec sleep 5
STUB_EOF
chmod +x "$DEST/bin/agent-director"
