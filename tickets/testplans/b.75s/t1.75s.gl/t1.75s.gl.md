# Go client library (`go-lib`) testplan

t1 collector for the Go client library (`pkg/api` — Epic 4 of `b.qe2`). Each
t2 is a plain-shell case the Docker harness executes. Runs under
`make test-docker EPIC=go-lib`; the driver resolves the slug by
case-insensitive substring match against the t1 frontmatter title (see
`test/driver/run-testplan.sh`), so the canonical substring `go-lib` must
remain in the title verbatim.

## Sourced from

- Source Epic: `t1.qe2.9x` (pkg/api Go client library publish + README).
- Testplan-adding Task: `t2.qe2.9x.cz`.

## Driver mode

All cases use `DRIVER_MODE=shell`. Each t2 body has a single fenced `bash`
block executed by the driver after the per-case DB-reset fixture.

Cases 1–4 build against `/work/source` (the read-only source bind-mount).
Each case copies the source tree (or a working subset) to a writable temp
dir under `/tmp/` if a writable workspace is needed for `go build` / `go
test` output; `/work/source` itself is never written.

## Notes for the orchestrator

- Audit: `make test-docker EPIC=go-lib`; inspect the JSON stream
  case-by-case.
- All four cases need `go` in PATH (already present in `test/Dockerfile`).
  Case 4 also needs `sha256sum` (coreutils — already present).
- None of these cases use `agent-director spawn` or tmux directly. The
  db-reset fixture's `cd-*` session sweep is a no-op for all four cases.
- Case 4 explicitly creates a seed file at `~/.agent-director/state.db`
  inside the container BEFORE the smoke run so the "byte-identical" check
  has something to compare. Without that seed the snapshot is nil and the
  test trivially passes — the seed makes the assertion load-bearing.

## tmux session sweep

No `cd-*` sessions are created. The db-reset fixture's session sweep is a
no-op for these cases.
